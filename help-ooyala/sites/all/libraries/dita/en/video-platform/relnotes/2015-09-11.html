2015-09-11 Release

Notable changes for this release include updated documentation for Theme Builder, closed captions, and Player v3.

Documentation Releases

  • Supported Configurations for iOS: Added a note explaining that because iOS9 will turn on App Transport Security (ATS) by default, all apps that use the iOS playback SDK will stop working. To allow playback to happen, you need to disable ATS.
  • Programming Closed Captioning in Player V3 Using JavaScript (Deprecated): Added a new topic about getting and setting closed captions for Player v3.
  • Implementation Notes for Theme Builder: Updated the explanation of what Theme Builder features are supported for Flash (player customizations, playlists and pods) and HTML5 (playlists and pods only).
  • Implementing an iOS OOEmbedTokenGenerator: Updated the link to the repository containing the Ooyala token-related sample, updated the page to indicate the discussion is for iOS, and replaced the code samples and explanations.
  • About VisualOn Integrations: Added a link to the VisualOn and Secure Player sample apps.
  • Implementing Custom Controls for Android: Updated the link to the custom controls sample app.
  • docs.brightcove.com/apidocs-ooyala/: In the Player JavaScript API, the OO.ready() function is now included and described in the OO namespace.
  • Enabling HLS Playback on Android Web in Player V3 (Deprecated): Added a description of HLS support on Android 5.0 devices.
  • Player V3 Platform and Browser Support (Deprecated): Added description of fullscreen support for IE11.
  • Advanced: Closed Captions with the HTML5 Player V3 (Deprecated): Added a developer tutorial on how to use Player JavaScript to use close caption features with the Ooyala Player.
