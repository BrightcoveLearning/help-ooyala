2014-07-17 Release

Notable changes include the release of v.2.6.0 of the iOS and Android mobile SDKs.

Software Feature Releases

The following are new software release items.

iOS SDK v.2.6.0

Version 2.6.0 of the Ooyala Mobile SDK for iOS has been released. A notable change in version 2.6.0 is that Ooyala no longer provides Google IMA libraries alongside our integration. Please download these libraries from Google IMA directly. We have transitioned to a newer packaging script. If there is anything missing from an SDK package, please file an issue with Technical Support.

Full Release Notes for iOS SDK v.2.6.0

Android SDK v.2.6.0

Version 2.6.0 of the Ooyala Mobile SDK for Android has been released. A notable change in version 2.6.0 is that Ooyala no longer provides Google IMA libraries alongside our integration. Please download these libraries from Google IMA directly.

We have also released a VisualOn Integration, available to all Ooyala customers. Please contact your Customer Success Manager for more details.

Full Release Notes for Android SDK v.2.6.0

New “name_live” Widevine DRM Policy Property

The new DRM Policy property, “name_live”, allows customers to apply Widevine DRM policies to videos that they have transcoded themselves using their own Widevine portal. The DRM policy is applied during the Widevine license acquisition.

Documentation Release Items

The following are new documentation release items:

  • iOS SDK v.2.6.0

    • Mobile SDK v.2.6.0 for iOS —Updated iOS SDK documentation for version 2.6.0.
    • Integration with Google IMA on iOS —Updated “What You Need” section of the documentation about integration with Google IMA on iOS. You now need to download the Google IMA Libraries from the Google Developer website.

  • Android SDK v.2.6.0

    • Mobile SDK v.2.6.0 for Android—Updated Android SDK documentation for version 2.6.0.
    • Integration with Google IMA on Android—Updated “What You Need” section of the documentation about integration with Google IMA on Android. You now need to download the Google IMA Libraries from the Google Developer website.

  • VAST ad tag Embedded Parameter

    • VAST Ads Embedded Parameters—Added the clarification that in order to use the VAST ad tag embedded parameter an ad set must also be applied in Backlot and the ad set must contain a valid VAST ad tag.

  • New “name_live” DRM Policy Property

    • DRM Policies—Added documentation on the new “name_live” property.

  • Cross Device Resume (XDR) Support

    • Players: Supported Technologies and Devices—Updated documentation on what devices support cross device resume.

  • High Performance API Endpoint Description

    • High Performance API Endpoint—Updated the description of the high performance API endpoint.
