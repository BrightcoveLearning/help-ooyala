2014-10-13 Release

Notable changes include the release of v.3.0.0 of the iOS and Android mobile SDKs.

Software Feature Releases

Android Mobile SDK v.3.0.0

This release includes the following updates and improvements:

  • Per the FCC guidelines for TV ratings, TV rating watermark can now be applied to your videos for Android 2.3 and newer devices. TV ratings are applied at the asset level, and each asset can only have one TV rating. The TV rating watermark appears when the video starts playing and appears again when the video resumes after ad playback.
  • Cue points are visual markers that enable an end user to predict ad or segment breaks. This feature is active by default for Android 2.3 and newer devices, upon upgrading to Ooyala SDK v3.0 , however the Ooyala FreeWheel Integration libraries must also be updated.
  • The new Seek Over Ad Pods (SOAP) feature provides new functionality to play the last ad in queued sequence even when the user seeks past an ad in the player. This feature is active by default upon upgrading to Ooyala SDK v3.0, however the Ooyala FreeWheel Integration libraries must also be updated.
  • This release includes support for Google IMA beta 8.
  • This release includes new FreeWheel sample applications.
  • This release includes eHLS support for use with VisualOn video libraries. New sample embed codes are provided for testing.

iOS Mobile SDK v.3.0.0

This release includes the following updates and improvements:

  • Per the FCC guidelines for TV ratings, TV rating watermark can now be applied to your videos. TV ratings are applied at the asset level, and each asset can only have one TV rating. The TV rating watermark appears when the video starts playing and appears again when the video resumes after ad playback. This feature is available for use with iOS 6 and newer devices.
  • Cue points are visual markers that enable an end user to predict ad or segment breaks. This feature is enabled by default and is available for use with iOS 7 and newer devices. This feature is active by default upon upgrading to Ooyala SDK v3.0, however the Ooyala FreeWheel Integration libraries must also be updated.
  • The new Seek Over Ad Pods (SOAP) feature provides new functionality to play the last ads in queued sequence even when the user seeks past an ad in the player. This feature is active by default upon upgrading to Ooyala SDK v3.0, however the Ooyala FreeWheel Integration libraries must also be updated.
  • This release includes support for Google IMA beta 7.
