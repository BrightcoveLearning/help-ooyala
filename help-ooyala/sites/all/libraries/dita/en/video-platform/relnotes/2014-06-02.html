2014-06-03 Release

Notable changes include software updates to iOS SDK v. 2.4.0, Android SDK v. 2.4.0, Discovery, and documentation updates for iOS SDK v. 2.4.0, Android SDK v. 2.4.0, TinyPass Paywall Integration, Google IMA Ad Rules, Facebook Active Sharing, Theme Builder supported operating systems and browsers, DFXP, and country code update in Backlot.

Software Feature Releases

The following are new software release items.

iOS SDK v.2.4.0

Version 2.4.0 of the Ooyala Mobile SDK for iOS has been released. Version 2.4.0 includes:

Note : These changes may require you to update your integration.

  • Major updates to the Closed Captions UI and API to enable FCC 708 compliance for iOS mobile devices, allowing end-users to easily set preferences for closed captions, including language and appearance.

    To be FCC 708 compliant, you must link QuartzCore.framework and MediaAccessibility.framework to your binaries. You may mark MediaAccessibilty.framework as optional.

  • A new OOPlayerDomain class that must be used for all instances of OOOoyalaPlayerViewController, to improve the accuracy of code.

  • Improved Analytics ingestion and reporting accuracy, which requires adding the string ‘http://’ when providing a domain through the PlayerDomain class.

    Full Release Notes for iOS SDK v.2.4.0

Android SDK v.2.4.0

Version 2.4.0 of the Ooyala Mobile SDK for Android has been released. Version 2.4.0 includes:

Note : These changes may require you to update your integration.

  • Major updates to the Closed Captions UI and API to enable FCC 708 compliance for Android mobile devices, allowing end-users to easily set preferences for closed captions, including language and appearance.

  • New PlayerDomain class (new PlayerDomain(DOMAIN)) that must be used for all instances of OoyalaPlayerLayoutController and OptimizedOoyalaPlayerLayoutController, to improve the accuracy of code.

  • Improved Analytics ingestion and reporting accuracy, which requires adding the string ‘http://’ when providing a domain through the PlayerDomain class.

    Full Release Notes for Android SDK v.2.4.0

Discovery

Fixed a bug where Shared Labels and Specific Labels would not apply across HTML5 players for asset recommendations when implemented in the Backlot UI.
