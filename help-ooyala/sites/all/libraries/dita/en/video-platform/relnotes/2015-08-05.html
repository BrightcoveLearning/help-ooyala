2015-08-05 Release

Notable changes for this release include a new DRM attributes route in the Backlot API that generates a PlayReady key_id and content_key.

New Route to Generate PlayReady Attributes for DRM Protection

The Backlot API has a new DRM attributes route for generating a PlayReady key_id and content_key.

You now have a way to generate a PlayReady key_id and content_key through Ooyala APIs. This route will generate a random key id and content key for the asset and associate them with the video. If you use this route, you will not have to use a PATCH route to store these attributes in Ooyala's datastore.

Note: For security reasons this route will only work over https. API calls made over plain http will be rejected.

Documentation Releases

  • DRM Attributes for Remote Assets (Including Live Streams): Added documentation on the new route to generate a PlayReady key_id and content_key with the Backlot API.
  • Properties for Device Registration: Updated documentation on device nickname limitations.
  • Ooyala Ingest Server Endpoints: Updated documentation to list spare FTP and Aspera servers for customers to use when servers are slow or unresponsive.
