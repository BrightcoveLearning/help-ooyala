Chromecast Integration Glossary

Parent topic: Chromecast and Ooyala (Deprecated)

Cast Button

The button which invokes a menu to connect, control and disconnect from cast receivers.

Cast Menu

The menu that pops up when the cast button is clicked. The cast menu lets users connect, control and disconnect from cast receivers.

Chromecast Home Screen

The default screen when no receivers have been connected to the Chromecast dongle.

Custom Receiver Application

A verbose receiver application in the Ooyala Chromecast Integration that is used as a platform for your own customization. You can customize components of the custom receiver or use it “as-is”.

Default Receiver Application

A receiver application in the Ooyala Chromecast Integration that you can use “as-is”.

Mini Controller

Controls should appear when the user navigates away from the current content page or expanded controls. These controls provide instant access and a visible reminder for the currently casting video.

Receiver Application

The receiver application is the web page that embeds and loads the Ooyala V3 HTML5 player.

Receiver Application ID

After you register your application you will receive an application ID that your sender application must use to perform API calls. This lets the Chromecast device know which page to load, and so on.

Sender Application

The application of the ‘primary’ viewing device that is responsible for locating nearby Chromecast devices. Also used to refer to the sender application in the Ooyala Chromecast Integration that you can customize or use “as-is”.
