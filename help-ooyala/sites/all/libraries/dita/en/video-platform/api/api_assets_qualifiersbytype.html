Types of Assets and Correlation to /v2/assets Qualifiers

The /v2/assets route deals with all types of assets.

Note: For more information about Backlot REST API commands, see the Backlot API Reference.

When you create a new asset using the Backlot REST API, the asset_type property in the request body can specify any of the following values.

  • video
  • ad: An ad is simply a video, marked as an ad in the Backlot UI.
  • audio
  • remote_asset
  • youtube: A video destined for YouTube
  • live_stream (deprecated)
  • live_audio (deprecated)
  • channel (deprecated)
  • channel_set (deprecated)

After an asset is created with [POST] /v2/assets (as described in Creating an Asset: POST), in general you can update it in a variety of ways with [PATCH] /v2/assets. See the generalized process in Editing Asset Information: PATCH.

In addition, the /v2/assets route has qualifiers for working with assets in general or specific types of assets, such as associating labels, publishing rules, ad sets, and more. Exact use of these qualifiers is detailed in other sections.
