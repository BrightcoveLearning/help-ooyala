Assets

The /v2/assets route deals with all types of assets.

Note: For more information about Backlot REST API commands, see the Backlot API Reference.

When you create a new asset in the Backlot REST API, the asset_type property in the request body can specify any of the following values.

  • video
  • ad: An ad is simply a video, marked as an ad in the Backlot UI.
  • audio
  • remote_asset
  • youtube: A video destined for YouTube
  • live_stream (deprecated)
  • live_audio (deprecated)
  • channel (deprecated)
  • channel_set (deprecated)

After an asset is created with [POST] /v2/assets (as described in ), in general you can update it in a variety of ways with [PATCH] /v2/assets or change its associations with [PUT] /v2/assets. See Routes and Attributes Common to All Asset Types.

  • Routes and Attributes Common to All Asset Types
    The basic routes common to all assets have the same syntax, regardless of their type, in the Backlot REST API.
  • Asset Properties by Asset Types
    Some basic properties are common to all assets, regardless of their type, in the Backlot REST API.
  • Types of Assets and Correlation to /v2/assets Qualifiers
    The /v2/assets route deals with all types of assets.
  • Video and Audio Assets
    When you upload a video to Ooyala, Ooyala automatically transcodes it using the settings specified for your account and makes the videos available for playback through your player.
  • Remote Assets
    A remote asset is a piece of content that is hosted outside of Backlot. Instead of uploading, transcoding, and storing the content in Backlot, you simply upload its URL and other descriptive information so that Backlot can find the content when needed.
  • Uploading VR 360 Content
  • Live Streams (Deprecated)
    Live streaming is the delivery of video content in real time.
  • YouTube Assets
    A YouTube asset is a video hosted by YouTube and played through the Ooyala player.
  • Channel and Channel Set Assets (Deprecated)
    A channel is a collection of videos in a single player. A channel set is a collection of channels.
  • Custom Metadata
    You can add up to 100 name-value pairs to any type of asset.
  • Working with Closed Captions
    You can add closed captions to videos, ads, remote assets, or YouTube videos.
  • Associate Ad Sets with Assets
    You can associate ad sets with videos, live streams, and remote assets. For information about creating ad sets, refer to the Backlot User Guide.
  • Associate Publishing Rules with Assets
    You can associate publishing rules with videos, live streams, remote assets, YouTube videos, channels, and channel sets.
  • Associate Labels with Asset
    You can associate labels with videos, ads, live streams, remote assets, and YouTube videos.
  • Associate Player with Asset
    You can associate a player with videos, ads, live streams, remote players, and YouTube videos.
  • Query Construction
    To limit the amount of data returned in a GET call, you can specify where queries.
  • Deleted Assets
    Use this request to get information about assets that have been deleted.
  • DRM Attributes for Remote Assets (Including Live Streams)
    For remote assets protected by Digital Rights Management (DRM) systems, you need to associate information about that system.
