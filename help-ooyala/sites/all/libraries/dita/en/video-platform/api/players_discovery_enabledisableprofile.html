Enabling and Disabling Discovery Profiles for a Player

To enable discovery profiles for player set the discovery_enabled property to true in the request body and use the PATCH method. The following request and response examples show how to enable a discovery profile associated with a player.

Note: This is equivalent to the Turn On Discovery For This Player setting in the Backlot UI, described in About Creating and Managing Profiles with the Backlot UI.

Request:

[PATCH] /v2/players/{player_id}/discovery/profile
{
"discovery_enabled": "true"
}

Response:

...
"optimization_goal": "play_time",
"compute_trending_labels": "false"
},
"discovery_enabled": "true"
}
...

To disable discovery profiles for player set the discovery_enable property to false in the request body and use the PATCH method. The following request and response examples how to disable a discovery profile associated with a player.

Request:

[PATCH] /v2/players/{player_id}/discovery/profile
{
"discovery_enabled": "false"
}

Response:

...
"optimization_goal": "play_time",
"compute_trending_labels": "false"
},
"discovery_enabled": "false"
}

Possible HTTP response codes and bodies:

  • 200: The response will be the JSON object of the discovery profile.
  • 40x: The response body will be an error message in JSON.
