DRM Attributes for Remote Assets

If you package content using Digital Rights Management (DRM) and want Ooyala to serve licenses for that content, you will need to set some attributes per asset.

These attributes need to be applied only to remote assets.

Fetch Attributes for an Asset and Given DRM Type

[GET]/v2/assets/asset_id/drm_attributes/drm_type

Adding or Updating an Attribute

[PATCH]/v2/assets/asset_id/drm_attributes/drm_type{
   properties
}

Route Attributes

The following table describes all attributes that can be expressed through the route.

Route Attribute Description
asset_id See Routes and Attributes Common to All Asset Types for a description of this attribute.
drm_type

The type of DRM the asset is associated with.

Type: String

Valid Values: playready | fps

Default: None

Example: playready

Properties

The following table describes the properties you need to set for each DRM type.

drm_type Property Description
fps fps_content_key_version The FairPlay Streaming identifier for the asset.
playready key_id The PlayReady identifier.

Type: String

Default: None

Example: "1234"
playready content_key_version The PlayReady content key. The version of the parameter name is an integer that matches the version in the license acquisition URL for the specific asset.

For example, if the license acquisition URL is: 

http://player.ooyala.com/sas/drm2/jqbkH9m/UPHGNsr/playready/ooyala/version/3
Then the corresponding parameter name is content_key_3.

Type: String

Default: None

Example: "content_key_2"
