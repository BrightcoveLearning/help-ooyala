Upload a Watermark Image Using the Backlot REST API

You can upload a partially transparent image that displays during content playback.

Note: For more information about REST API commands for the Ooyala player, see the Backlot API Reference.

Upload Watermark

Note: This functionality applies to Ooyala Player V4 and Player V3 (Deprecated).
[PUT]/v2/players/player_id/watermark
 <file_contents>

Remove Watermark from a Player

Note: This functionality applies to Ooyala Player V4 and Player V3 (Deprecated).
[DELETE]/v2/players/player_id/watermark

Route Attributes

The following table describes all attributes that can be expressed through the route.

Route Attribute Description
player_id

The ID of the player.

Type: String

Default: None

Example: /players/IzNnllMjphu2XF3_UgPROoCi9B2BwtSg

Examples

This example uploads a watermark image for a player: 

[PUT]/v2/assets/IzNnllMjphu2XF3_UgPROoCi9B2rtWs/watermark

Backlot returns a 200 response.

Note: Try out the code samples using your account credentials in the Ooyala Scratchpad. To launch the Scratchpad, go to Ooyala API Scratchpad. For information about using the Scratchpad, see Practice Making Requests with the Scratchpad.
