Setting Permissions for Android Apps

Learn about setting permissions in the manifest file for your Ooyala Mobile SDK for Android app.

As of release 4.1.0 the following permissions must be set in the manifest for your Android apps:

android.permission.INTERNET Provides the mandatory permissions to access network streams.
android.permission.ACCESS_WIFI_STATE Provides the mandatory permissions to select the optimal stream for advertisements.
android.permission.ACCESS_NETWORK_STATE Provides the mandatory permissions to access network status information. 
<?xml version="1.0" encoding="utf-8"?>
   <manifest xmlns:android="http://schemas.android.com/apk/res/android"
   package="com.ooyala.android.sampleapp"
   android:versionCode="1"
   android:versionName="1.0" >
   
   <uses-sdk android:minSdkVersion="9" />
       <uses-permission android:name="android.permission.INTERNET"/>
    <uses-permission android:name="android.permission.ACCESS_WIFI_STATE" />
    <uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" />
   ...
Note: Make sure the minimum SDK version is set to 9. 
<uses-sdk android:minSdkVersion="9"/>
