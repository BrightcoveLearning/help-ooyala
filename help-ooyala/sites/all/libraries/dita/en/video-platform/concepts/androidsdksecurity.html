About Access Control

The SDK is designed to work a minimum of security constraints. The following options are available for restricting access,

  • Restrict the playback of videos to an Internet domain configured in Ooyala platform, either through the Backlot UI Syndication Controls page or with the publishing rule routes detailed in the Backlot API Reference.
  • Use Ooyala Player Token (OPT) to control access to individual assets (videos). For more information see, Ooyala Player Token for Player V3 (Deprecated).
Note: Other access control requirements are beyond the scope of the mobile SDK but is available through Ooyala, such as Digital Rights Management (DRM) systems or other configurations.

To use OPT, implement the EmbedTokenGenerator interface. For example usage see the sample applications listed in Sample Applications for the Android Mobile SDK. For documentation about OPT, see Ooyala Player Token for Player V3 (Deprecated).
