Player Parameters

You can use query string parameters and embedded parameters to modify the player or player experience.

The Ooyala Player has two types of parameters that you can use to modify the player or player experience. The primary difference between the two parameter types is in where you place them:
  1. Query string parameters: in the URL string that invokes the Ooyala Player
  2. Embedded parameters: within the player DIV container.
Note: The Ooyala legacy player API featured a set of Query String Pass-Thru Parameters that enabled you to pass tags of ad server and ad networks to the Ooyala player. While not all pass-through parameters are supported in Player, the following sections describes the ones that are supported.
