Discovery Best Practices

Consider these best practice recommendations to maximize the benefits of your Discovery implementation and increase engagement on your site or app.

Feature Benefits
Labels Use labels to tailor Discovery recommendations on your site or app by excluding or including content categories. For example, if a consumer is watching a video about tennis, you can restrict recommendations to other similarly labeled tennis videos. For more information, see Labels.
Editorial Content Promotion Promote drive traffic to key content by specifying a label for any position in the Discovery tray or specifying results using the Discovery APIs. For example, a large media site with both VOD and Live concert streaming uses this feature to drive more traffic from their VOD views to Live events. For more information, see Discovery Profiles and Discovery Editorial Query String Parameters.
Sequential Content Promotion Recommend series based content sequentially using custom metadata keys to identify related content. For example, specify custom metadata so that Part 2 of a series is the first recommendation provided when your Part 1 is the active asset. For more information, see Custom Metadata.
Discovery APIs Use the Discovery APIs to populate any section of your site or app with content that has seen high engagement. For example, include a menu with videos that have gained significant momentum in the last 1 hour in the country of the consumer. For more information, see Trending Content Recommendations.
Permalink URL Increase monetization, and improve tracking and social compatibility, by configuring lifted Discovery recommendations videos to open in the dedicated page, specified by its Permalink URL (in the asset metadata), in a new browser tab. This setting is configured from the Publish > Discovery tab, in the Backlot user interface. For more information, see Managing Remote Asset Details and Configuring Discovery in the Backlot UI.
Discovery Profiles Use discovery profiles to creating and manage settings bundles based on Discovery API calls. For example, create a ‘December Spotlight Promo’ discovery profile with the first recommended video always being one with the ‘Spotlight’ label that month. For more information, see Discovery Profiles.
Editorial Query String Parameters Create ad hoc Discovery results using editorial query string parameters in your API requests. This options allows you to include, exclude, limit, and join content based on labels. For example, dynamically show popular content with the ‘Country Music’ label if a consumer went to that page on your site. For more information, see Discovery Editorial Query String Parameters.
Discovery Insights Reporting Use the reporting features to discover usage patterns to and leverage content relationships. While many of your asset relationships are obvious based on subject matter, less obvious relationships are revealed from usage patterns by Discovery’s machine learning algorithms. For more information, see Discovery Insights Reports (No Longer Available).
Discovery Profile API Update existing discovery profiles from your internal tool or UI by integrating with the Discovery Profile API. For more information, see the Discovery REST API Reference.
