ビデオストリーム情報の表示

アカウント設定に基づいて、ビデオはさまざまな解像度、コーデック、およびビットレートを使用してトランスコードされます。

注意： Backlot REST APIコマンドについて詳しくは、以下を参照してください。 バックスロットAPIリファレンス.

用語 ストリーム この文脈では、あなたのビデオをトランスコードするためにシステムによって開始される内部からOoyalaへのプロセスを意味します。

注意： これらのストリームを制御または操作することはできません。 それらはシステムによって制御されます。 あなたはそれらの名前を見ることができるだけで、変更することはできません。

ビデオのストリームを表示するには：

ストリームを表示するには、 GET / v2 / assets、アセットID、および /ストリーム 修飾子 ABRストリームは、「stream_type」フィールド（「stream_type」：「abr」）で区別できます。 次の例は、のストリームを表示します。 しぶき 資産： 
[GET] / v2 / assets / JxbzdkMjqB / streams

バックスロット 次のような応答を返します。

[{"profile"： "baseline"、 "video_height"：360、 "is_source"：false、 "video_width"：640、 "file_size"：17937644、 "audio_codec"： "aac"、 "video_codec"： "h264"、 "average_video_bitrate"：300}、{"プロファイル"： "ベースライン"、 "ビデオの高さ"：360、 "is_source"：false、 "ビデオの幅"：640、 "ファイルのサイズ"：41716636、 "オーディオのコーデック"： "aac"、 "video_codec" "：" h264 "、" average_video_bitrate "：900}]
注意： Ooyala Scratchpadでアカウントの認証情報を使用してコードサンプルを試してみてください。 スクラッチパッドを起動するには、にアクセスしてください。 Ooyala APIスクラッチパッド。 スクラッチパッドの使用方法については、 スクラッチパッドを使った練習の依頼.
