プレイヤーV3のOoyalaプレイヤートークン（廃止予定）

注意： このトピックでは、推奨されなくなったPlayer V3のOoyala Playerトークン機能について説明します。 Player V4については、 プレイヤーV4のOoyalaプレイヤートークン.

Ooyalaは、コンテンツ発行者およびプロバイダに、Ooyalaのコンテンツ保護ソリューションの1つであるOoyala Player Tokenを使用してデジタルコンテンツを保護する機能を提供します。

注意： あなたのOoyalaアカウントがこの機能を含んでいる場合にのみ利用可能です。 Ooyala Player Tokenを有効にするには、アカウントマネージャに連絡してください。

Ooyalaプレーヤートークンを使用して、あなたのデジタルコンテンツを不正に使用しようとする可能性のあるユーザーからあなたを保護することができます。 たとえば、Backlotで生成された埋め込みプレーヤーのJavascript（.js）スクリプトタグを他人に許可なく配布したり、他の同様の再生攻撃を試みることができます。

そのような不正使用を防ぐために、OoyalaはOoyala Player Tokenを提供しています。これは、コンテンツをこれらの種類のリスクから保護するのに役立ちます。 この機能を使用して、収益化されたコンテンツを保護し、不正な配布を防ぐことができます。

Ooyalaプレーヤートークン機能はあなたに安全なプロセスを提供します。 認証する それがあなたのデジタルコンテンツを再生することを可能にするそれの前のプレーヤー。 トークンベースの認証では、クライアントプレーヤーが認証されるまでデジタルコンテンツをダウンロードまたは再生できません。 クライアントプレーヤーが認証されると、クライアントプレーヤーは要求されたコンテンツを再生することを許可されます。

Ooyalaプレーヤートークンの使用

次の手順では、基本的なWebアプリケーションでOoyala Playerトークンを使用する方法について説明します。
  • ステップ1：BacklotでOoyalaプレイヤートークンを設定する
  • ステップ2：基本HTMLページを作成する
  • ステップ3：embedToken埋め込みパラメータの指定
  • ステップ4：トークン要求値を作成して割り当てる
  • 完全なWebの例：再生の許可とトークンの取得

Ooyala Player JavaScript APIを使用してWebアプリケーションを構築する方法の詳細については、以下を参照してください。 Player V3 JavaScript APIを使用した開発（非推奨）.s

ステップ1：BacklotでOoyalaプレイヤートークンを設定する

バックスロットUI を設定してコンテンツを保護するためのオプションを提供します。 シンジケーション このシンジケーショングループにコンテンツアセットを追加するグループフラグ。 これにより、プレーヤーを実行するようにWebページを設定し、認証トークンを送信することができます。 Backlotでコンテンツを保護するには、次の手順に従ってください。

Ooyalaプレーヤートークンを使用した再生コンテンツの保護

Backlotでコンテンツを保護するには、次の手順に従ってください。

  1. Backlotアカウントにログインします。
  2. 「公開」タブをクリックして、「シンジケーション制御」タブを選択します。
  3. シンジケーショングループからビデオを選択するか、シンジケーショングループを設定してから自分のビデオを選択します。
  4. [シンジケーションコントロール]ペインで、[ Ooyalaプレイヤートークンを要求する チェックボックスをオンにします。
  5. （オプション）[有効期限]フィールドに、表示セッションの有効期限（秒）を設定します。 時間を指定しない場合は、推奨されるデフォルトの10分が使用されます。 このフィールドは数値入力のみを受け入れます（たとえば、600）。 同じオリジンポリシーで配布が保護されるため、必要に応じてトークンの有効期限を長くすることができます。 有効期限の詳細については、 Ooyalaプレイヤートークンの有効期限.
  6. [管理]タブをクリックし、[埋め込み]タブを選択します。
  7. ストリーミングを停止する場合は上部の コピー プレーヤーのJavaScriptを取得するためのボタン <script> あなたのWebページに貼り付けるためのタグ。 このコンテンツは、必須のオプションとしてOoyalaプレーヤートークンを指定したシンジケーショングループに含まれている必要があります。
