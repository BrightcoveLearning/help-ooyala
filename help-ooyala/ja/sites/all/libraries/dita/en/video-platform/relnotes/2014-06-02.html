2014-06-03リリース

注目すべき変更には、iOS SDK v。2.4.0、Android SDK v。2.4.0、Discovery、およびiOS SDK v。2.4.0、Android SDK v。2.4.0、TinyPassペイウォール統合、Google IMA Ad Rules、Facebook Active Sharing、テーマのソフトウェアアップデートが含まれます。 Builderは、オペレーティングシステムとブラウザ、DFXP、およびBacklotの国コードの更新をサポートしていました。

ソフトウェア機能リリース

以下は新しいソフトウェアリリース項目です。

iOS SDK v.2.4.0

iOS用Ooyala Mobile SDKのバージョン2.4.0がリリースされました。 バージョン2.4.0には以下が含まれます。

注 注：これらの変更により、統合の更新が必要になる場合があります。

  • クローズドキャプションのUIとAPIのメジャーアップデートにより、iOSモバイルデバイス用のFCC 708準拠が可能になり、エンドユーザーは言語や外観などのクローズドキャプションの設定を簡単に設定できるようになりました。

    FCC 708に準拠するには、リンクする必要があります QuartzCore.framework そして MediaAccessibility.framework あなたのバイナリに。 あなたがマークすることがあります MediaAccessibilty.framework オプションとして。

  • 新しい OOPlayerDomain のすべてのインスタンスに使用する必要があるクラス OOOoyalaPlayerViewController、コードの精度を向上させるために。

  • Analyticsの取り込みとレポートの正確性が向上しました。文字列を追加する必要があります。 'http：//' を通じてドメインを提供する場合 PlayerDomainクラス.

    iOS SDK v.2.4.0のフルリリースノート

Android SDK v.2.4.0

Android用Ooyala Mobile SDKのバージョン2.4.0がリリースされました。 バージョン2.4.0には以下が含まれます。

注 注：これらの変更により、統合の更新が必要になる場合があります。

  • クローズドキャプションUIおよびAPIのメジャーアップデート。Androidモバイルデバイス向けのFCC 708準拠を有効にし、エンドユーザーが言語や外観などのクローズドキャプションの設定を簡単に設定できるようにします。

  • 新しい PlayerDomainクラス （新しい PlayerDomain（ドメイン）のすべてのインスタンスに使用する必要があります） OoyalaPlayerLayoutController そして OptimizedOoyalaPlayerLayoutController、コードの精度を向上させるために。

  • Analyticsの取り込みとレポートの正確性が向上しました。文字列を追加する必要があります。 'http：//' を通じてドメインを提供する場合 PlayerDomainクラス.

    Android SDK v.2.4.0のフルリリースノート

Discovery

Backlot UIで実装した場合、共有ラベルおよび特定ラベルがHTML5プレーヤー間でアセットの推奨事項に適用されないというバグを修正しました。

ドキュメントリリース項目

以下は、新しいドキュメントのリリース項目です。

  • Google IMAの広告ルール

    Google IMAの広告ルール- 広告ルールとGoogle IMAに関する情報を追加しました（各プラットフォームでサポートされている広告の種類を含む）。

  • Tinypassを使用した新しいペイウォールソリューション

    Tinypassと統合する - Tinypassを使用した新しい軽量ペイウォールソリューションに関するドキュメントを追加しました。 このペイウォールソリューションは、FlashとHTML5に広範なデバイスの適用範囲を提供します。

  • Facebookのアクティブ共有設定

    アクティブ共有設定 - メタタグコードの例を更新しました。

    テーマビルダーでサポートされているオペレーティングシステムとブラウザ

    サポートされているオペレーティングシステムとブラウザ - テーマビルダーでサポートされているオペレーティングシステムおよびブラウザに関するドキュメントを追加しました。

  • iOS SDK v.2.4.0

    • iOS用Mobile SDK v.2.4.0- 新しいトピックページ「はじめに：環境の設定」を追加しました。
    • iOS用Mobile SDK v.2.4.0- 新しいトピックページ「iOS FCC 708クローズドキャプションのコンプライアンス」を追加しました。iOS用のOoyalaモバイルSDKが追加されました 連邦通信委員会（FCC）708準拠 クローズドキャプションのサポート この機能は、エンドユーザーがクローズドキャプションをプレーヤーにどのようにレンダリングするかを簡単に操作できるようにするために、FCCで規定されているユーザーコントロールをサポートしています。
    • iOS上のFreeWheelとの統合 - 最新のiOS SDKドキュメント。 iOS用FreeWheel SDK URLのリンク切れを修正しました。 FreeWheelライブラリフォルダへの参照を削除しました。 FreeWheel Headersファイルの記述を修正しました。

  • Android SDK v.2.4.0

    • Android用Mobile SDK v.2.4.0- バージョン2.4.0のAndroid SDKドキュメントを更新。
    • Android上のFreeWheelとの統合 - アップデートされたAndroid SDKのドキュメント。 Android用FreeWheel SDK URLのリンク切れを修正しました。

  • DFXP

    • クローズドキャプションのDFXP形式- バージョンサポート情報でDFXPドキュメントを更新しました。

  • Backlotのベトナム国コード

    • 国と場所のコード- Backlotのベトナムの国名が「Viet Nam」であるというメモを追加しました。
