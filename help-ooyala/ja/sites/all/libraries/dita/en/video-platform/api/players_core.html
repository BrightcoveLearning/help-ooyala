Backlot REST APIを使用したプレーヤーの管理

プレーヤーを作成してアカウント内の1つ以上のアセットに割り当てると、再生オプションとコンテンツの外観が決まります。

注意： Ooyalaプレーヤー用のREST APIコマンドについて詳しくは、以下を参照してください。 バックスロットAPIリファレンス.
注意： デフォルトのプレーヤーを変更するには is_default）、単にデフォルトとしてあなたが欲しい新しいプレーヤーを指定します。 以前のデフォルトを「設定解除」しないでください。

プレーヤーを作成する

[POST] / v2 / players {
   プロパティ

}

プレーヤーのリスト

[GET] / v2 / players

プレイヤー情報を見る

[GET] / v2 / players /player_id

プレイヤーを削除する

[削除] / v2 / players /player_id

ルート属性

次の表は、ルートを通じて表現できるすべての属性を示しています。

ルート属性 説明
player_id

プレイヤーのID プレイヤーのリストを取得するには、 /選手 ルート。

タイプ：文字列

デフォルト：なし

Example: /players/b4f1a93b779145b4a9c8ef15f8502345
