ウォッチリスト内でアイテムの位置を移動する

このエンドポイントは、指定されたアイテムを現在の位置から指定されたウォッチリスト内の別の位置に移動します。

HTTPリクエスト

PUT https://player.ooyala.com/watchlist/v3/<watchlist_id>/items/<item_id>?account_token= <account_token> -d 'position = "<新しい位置>"'

クエリパラメータ

パラメーター 説明 タイプ 必須
api_version でなければなりません v3. 文字列 はい（URL）
watchlist_id 更新するアイテムを含むウォッチリストのID。 ユーザーのアカウントに関連付けられている既存のウォッチリストにリンクされている有効なUUIDである必要があります。 指定しない場合、アイテムはユーザーのデフォルトのウォッチリストで更新されます。 UUIDv4または文字列 いいえ（URL）
item_id 取得するアイテムのID。 指定されたウォッチリスト内の有効な項目（負でない整数値）でなければなりません。 文字列 はい（URL）
account_token 特定のエンドユーザに関連付けられているウォッチリストを参照するためにウォッチリストサービスによって使用されます（UUIDによって識別されます）。 詳しくは、 OoyalaアカウントトークンAPI. 文字列 はい（URL本体）
位置 アイテムを移動する新しい位置。 指定されたウォッチリスト内の有効な位置でなければなりません。 アイテムが移動されると、指定されたウォッチリスト内の他のすべてのアイテムの位置は自動的に更新されます。
  • 新しい位置の値が指定されたウォッチリストの項目数を超える場合、新しい位置は項目リストの最後のインデックスと見なされます。 たとえば、ユーザーのウォッチリストに5から0までの位置を占める4アイテムが含まれている場合、リクエストが3から10の位置にアイテムを移動しようとすると、そのアイテムは実際には4（10ではなく）に配置されます。
  • 指定された位置が10進数（3.6など）の場合、整数部分が使用され、小数点の右側の桁は無視されます。
 文字列（数値） はい（URL本体）

リターンコード

コード 理由 説明
200 OK PUT要求は成功しました。 アイテムは正常に更新されました。
400 リクエストエラー 次の問題が原因で失敗しました。
  • パラメーターが欠落しているかフォーマットが正しくありません。
  • 指定されたアイテムはウォッチリストに存在しません。
  • 指定された watchlist_id 空ではなく、存在しません。
  • 指定された 位置 正の整数ではありません。
401 無断 指定された account_token 無効です。
404 見つかりませんでした サーバーが停止しています。
500 内部サーバーエラー 内部サーバーエラーが発生しました。
