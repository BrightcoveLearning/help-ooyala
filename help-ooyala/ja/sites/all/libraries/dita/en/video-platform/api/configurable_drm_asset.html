DRMポリシーとアセットの関連付け

DRMポリシーを1つ以上の特定の資産に関連付けることができます。 DRMポリシーがアセットに関連付けられていない場合は、デフォルトのポリシーが使用されます。

注意： Backlot REST APIコマンドについて詳しくは、以下を参照してください。 バックスロットAPIリファレンス.

資産のポリシーを取得する

[GET] / assets /asset_id/ drm_policy

ポリシーとアセットの関連付け

[PUT] /資産/asset_id/ drm_policy /drm_policy_id

資産からポリシーを削除する

[削除] / assets /asset_id/ drm_policy

例

この例では、 86ff97ae7c81495eacbd9a01feff0f21 IzNnllMjphu2XF3_UgPROoCi9B2rtWs資産を持つポリシー： 

[PUT] / v2 / assets /asset_id/ drm_policy /drm_policy_＃

バックスロット 200応答を返します。

注意： Ooyala Scratchpadでアカウントの認証情報を使用してコードサンプルを試してみてください。 スクラッチパッドを起動するには、にアクセスしてください。 Ooyala APIスクラッチパッド。 スクラッチパッドの使用方法については、 スクラッチパッドを使った練習の依頼.
