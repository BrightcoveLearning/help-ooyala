Ooyalaプレイヤートークン

The Ooyala Player Token offers a fault tolerant design that enables you to provision your users with a session token, without which playback is prohibited. Both the cookie and the player tag used to render the player will expire, such that, with the correct configuration, replay attacks will also expire prior to propagation. The Ooyala Player Token consists functionally of two units: an expiring token request made through the player as soon as it is initialized and an expiring token object response. The player-hosting page sets the former via module parameters, and Ooyala returns the latter to the client. These two components of the Ooyala Player Token enable you to successfully authorize users to play video or audio assets and shield against unauthorized use of the content.

The token request consists of an authorization URL that you craft on the server-side. It contains the following elements:
