2015-05-01リリース

このリリースでの注目すべき変更には、AndroidおよびiOS用のMobile SDK 4.1.0リリースが含まれます。

Android用Mobile SDK 4.1.0リリース

4.0.1をリリース4 / 15をリリース

アップデートと機能強化

このリリースには、以下の更新および機能拡張が含まれています。

  • Ooyalaは、PlayReady HLSをサポートして、AndroidおよびiOSデバイス上の高品質コンテンツのコンテンツ保護要件を満たします。 このサポートには、コンテンツを保護するために連携して機能する次の包括的なコンテンツ保護機能が含まれています。

    • PlayReady統合によるコンテンツ認証
    • Ooyala Secure Playerトークンを使用したユーザー認証
      注意： この機能の設定と使用方法については、 プレイヤーV3のOoyalaプレイヤートークン（廃止予定）.
    • 許可要求を管理するための許可API
      注意： Authorization APIの詳細については、 プレーヤV3用プレーヤ認可API.

  • VisualOn APIには、以下の機能を持つビルダークラスが含まれています。

    • setupperBitrateThreshold
    • setlowerBitrateThreshold
    • setinitialBitrate
    • setInitialBufferingTime
    • setPlaybackBufferingTime
  • VisualOn APIに、バッファリングの開始/終了の追加および権利の開始/終了の通知が追加されました。

iOS用Mobile SDK 4.1.0リリース

5 / 1 / 15をリリースしました

アップデートと機能強化

このリリースには、以下の更新および機能拡張が含まれています。

クローズドキャプションメソッドをより簡単に変更できるようにAPIが更新されました。 次の2つのプロパティは、 OOOoyalaPlayer 〜へ OOOoyalaPlayerViewController:

  • closedCaptionsLanguage
  • OOClosedCaptionsStyle

解消された問題

このリリースでは、以下の問題が解決されています。

  • Widevineストリームが暗号化されたループバックを使用して再生されたときにプレーヤーがクラッシュしました。
  • そのプレイヤーは、AirPlayを使ったWidevineミッドロール広告で断続的なスクラバーの問題を抱えていた。
  • Playready HLSを使用すると、すべての広告タイプで間欠的な再生の問題が発生しました。
  • フリーホイール VideoView コンテンツの開始時ではなく、プレロール広告中に発生したイベント。
  • PlayReady HLSコンテンツの再生中にスクラバーが機能しませんでした。

ドキュメントリリース

  • Player V3 JavaScript APIパラメータ（廃止予定）：Ooyala V3 Player APIのすべてのクエリパラメータと埋め込みパラメータをまとめた新しいテーブルを追加しました。
  • 多機能サンプルの場所 https://github.com/ooyala/code-samples PHP関連のエラーに対する修正が含まれるように更新されました。 これらのサンプルは、Ooyala Playerおよびその他のテクノロジ用の新しく改善されたOoyalaコードサンプルに置き換えられる予定です。
  • でのPlayer v3 APIリファレンス docs.brightcove.com/apidocs-ooyala/ シーク、一時停止、および再生方法の説明を含むように更新されました。 ドキュメントには、seekコマンドを使用するときにBUFFEREDイベントを監視するように開発者にアドバイスすることが記載されています。
