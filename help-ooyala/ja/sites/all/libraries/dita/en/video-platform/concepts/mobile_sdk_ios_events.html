イベントでの作業

プレーヤオブジェクトのイベント通知を監視する必要があります。

以下に示すように、最初に通知オブジェクトを設定する必要があります。 イベントはヘッダーファイルに列挙されています ヘッダ/ OOOoyalaPlayer.h 説明コメント付き： 
extern NSString const OOOoyalaPlayerTimeChangedNotification; / *再生ヘッドの時間が変更されたときに発生します* / extern NSString const OOOoyalaPlayerStateChangedNotification; / *プレーヤーの状態が変更されたときに発生します* / extern NSString const OOOoyalaPlayerContentTreeReadyNotification; / *コンテンツツリーのメタデータの準備ができてアクセス可能になったときに発生します* / extern NSString const OOOoyalaPlayerAuthorizationReadyNotification; / *認証ステータスが準備完了でアクセス可能になったときに発生します。* / extern NSString const OOOoyalaPlayerPlayStartedNotification; / *再生開始時に発生します* / extern NSString const OOOoyalaPlayerPlayCompletedNotification; / *再生が完了したときに発生します* / extern NSString const OOOoyalaPlayerCurrentItemChangedNotification; / *現在の項目が変更されたときに発生します。* / extern NSString const OOOoyalaPlayerAdStartedNotification; / *広告の再生が開始されたときに発生します。* / extern NSString const OOOoyalaPlayerAdCompletedNotification; / *広告の再生が完了したときに発生します* / extern NSString const OOOoyalaPlayerAdSkippedNotification; / *広告がスキップされたときに発生します* / extern NSString const OOOoyalaPlayerErrorNotification; / *エラーが発生したときに発生します。* / extern NSString const OOOoyalaPlayerAdErrorNotification; / *広告の再生中にエラーが発生したときに発生します。* / extern NSString const OOOoyalaPlayerMetadataReadyNotification; / *コンテンツメタデータにアクセスする準備ができたときに発生します* /

例については、 iOS Mobileのエラー処理.
