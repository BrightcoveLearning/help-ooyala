共通属性とクエリ文字列パラメータ（廃止予定）

これらの属性とクエリ文字列パラメータは、Analyticsルートで使用されます。

警告： v2 Analytics APIは非推奨になりました。 見る ウーヤラIQアナリティクス 新しいv3 Analytics APIの詳細については、

ルート属性

次の表は、経路のすべての属性について説明しています。

ルート属性 説明
asset_id 資産のIDを指定します。

タイプ：文字列

デフォルト：なし

例：

/ v2 / analytics / reports / asset / fjkl3s23fdsf2 / Engagement

シティ

情報都市を返します。

タイプ：文字列

デフォルト：上位10都市を返します。

例：

[GET] /v2/analytics/reports/account/performance/cities/2011-08-01...2011-08-08（全都市）

[GET] /v2/analytics/reports/account/performance/countries/us/regions/california/cities/2011-08-01...2011-08-08（カリフォルニアのすべての都市）

[GET] /v2/analytics/reports/account/performance/countries/us/regions/california/cities/san+francisco/2011-08-01...2011-08-08（サンフランシスコのみ）

[GET] / v2 / analytics / reports / asset /asset_id/performance/cities/2011-08-01...2011-08-08（指定資産、すべての都市）

[GET] / v2 / analytics / reports / asset /asset_id/performance/countries/us/regions/california/cities/2011-08-01...2011-08-08（指定資産、カリフォルニア内のすべての都市）

[GET] / v2 / analytics / reports / asset /asset_id/performance/countries/us/regions/california/cities/san+francisco/2011-08-01...2011-08-08（指定資産、サンフランシスコのみ）

国

国に関する情報を返します（大文字と小文字を区別しません）。

タイプ：文字列

デフォルト：上位10カ国を返します。

有効な値：有効な国別コードのリストについては、2文字を参照してください。 ISO-3166 1 国コード 未知の国では、 バックスロット 使用されます u0.

例：

[GET] /v2/analytics/reports/account/performance/countries/2011-08-01...2011-08-08（すべての国）

[GET] /v2/analytics/reports/account/performance/countries/us/2011-08-01...2011-08-08（米国のみ）

[GET] / v2 / analytics / reports / asset /asset_id/performance/countries/2011-08-01...2011-08-08（すべての国で指定された資産）

[GET] / v2 / analytics / reports / asset /asset_id/performance/countries/us/2011-08-01...2011-08-08（米国でのみ指定された資産）
日付範囲

日付範囲を指定します。

タイプ：文字列

デフォルト：なし

例：

[GET] / v2 /アナリティクス/レポート/アカウント/パフォーマンス/地域/ 2012-01-01（1日）

[GET] /v2/analytics/reports/account/performance/regions/2012-01-01...2012-01-02（1日）

[GET] /v2/analytics/reports/account/performance/regions/2012-01-01...2012-01-08（毎週）

[GET] /v2/analytics/reports/account/performance/regions/2012-01-01...2012-02-01（毎月）

[GET] /v2/analytics/reports/account/performance/regions/2012-01-01...2012-04-01（四半期）

注意： 範囲内の最後の日付は結果に含まれません。
デバイスタイプ

デバイスタイプに関する情報を返します。

タイプ：文字列

有効値：モバイル| タブレット デスクトップ| セットトップ| unknown.devicetype

デフォルト：なし

例：/analytics/reports/account/performance/device_types/tablet/2011-08-01...2011-08-08
DMA

指定マーケティングエリア（DMA）に関する情報を返します。

デフォルト：上位10個のDMAを返します。

例：

[GET] /v2/analytics/reports/account/performance/dmas/2011-08-01...2011-08-08（すべてのDMA）

[GET] /v2/analytics/reports/account/performance/dmas/new+york/2011-08-01...2011-08-08（ニューヨークDMA）

[GET] / v2 / analytics / reports / asset /asset_id/performance/dmas/2011-08-01...2011-08-08（指定資産、すべてのDMA）

[GET] / v2 / analytics / reports / asset /asset_id/performance/dmas/new+york/2011-08-01...2011-08-08（指定資産、ニューヨークDMA）
ドメイン

ドメインに関する情報を返します。

タイプ：文字列

デフォルト：上位10ドメインを返します。

例：

[GET] /v2/analytics/reports/account/performance/domains/2011-08-01...2011-08-08（すべてのドメイン）

[GET] /v2/analytics/reports/account/performance/domains/ooyala.com/2011-08-01...2011-08-08（ooyala.com domain）

[GET] / v2 / analytics / reports / asset /asset_id/performance/domains/2011-08-01...2011-08-08（指定資産、全ドメイン）

[GET] / v2 / analytics / reports / asset /asset_id/performance/domains/ooyala.com/2011-08-01...2011-08-08（指定アセット、ooyala.comドメイン）
レベル

レポートをアカウントレベルと資産レベルのどちらで実行するかを指定します。 資産を指定するときは、資産IDを入力する必要があります。

タイプ：文字列

デフォルト：なし

有効な値：account | 資産/asset_id

例：

[GET] /v2/analytics/reports/account/performance/total/2011-08-01...2011-08-08

[GET] /v2/analytics/reports/asset/fjkl3s23fdsf2/performance/total/2011-08-01...2011-08-08

プラットフォーム

プラットフォームに関する情報（デバイスタイプ、オペレーティングシステム、ブラウザ）を返します。

タイプ：文字列

デフォルト：なし

例：

[GET] /v2/analytics/reports/account/performance/platforms/2011-08-01...2011-08-08（すべてのプラットフォーム）

[GET] /v2/analytics/reports/account/performance/platforms/desktop:mac-chrome/2011-08-01...2011-08-08（Chromeを実行しているMac）

[GET] / v2 / analytics / reports / asset /asset_id/performance/platforms/2011-08-01...2011-08-08（指定資産、すべてのプラットフォーム）

[GET] / v2 / analytics / reports / asset /asset_id/performance/platforms/tablet:android-unknown.browser/2011-08-01...2011-08-08（指定アセット、iOSタブレット）

[GET] /v2/analytics/reports/account/performance/device_types/desktop/platforms/2011-08-01...2011-08-08（すべてのデスクトッププラットフォーム）

地域

地域に関する情報（米国の州など）を返します。

タイプ：文字列

デフォルト：上位10地域を返します。

例：

[GET] /v2/analytics/reports/account/performance/regions/2011-08-01...2011-08-08（全リージョン）

[GET] /v2/analytics/reports/account/performance/countries/us/regions/2011-08-01...2011-08-08（米国内のすべての地域）

[GET] /v2/analytics/reports/account/performance/countries/us/regions/california/2011-08-01...2011-08-08（カリフォルニア州のみ）

[GET] / v2 / analytics / reports / asset /asset_id/performance/regions/2011-08-01...2011-08-08（指定アセット、すべての地域）

[GET] / v2 / analytics / reports / asset /asset_id/performance/countries/us/regions/2011-08-01...2011-08-08（指定資産、米国内のすべての地域）

[GET] / v2 / analytics / reports / asset /asset_id/performance/countries/us/regions/california/2011-08-01...2011-08-08（指定資産、カリフォルニア州のみ）
tag_id

タグのID

タイプ：文字列

デフォルト：なし

例：/ analytics / tags / extreme_couch_potato
URL

指定されたURLに関する情報を返します。

タイプ：文字列

デフォルト：上位10個のURLを返します。

例：/analytics/reports/account/performance/domains/ooyala.com/support

ビデオ

トップビデオを返します。

タイプ：文字列

デフォルト：10

例：

[GET] /v2/analytics/reports/account/performance/videos/2011-08-01...2011-08-08&limit=20

クエリ文字列パラメータ

次の表に、ルートのクエリ文字列パラメータを示します。

注意： わかりやすくするために、ここに示す例はURLエンコードされていません。 ただし、実際には、すべてのクエリ文字列パラメータと値はURLエンコードされている必要があります。
パラメーター 説明 必須？
breakdown_by

メトリックを単一の値として返すのではなく、日、週、または月ごとに結果を返します。 このパラメータは、ディメンションが 合計 そしてドリルダウンはありません。

タイプ：文字列

有効な値：day | 週| 月

デフォルト：day

例：breakdown_by =月

 いいえ
制限

返される結果の最大数。 指定した値よりも多くの結果が表示された場合は、ページングトークンを使用して次のページの結果を取得できます。

タイプ：整数

デフォルト：10

有効値：1〜500

例：limit = 20

 いいえ
order_by

ソートするフィールド（降順）。 演劇、ディスプレイ、または視聴時間順にソートされたクエリは、最大1000件まで返されます。

タイプ：文字列

デフォルト：plays

有効な値： ディスプレイ time_watched | video_starts | ad_requests | ad_impressions | ad_clicks

例：order_by = Displays

 いいえ
paging_token

or

page_token
結果の次のページへのURL。
注意： 特定のページの結果数がの値で割り切れる場合 制限サービスは404応答を返します。

タイプ：文字列

デフォルト：なし

例：[GET] /v2/analytics/reports/account/performance/videos/2012-01-01...2012-01-24?page_token=W1siY29sdXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXMXXXXXXXXXXXXXXXXき合合表1基本情報をすべて表示します。

 いいえ
