v3 Analytics Reporting APIの構文

The v3 Analytics Reporting API のGET or POST request must be signed by your Ooyala-issued API key and secret as detailed in v3 Analytics Reporting APIのセットアップ/メカニズム and must include the 有効期限が切れます keyword and timestamp.

In addition, the individual values of query string parameters and the entire value for filters = URLエンコードされている必要があります。

レポートGETの一般的な構文

ルートとクエリ文字列の基本的な構文は次のとおりです。 読みやすくするために、単一行の要求は複数の行に分割されています。 

[GET] / v3 / analytics / reports /？
            report_type=type
            ＆大きさ=大きさ
            ＆メトリクス=メトリクス
            ＆フィルター=filter_type == 'filter_value'
            ＆START_DATE=date
            ＆終了日=日付＆other_parms
            ＆api_key = your_api_key
  • 必須 クエリ文字列パラメータ（太字で表示）は report_type, START_DATE、そしてapi_key。
  • 次元が指定されていない場合は、すべての次元にわたる合計値が返されます。
  • 最大1月の時間範囲では、無制限のデータ行を使用して、一度に最大3ディメンションで照会できます。 1か月を超える時間範囲では、最大2行のデータを使用して、一度に最大10,000ディメンションで照会できます。
  • メトリックが指定されていない場合は、すべてのメトリックが返されます。
注意： APIキーはクエリパラメータとして、またはヘッダX-API-KEYを介して渡すことができます。
注意： 現時点では、 report_type is パフォーマンス.
注意： 1ディメンションのみを使用できます（URL）のための URL クエリ

GET Concurrent Users

に のGET 情報 同時ユーザー, send a のGET の v3 / analytics / reports endpoint, in which the report_type variable takes the value "concurrent":

[GET] /v3/analytics/reports?report_type=concurrent&metrics=<comma separated metrics>&start_date=<start>&end_date=<end>

Below are detailed descriptions of the parameters:

  • report_type: A required parameter. In this case, it should be set to 同時.
  • START_DATE: The start date is specified as YYYY-MM-DD or YYYY-MM-DDTHH：MM. If no HH：MM time is specified, it is set to 00：00 by default. The start date's time, if specified, would be in the provider's time zone. This parameter is required when report_type = 同時.
  • 終了日: The end date is specified as YYYY-MM-DD or YYYY-MM-DDTHH：MM. If no HH：MM time is specified, it is set to 23：59 by default. The end date's time, if specified, would be in the provider's time zone. The end date itself is excluded from the time range, so that the time range is actually [start_date, end_date). This parameter is required when report_type = 同時.
  • メトリクス: An optional parameter. A comma-separated list of the metrics to be returned in the response. Different report types have different available metrics. See the メトリクス セクションを参照してください。 * means all metrics. This parameter's default value is *.
