パフォーマンスメトリクスの表示

[パフォーマンス]タブでは、表示、再生要求、ビデオの開始数、コンバージョン率など、一般的なパフォーマンス指標が強調表示されます。

タブの左側にある凡例の項目は、チャート上の線に色分けされています。

チャートから指標を非表示にするには、凡例の指標の名前の左側にあるチェックボックスをオフにします。

[パフォーマンス]タブに表示される指標は次のとおりです。

  • ディスプレイ： これは表示イベントの発生回数です。つまり、埋め込みコードが変更された（新しいコンテンツが読み込まれた）ことを意味します。
  • 要求された演劇数： 「再生」ボタンが手動または自動でトリガーされた回数。 これはplayRequestedイベントの発生数です。つまり、任意のビデオコンテンツ（広告ビデオコンテンツを含む）が要求されたことを意味します。
    注意： このメトリックの計算が改善されました。 この実装により、5 / 11 / 11の後に、HTML2014プレーヤーおよびSDKプレーヤーから要求された再生数が増加することがあります。
  • ビデオ開始： ユーザーが実際のビデオコンテンツ（広告以外のコンテンツ）の視聴を開始した回数。これはplayStartedイベントの発生数です。つまり、要求されたビデオコンテンツ（広告コンテンツではない）が実際の再生を開始したことを意味します。
    注意： このメトリックの計算が改善されました。 この実装により、5 / 11 / 11の後、HTML2014およびSDKプレーヤーからのビデオ開始数が減少することがあります。
  • プレイコンバージョン率： 表示イベントに対する再生要求の比率。 [再生コンバージョン率]ビューは、サイトまたはアプリで消費者に表示されている動画間のクリック率（クリックスルー率）、およびクリックスルーと再生に対する消費者の意欲を理解しようとするパブリッシャにとって便利です。 自動再生プレイヤーはこの比率を妨げます。
    注意： 要求された戯曲の計算が改善されました。 この実装により、5 / 11 / 11の後に、HTML2014プレーヤーとSDKプレーヤーの再生コンバージョン率の指標が増加することがあります。
  • 動画変換率： 要求されたイベントを再生するビデオに対するビデオの比率。 動画のコンバージョン率ビューは、動画の開始前に何らかの形式の「バンパー」を提供したり、プレロール広告を実行して、要求された動画の開始前にプレーヤーから退出した消費者の放棄率を追跡するパブリッシャにとって便利です。
    注意： 動画の開始数と再生回数の計算が改善されました。 この実装により、5 / 11 / 11の後にHTML2014およびSDKプレーヤーからのビデオの開始数が変わることがあります。

の概念の概要を参照してください。 典型的なビデオの表示、再生、ビデオ開始（およびその他のイベント）のグラフィック表示。 グラフィックの最初の3つのイベントは、表示、再生要求、およびビデオの開始に相当します。

ビジネスインテリジェンスページのレポートをCSVファイルとしてエクスポートする方法の詳細については、 ビジネスインテリジェンスレポートのエクスポート.

タイムセグメント

チャートの右上には、表示されているデータの粒度（詳細レベル）を変更するために使用できる一連のボタンがあります。

以下の時間区分が利用可能です。

  • 15ミン
  • 時間
  • 日
  • 週間
  • 月

デフォルトの時間セグメントはDayです。

注意：

日時セグメントのデフォルトの日付範囲は、15分および時セグメントのデフォルトの日付範囲とは異なります。

日時セグメントの場合、デフォルトの日付範囲は、週の最初の日の12：00amから最後の日の11：59pm（8 24時間セグメント）です。 15の分と時の時間セグメントの場合、デフォルトの日付範囲は、週の最初の日の12：00pmから最後の日の12：00pm（7 24時間セグメント）です。

デフォルトの日付範囲の違いは、日の時間セグメントと15の分または時間のセグメントを切り替えると、メトリックデータが変わることを意味します。 今後のリリースでは、15の分と時間の時間セグメントの日付範囲の制限を日の時間範囲（8 24の時間セグメント）と一致するように変更します。
