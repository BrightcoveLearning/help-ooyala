ディスカバリーの使用

Q：ディスカバリーとは何ですか？

A: Ooyala Discoveryの推奨エンジンは、HadoopやMapReduceなどの業界標準のビッグデータおよび推奨技術、ならびにコラボレーティブフィルタリングを使用して、消費者向けの関連および傾向のあるコンテンツ推奨を生成します。 推奨事項は高度に設定可能です。 表示される推奨事項をカスタマイズするためにラベルとフィルタを使用できます。 消費者が選択されたアセットを見終わったとき、および/またはプレーヤーが一時停止したときに推奨を提供するようにプレーヤーを設定できます。

Q：Discoveryはどのように機能しますか？

A: ディスカバリーは視聴者に推奨事項を提供します。 サイト管理者、および配布リストに追加されたユーザーは、表示されている推奨事項と表示されている推奨資産を示すInsightレポートを毎週受け取ります。

Q：Discoveryはどのように有効化および管理されていますか？

A: 販売担当者に連絡して、サイトのディスカバリ機能を有効にしてください。

ディスカバリーが有効になっている場合、Backlotを使って特定のOoyalaプレーヤーのための推奨オプションを設定することができます。 パブリッシュ タブ）。 あなたのサイトが複数のOoyalaプレイヤーを使用している場合、Discoveryはそれぞれに対して別々に有効にされなければなりません。 検出プロファイル設定は、リンクされたプレーヤーに対して一意の名前が付けられています。

Ooyalaプレーヤーはディスカバリープロファイルを作成する必要がありますが、プレーヤーとBacklotインターフェースはあなたのサイトの推奨事項に電力を供給する必要はありません。 検出がBacklotで有効になると、検出設定はBacklot検出インターフェースと検出APIから管理できます。 OoyalaプレーヤーのDiscoveryインターフェースのみを使用してコンシューマーに推奨事項を配信する場合は、Backlotを使用してデプロイメントを完全に管理できます。 ただし、Discovery APIはサイトの推奨事項を管理するための強力なオプションであり、プレーヤーのディスプレイを設定するためのオプションもあります。 詳しくは、以下を参照してください。

  • Backlot UIを使用した検出プロファイルの作成
  • Ooyala Discovery APIリファレンス

Q：私のサイトは各ビデオアセットに対して異なるURLを使用しています。 ディスカバリーはこれをサポートできますか？

A: 絶対に！ 同じプレーヤ、同じページ、または新しいページで推奨動画を開くように検出を設定できます。 パーマリンクURLがそのアセットのメタデータで利用可能である限り、推奨動画は新しいページで開きます。

Discoveryはモバイルプラットフォームでも機能しますか？

A: はい。

モバイルプラットフォームでのディスカバリーに関する情報は、 Androidネイティブアプリの検出について そして iOSネイティブアプリの検出について.

提言

Q：ディスカバリーは編集者による推奨事項の管理を許可しますか？

A: 編集者はいくつかの方法でDiscoveryの推奨事項に影響を与えることができます。

  • ラベルを使用して、適用されたラベルに基づいて特定のコンテンツを含めるか除外します。
  • ディスカバリートレイ内のスポット（1st推奨ビデオ、2nd推奨ビデオなど）を選択して、特定のラベルを含む可能性のある最も新鮮なコンテンツ（アセット）で埋めます。 これにより、編集者は特定のコンテンツを的確に宣伝することができます。
  • 共有ラベルを持つアセットを選択して、Discoveryの推奨を上書きします。 これらのオプションの詳細については、 ラベル.

Q：ディスカバリーの推奨事項が私のサイトのエンゲージメントを高めていることをどのように確認できますか？

A: Discovery Insights Reportは毎週生成されます。 このレポートは、消費者エンゲージメントに対するディスカバリー推奨事項のディスカバリーの影響を判断するのに役立つエンゲージメントカテゴリに関するメトリックを提供します。 Insightレポートについて詳しくは、以下を参照してください。 ディスカバリーインサイトレポート（利用不可）.

インサイトレポートとは関係なくメトリクスをテストする場合は、A / Bテスト用に、ディスカバリを有効にしたものとディスカバリを含まないものを2人使用します。 A / Bテストは、Discovery APIを使用してサイトのOoyalaプレーヤーの外部に推奨事項を配信する場合にも役立ちます。

Q：ストリームとエンゲージメントの向上に関して、私は何を期待できますか？

A: Discoveryの推奨事項に基づくエンゲージメントに関する指標は、サイトのトラフィック量によって異なります。

Q：私は今日、独自の推奨エンジンを持っています。 どうやってDiscoveryを試して現在のエンジンの成長を測定できますか？

A: Discoveryの試用版が入手可能です。 推奨エンジンの試用版を使用すると、Insightsレポートから入手可能なメトリックを確認して、A / Bテストを実行できます。 ディスカバリトライアルの開始に関する詳細については、営業担当者にお問い合わせください。

Q：検出トレイから次のビデオの自動再生を無効にするにはどうすればよいですか？

A: 自動再生を完全に無効にすることはできません。 ただし、Discovery APIを使用して default_countdown 高い数値へのパラメータ。 この数値は、次のビデオが自動再生されるまでの秒数を指定します。 デフォルトは10です。 Backlot Discoveryインターフェースは、この設定を99秒以下に制限します。 APIを使用してこの制限を99秒よりも大きく設定する方法の詳細については、 default_countdown パラメータ、を見てください 検出APIを使用した検出プロファイルの管理.

Q：Discoveryによるエンゲージメントを追跡するために利用できるデータは何ですか？

A: Discovery Insights Reportは毎週生成されます。 このレポートは、本源的消費に関する、消費者エンゲージメントに対するディスカバリーの影響を判断するのに役立つエンゲージメントカテゴリに関するメトリックを提供します。 Insightレポートについて詳しくは、以下を参照してください。 ディスカバリーインサイトレポート（利用不可）.

Q：前のバッチと同じ資産が推奨されるのはなぜですか？

A: ディスカバリーは、ステートレスな推奨エンジンです。 セッション固有の消費者行動データは保存されません。 任意の所与のセッションにおいて、消費者の過去の使用データが活用され、それは同じビデオ資産がその後の推奨リストにわたって再び推奨されることをもたらし得る。 OoyalaプレーヤーのDiscovery UXは、以前の一連の推奨事項に含まれていた可能性のあるすべてのビデオを除外します。 Ooyalaは、Discovery APIを使用する際に、発行者サイトが同様のフロントエンド/クライアントサイドフィルタを使用することを推奨します。

アナリティクス

Q：発見フィードバックループとは何ですか？

A: 発見の推奨は、消費者の行動を追跡する機械学習アルゴリズムに基づいています。 フィードバックループはこのインフラストラクチャの重要な部分であり、推奨コンテンツの表示回数とクリック数を登録し、データを処理して各消費者の好みをリアルタイムで調整します。 詳細については 発見フィードバックループ.

Q：Discoveryは、各消費者に対してフィードバックループAPIを介して渡されるさまざまなインプレッションとクリックのデータをどのように関連付けますか？

A: デバイスID 消費者インプレッションとクリックデータを記録するために必要な識別子です。 データポイントの統合は、デバイスIDに基づいています。 さらに、Ooyalaプレーヤー バケツ情報 フィードバックループを介して送信することもできます。これにより、Discoveryはインプレッションまたはクリックイベントに関する追加データを収集できます。 詳細については 発見フィードバックループ.

Q：Discoveryが使用するアルゴリズムに影響を与えることはできますか？

A: ディスカバリーは、消費者の行動に基づいて継続的に学習する機械学習アルゴリズムを使用します。 コールドスタートシナリオでは、利用者の利用状況データが利用できない場合は、現在の資産に関連するコンテンツをお勧めします。 利用可能な現在の資産がない場合は、傾向データが推奨されます。

Discoveryには、資産の推奨事項に影響を与えるのに役立ついくつかのオプションがあります。

  • ラベルを使用して、ラベルに基づいて特定のコンテンツを含めるか除外します。 詳細については ラベル.
  • 資産プロファイルを構成するには、Backlot Discoveryインターフェースを使用します。 詳細については Backlot UIを使用した検出プロファイルの作成.
  • Discovery APIを使用して、コンテンツをフィルタリングするための編集パラメータを設定します。 詳細については クエリ文字列パラメータ.

API

Q：Discoveryはクライアントサイド（JavaScript）APIを提供していますか？

A: Discovery APIはRESTfulです。 ユースケース、実装、および環境に応じて、 XMLHttpRequestの Discovery APIを利用するためのオブジェクト。 詳細については Ooyala Discovery APIリファレンス JavascriptまたはAjaxの実装に固有の参照情報。

Q：編集用のクエリ文字列パラメータとDiscovery Profilesは同じことをしているようです。 違いはありますか？

A: ディスカバリーAPI要求は、機械学習アルゴリズムから基本セットの推奨事項を返します。 このベースセットは、ディスカバリプロファイルを使用して変更できます（ ディスカバリープロファイルまたは、編集用のクエリ文字列パラメータを利用して（ ディスカバリー編集用照会ストリングのパラメーター） 両方のオプションを使用して推奨事項の基本セットをカスタマイズできますが、Ooyalaでは、編集用のクエリ文字列パラメータではなく、Discoveryプロファイルを使用することをお勧めします。
