Syntax for Embedded Parameters

  • Enclose parameter names in double quotes ("). Example: "autoplay"
  • Enclose string values in double quotes. Example: "pcode": "YOUR_PCODE"
  • Omit quotes for Boolean values and numbers. Example: "autoplay": false
  • For non-string parameter values, refer to the parameter example. Example: "initialTime": 10

必須パラメータ

The following are required embedded parameters, represented as key value pairs, that you must use when creating a V4 player.

The pcode is your account identifier. This is an alphanumeric string that precedes the period in your API key. You can get your pcode from your API keys. If you do not include your pcode, the player will not load.

自動再生

Enables the automatic playing of an asset (video or audio) upon loading. This is useful for UIs that do not have play/pause controls or conditions where you want the content to play immediately.

  • 'autoplay': false (default) - disables automatic playback of the main video.
  • 'autoplay': true - enables automatic playback for the main video.
重要： Autoplay on mobile browsers requires that the audio be muted. Therefore, if you set the player to autoplay on mobile browsers (Chrome and Safari Mobile), the video will play in a muted state until the user interacts with the video. Additionally, for Safari Mobile, autoplay requires setting an additional parameter (iosPlayMode 〜へ 'inline') to set playback to inline mode (the default is full-screen).
注意： Facebookでは、Facebookフィード内のサードパーティのプレーヤーによる動画の自動再生は許可されていません。 そのため、プレーヤーがFacebookに埋め込まれている場合、自動再生設定は無視されます。

initialBitrate

Use this parameter to set the initial minimum bitrate level (immediately after video playback) and to sustain that level for a specific period of time. Once the duration is reached, the bitrate level changes to the video plugin's automatic bitrate level.

設定:

You can specify two child parameters:
名前 タイプ 有効な値 説明
レベル 文字列 次のいずれか
  • "auto" （デフォルト）
  • number between 0 (zero) and 1, inclusive
  • 指定 "auto" to defer to the default bitrate set by the video plug-in's ABR logic. The behavior is equivalent to omitting initialBitrate entirely.
  • Specify a number if you want to set a particular level between zero and 1, inclusive.
If you specify a number:
  • 指定 0 (zero) to select the lowest bitrate first for the associated video plugin
  • 指定 1 to select the highest bitrate (MAX_BITRATE) for the associated video plug-in.
  • Specify a number between 0 そして 1 to indicate which stream to use. From the array of available bitrates, the player selects the maximum value that is 以下 the preferred bitrate factor. For example, if your available bitrates were 0.1, 0.75, and 1.0mbps, then a value of 0.5 would select the lowest bitrate (0.1), a value of 0.8 would select the middle bitrate (0.75), and so on.
To determine the level, consider such factors as the approximate network speed of the end user, the type of device (desktop or mobile), and so on.
デュレーション 数 a number greater than zero Represents the number of seconds to sustain the initial bitrate level.
注意：

The following APIs also override the initialBitRate and automatic bitrate, starting from when they are called and for the remainder of the playback session:

  • SET_TARGET_BITRATE
  • setTargetBitrate（）

      "pcode": "YOUR_PCODE",
      "playerBrandingId": "YOUR_PLAYER_ID",
      "skin": {
         // Config contains the configuration setting for player skin. Change to your local config when necessary.
          },
         // Set the initial bitrate to 80% of the maximum bitrate for 30 seconds
         "initialBitrate": {"level": 0.8, "duration": 30}
      };
      OO.ready(function() {
         window.pp = OO.Player.create("container", "YOUR_ASSET_ID", playerParam);
         });

場所

Use this parameter (within a "bit-wrapper" object) to specify individual, custom paths for Bitmovin HTML5, CSS, and Flash files if any of those resources are stored in a location 以外の どこ bit_wrapper.min.js plugin is stored.
注意：

Use this parameter only if you are using the bit_wrapper.min.js plugin and のみ if you do any of the following:

  • You self-host player resources そして you want to serve one or more of the associated resource files from a location that differs from where the bit_wrapper.min.js plugin is served.
  • You self-host player resources and you concatenate all of the Player V4 Javascript files into a single bundle.
  • You load the Player V4 files using a module loader library (most notably SystemJS) that bundles Javascript files. For SystemJS, this is true regardless of where the bitmovin resource files are located (custom or default).

Otherwise, this parameter is not needed.

注意： This parameter is identical to Bitmovin's ロケーション parameter, which is described in Bitmovin's documentation. Player V4 currently uses only the html5, CSS、および フラッシュ options. Other options (vr, ctrlsに電子メールを送るか、もしくは ctrls_css) are currently not used.

プリロード

Use this parameter to specify whether to start loading the video after the player is loaded and before the user starts playback (for non-autoplay settings) (true), or to wait until the user starts playback (false, the default). One of the following values:
  • 'preload': false （デフォルト）
  • 'preload': true - start preloading the video as soon as the player is loaded. Typically, a video will preload the stream up to filling the buffer, the size of which will vary by browser.
Preloading can significantly speed up the time-to-first-frame experience because the video has the opportunity to buffer before the viewer plays it.

検討事項：

  • Preloading is disabled if the initialTime パラメータが使用されます。
  • When using Google IMA with prerolls, preloading will be triggered at the start of the fourth quartile of the preroll ad or the last ad in a podded preroll. If ads are skipped before the fourth quartile of the ad is played, preloading of content will not occur.
  • Preloading is supported for the main_html5 and bit_wrapper video plugins.
  • Preloading is not yet supported for the Pulse or Freewheel ad managers.
  • Enabling content preloading can increase your overall video stream consumption because, for any given player embed, videos are preloaded even when playback is not initiated.

タイプ: Boolean

例:

<div id="container" style="width:640px; height:360px;"></div>
<script>
    var playerParam = {
        "pcode": "YOUR_PCODE",
        "playerBrandingId": "YOUR_PLAYER_ID",
        "preload": true
    };
    OO.ready(function() {
        window.pp = OO.Player.create("container", "YOUR_ASSET_ID", playerParam);
    });
</script>

preloadAds

  • 'preloadAds': false （デフォルト）
  • 'preloadAds': true - starts the preloading of ads.
注意： Please note that Ad Preloading is currently only supported with Google Ads Manager plugin (formerly Google IMA) when used with Prerolls and Ad Rules (with any ad position).

skin.config

The skin parameter references the skin.json config file. This loads the player skin. Use it if you want to use your own skin settings.

例:

<div id="container" style="width:640px; height:360px;"></div>
        <script>
            var playerParam = {
              "pcode": "YOUR_PCODE",
              "playerBrandingId": "YOUR_PLAYER_ID",
              
              "skin": {
                // Config contains the configuration setting for player skin. Change to your local config when necessary.
             }
            };
            OO.ready(function() {
              window.pp = OO.Player.create("container", "YOUR_ASSET_ID", playerParam);
            });
        </script>

skin.inline

Use this parameter to specify inline skin modifications. skin.inline will overwrite any settings in the skin.json config file. The JSON object within inline must have the same structure as skin.json (all parent objects going all the way back to the root object, as described in the プレーヤV4 JSONスキーマ). For example, if you want to overwrite the start screen play button color using inline, you need to include the start screen object, playIconStyle object, and color (as shown in the following example).

例:

<div id="container" style="width:640px; height:360px;"></div>
        <script>
            var playerParam = {
              "pcode": "YOUR_PCODE",
              "playerBrandingId": "YOUR_PLAYER_ID",
              "skin": {
                  "config": "url_where_hosted/skin.json",
                  "inline": {
                      // Put your player customizations here
                      // to override settings in skin.json. 
                      // The JSON object must match the structure of skin.json.
                      "startScreen": {"showDescription": false, "playIconStyle": {"color": "blue"}}
                  }
              }
            };
            OO.ready(function() {
              window.pp = OO.Player.create("container", "YOUR_ASSET_ID", playerParam);
            });
        </script>
注意：

You only need to declare the location of the skin.json ("config": "url_where_hosted/skin.json") for the following reasons:

  • You do not want to use the Ooyala-hosted/managed default skin.json as the reference skin.json ファイル。
  • You do not want any Player configurations set in Backlot (管 理 tab) used. Note that a skin.json file in the page has priority over any settings set in the 管 理 tab in Backlot. For more information about Ooyala Web Player Skin priorities, please see Order of Precedence in Player V4 Settings.

あなたが見るとき url_where_hosted in sample code, replace this (in your code) with the URL that points to where the resource is hosted. For a list of Ooyala-hosted paths, see OoyalaがホストするプレイヤーV4リソース. The URL can point to a location on the same host (internal link) or on a separate host (prefixed with http:// or https://). If you host resources yourself (see ホスティングプレイヤーV4リソース), be sure to check for any path dependencies within the files.

追加情報については、 skin.jsonを使用したPlayer V4スキンのカスタマイズ そして Embed a Simple Player with Inline Player Skin Modifications.

useFirstVideoFromPlaylist

Use this parameter to specify whether the player uses the first video from the playlist playlist (true) or not (false, the default). One of the following values:
  • 'useFirstVideoFromPlaylist': false （デフォルト）
  • 'useFirstVideoFromPlaylist': true - let the player set the first asset specified on the first item in a playlist.
注意： Before you set 'useFirstVideoFromPlaylist': true, contact Ooyala Tech Support to have your player plugin registered. If the plugin is not registered, playback will fail.
