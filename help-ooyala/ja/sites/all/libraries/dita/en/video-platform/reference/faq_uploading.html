アップロードに関してよく寄せられる質問

アスペラ

Q：ビデオをアップロードしましたが、Backlotに表示されません。 私は何をすべきか？

A: 動画と一緒に関連付けられたマニフェストファイルをアップロードしたことを確認します（ マニフェストファイル形式 詳細については）。 アップロード後、BacklotのMANAGEページに表示されるファイルに遅延がないはずです。 ファイルが表示されない場合は、問題が発生している可能性があります。
  • （OIS v1のみ）：FTPまたはAsperaを使用している場合は、ログでエラーを確認してください。 見る 取り込みログとエラーメッセージ 詳細については。
  • もう一度ファイルをアップロードしてみてください。
  • メタデータファイルの有効性もチェックして、正しくタグ付けされていることを確認する必要があります。
問題が解決しない場合は、テクニカルサポートに連絡してください。

Q：Backlotに入らない場合、私のビデオを保存しますか？

A: いいえ、問題が原因でファイルがBacklotデータベースに保存されない場合、そのファイルは削除されている可能性があります。 ファイルをもう一度アップロードしようとしました。

UTF-8文字

Q：FTP + CSVインジェストしたコンテンツの[詳細]タブに "Internal Server Error"と表示されるのはなぜですか？

A:BacklotはUTF-8文字のみを受け入れます。 FTP + CSV経由で取り込む場合は、UTF-8文字と似ていますが、有効なUTF-8ではない非UTF-8文字を含めることができます。 たとえば、引用符のように見える文字がたくさんあります。 以下が最も頻繁に使用されます。
  • "U + 0022見積もりマーク
  • “ U + 201Cダブルダブルクォーテーションマーク
  • ” U + 201D右側ダブルクォーテーションマーク
U + 0022は有効なUTF-8です。 他の2つのいずれかを指定した場合、Backlotはエラーを返します。 したがって、すべてのCSV文字が有効なUTF-8であることを忘れないでください。

ログとレコードのアップロード

Q：今月中にアップロードしたすべてのビデオはどこで見ることができますか？

A: デフォルトでは、BacklotのMANAGEページでは、ファイルは最新の作成日順に表示されます。 したがって、特定の月にアップロードされたファイルは常にまとめてグループ化されます。

Q：Backlotで、アップロードが完了していないビデオだけを見ることができますか？

A: はい、できます。 のトピックを参照してください。 ビデオの詳細を管理する。 ステータスをに設定します 加工、処理 ビデオがまだトランスコードされているのを見るため。

Q：私のFTP活動の記録はありますか？

A: あなた自身のFTPクライアントソフトウェアはあなたのすべてのFTPセッションの記録をログファイルあるいはあなた自身のコンピュータの他の場所に保存するかもしれません。 この情報がシステム上のどこに格納されているかをよく理解して、問題の可能性についてこれらのログを調べてください。 長期間アップロードする場合は、将来発生する可能性のある問題をデバッグする必要がある場合に備えて、将来の参照用にこれらのログを保存したり上書きしたりしないようにすることを検討してください。

境界

Q：一度に何GBまでアップロードできますか？

A: 顧客ごとの100GBのクォータが適用されます。 FTPまたはAsperaを使用してコンテンツをアップロードする場合、常にアップロード処理に含まれるべきデータの最大量は100 GBです。 この割り当て量を超えないようにするには、一度にアップロードするデータ量を追跡し、ファイルが完全に取り込まれて処理されるまで待ってから追加のアップロード操作を開始します。 ファイルのアップロードが完了すると、そのファイルはクォータに対してカウントされなくなります。

Q：APIアップロード制限は何ですか？

A: 無し。 割り当てはありません（FTPおよびAsperaの場合のように）。 ただし、Ooyalaでは、ファイルを細かく分割することを強くお勧めします（詳細は、 ビデオまたはオーディオアセットのアップロード）大規模ファイルをネットワーク経由で転送する際に発生する可能性がある問題を防ぐためです。

符号化および処理プロファイル

Q：エンコーディングとは何ですか？

A: ビデオストリームをさまざまな再生デバイス用に最適化されたいくつかのフォーマットに変換するプロセス。 最適化されたフォーマットを作成するために使用される値のセットはビデオのエンコードです。

Q：加工プロファイルとは何ですか？

A: 処理プロファイルは、動画をエンコードするときにテンプレートとして使用できるエンコードのセットです。 詳細については、 プレーヤV4のための推奨エンコーディング

コンテンツ移行

Q：このサービスは誰のためのものですか？

A: あなたが大規模なビデオライブラリを持っていて、あなたのコンテンツをOoyalaに移動するのに援助を必要とするならば、あなたはOoyalaのコンテンツ移行サービスから最も利益を得るでしょう。 自己移行（つまり、Backlot Webアップロード、FTP、Aspera、API経由）が選択肢にならない場合は、プロフェッショナルサービスによる移行が理にかなっています。

マネージドサービスの最も一般的な利点には次のものがあります。
  • プロセス全体を通じてOoyalaの指導。
  • 起動までの時間を短縮
  • ネットワーク飽和問題を減らしました。

Q：Ooyala Standard Packagesには何が含まれていますか？

A: Ooyala Standardパッケージに含まれるもの：
  • 既存のカタログとカテゴリに一貫してマッピングされる新しいビデオメタデータ分類法の定義をサポートします。
  • それぞれのメタデータを割り当てた、ビデオアセットの移動。
  • Ooyalaのサポートするビデオおよびオーディオコーデックを使用してエンコードされたすべてのビデオアセットのトランスコードが成功した。
  • 移行のニーズを満たすトランスコーディング分のバケット。
  • 定期的な進捗状況の更新。その頻度は、移行の期間に基づいて設定されます。
  • Ooyalaの埋め込みコードを含む出力CSVファイル。 このファイルは、あなたのビデオとそれらの新しい識別子（埋め込みコード）との間の一対一の対応付けによる完成レポートを表します。

Q：あなたから何が期待されていますか？

A: あなたはOoyalaを次のように援助する必要があります：
  • ビデオライブラリへのアクセスを提供します。
  • メタデータマッピングファイルをCSVまたはXML形式で配信します。 テンプレートはOoyalaによって提供されます。
  • 摂取が始まる前にあなた自身の処理プロファイルを承認してください。 処理プロファイルは、各出力エンコードの特性を決定するためにトランスコードプロセスによって使用されるテンプレートです。

新しいアセットを2回アップロードする必要がある時間枠があることに注意してください。1回はレガシーシステムへ、もう1回はOoyalaへです。

コンテンツを複数のBacklotアカウントに移行する場合は、提案されたアカウントの階層を承認する必要があります。

Q：コンテンツ移行に加えて購入できる追加のサービスは何ですか？

次のサービスは、コンテンツ移行パッケージサービスに含まれていません。 それぞれの自己管理方法についての説明が提供され、追加のプロフェッショナルサービス契約を購入することができます。
  • 古いプレイヤーを新しいプレイヤーに交換します。
  • クローズドキャプションおよび/または字幕の割り当て。
  • プレイリスト、チャンネル、チャンネルセットの移行

Q：Ooyalaでサポートされているビデオおよびオーディオコーデックはどこにありますか？

A: この情報はOoyalaのサポートページにあります。

素材の品質と推奨事項（ビデオとオーディオ）.

Q：顧客はどのようにしてOoyalaに自分のビデオライブラリへのアクセスを許可することができますか？

A: ライブラリへのアクセスは、MRSSフィードを使用した直接HTTPダウンロード、またはハードドライブ配信の2つの方法で許可できます。 HTTPSはサポートされていません。

Q：移行にはどれくらいかかりますか？

A: コンテンツの移行は5フェーズのプロセスであるため、完了時間はすべてのフェーズの期間の合計です。 フェーズの期間は、お客様の条件や要件（図書館の規模、専任の担当者、納期など）に大きく依存します。

Q：移行の5つのフェーズは何ですか？

A: 移行を構成する5フェーズは次のとおりです。
  1. 発見。
  2. 勧告。
  3. 準備とスケジューリング
  4. 実行。
  5. 受諾。
各フェーズのマイルストーンは次のとおりです。
相 マイルストーン
1. Discovery 移行のニーズが評価され、文書化されています。
2. 推奨 移住計画はすべての当事者によって完成され承認されます。
3. 準備とスケジュール
  • あなたはメタデータファイルを配信し、Ooyalaにそれらのライブラリへのアクセスを許可します。
  • 摂取開始日が設定されます。
4. 実行
  • 移行が始まります。
  • 定期的な移行レポートが配信されます。
  • 移行が完了しました。
5. 受け入れ 受け入れ文書は顧客によって署名されています。

Q：Ooyalaのコンテンツ移行サービスを続けたいのですが、次に何をすればよいですか？

A: ディスカバリー段階からすぐに始めましょう。 アカウントマネージャに連絡してください。
