Webキャッシュを設定する

あなたの顧客のために最高のパフォーマンスを得るために、からの応答のためにキャッシングシステムを設定してください。 バックスロットAPI。 キャッシングのための非常に人気があり無料のプログラムの1つはmemcacheです。これはLinux上で無料で提供され、Microsoft WindowsまたはAppleシステムで利用可能です。

たとえば、真夜中（またはフロントエンドシステムの稼働率が最も低いとき）に、Linuxを使用するcronジョブを設定できます。 バックスロットAPI's / v2 / assets からすべての動画（アセット）を取得するためのルート バックスロット。 memcacheを実行すると、将来の要求に備えてこれらのビデオへの参照が保存されます。 ユーザーから資産の要求を受け取ると、プログラムは要求された資産が既にキャッシュされているかどうかを確認します。 もしそうなら、ビデオが表示されます。 そうでなければ、への呼び出し バックスロットAPI それを検索し、将来の参照のためにそれを保存してそれを表示します。 PHPでmemcacheを使ったコーディングの例は、 http://www.search-this.com/2007/07/24/an-introduction-to-memcached/

この問題についてのインターネット上の豊富な情報もあります。 http://www.mnot.net/cache_docs/.

モバイルWebのプログラミングについて学ぶ

いくつかの推奨事項1:
  1. 基本的なHTMLとCSSを使用してください。
  2. 高速ダウンロードのために画像のサイズを小さくしてください。
  3. ブラウザ検出でリダイレクトを使用します。
  4. ユーザビリティの問題：あなたのページを過負荷にしないでください。 必要な基本を使いやすいサイズで提供してください。
    • ナビゲーションリンクが多すぎるのを避けます。
    • ページ上のコンテンツの量を最小限にします。
    • プレゼンテーションアイテムが邪魔になることがあります。
    • 必要に応じて、フルサイズバージョンへのリンクを提供してください。
