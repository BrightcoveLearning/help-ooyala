2016-02-10リリース

このリリースの注目すべき内容には、更新された新しいOoyala IQ機能が含まれています。

ソフトウェア機能リリース

サードパーティのプレイヤーでプレイされたバックスロットアセットでサポートされているOoyala IQメトリクス

Ooyala IQは以下の指標を提供できます。
  • Ooyalaアセット（Backlotでホストされている）をプレイしているOoyalaプレーヤー。
  • （新）Ooyalaアセットをプレイしているサードパーティ製プレイヤー（Backlotで開催）.
  • サードパーティの資産（サードパーティのCMSでホストされている）をプレイしているサードパーティのプレイヤー。
サードパーティのプレーヤーを使用してBacklotアセットを再生するのか、サードパーティのCMSホストアセットを再生するのかを設定できます。 次の例は、Ooyala-JW Playerアダプタを使用してコンテンツタイプを指定する方法を示しています。
  • OOYALA_CONTENT： MEDIAID はOoyalaの埋め込みコードです。 Ooyalaアセットを指定するために必要な情報を入手するには、Backlot APIクエリを作成します。 / assets / <資産コード> / streams アセットの場合は、目的のエンコーディングの結果をコピーします。 以下の例は、 OOYALA_CONTENT JW Playerアダプタで指定します。
    <script type = "text / javascript"> var jwPlayer = jwplayer（ "video"）。setup（{ファイル： 'pathToYourOoyalaMp4VideoAsset'、メディアID： 'yourOoyalaEmbedCode'、タイプ： 'mp4'}）; var jwOoyalaReporter = Ooyala.Analytics.JWReporter（ "<pcode>"、jwPlayer、{contentType：Ooyala.Analytics.MediaContentType.OOYALA_CONTENT}）; </script>
  • EXTERNAL_CONTENT： MEDIAID これは、Ooyalaの埋め込みコードではありません（そして、Ooyala以外のCMSからの顧客定義のメディアIDです）。 以下の例は、 EXTERNAL_CONTENT JW Playerアダプタで指定します。
    <script type = "text / javascript"> var jwPlayer = jwplayer（ "video"）。setup（{ファイル： 'pathToYourVideoAsset'、メディアID： 'yourMediaID'}）; ;））; var jwOoyalaReporter = Ooyala.Analytics.JWReporter（ "<pcode>"、jwPlayer、{contentType：Ooyala.Analytics.MediaContentType.EXTERNAL_CONTENT}）; </script>
（Backlotでホストされている）Ooyalaアセットでは、JW Playerなどのサードパーティのビデオプレーヤーで再生されたビデオアセットのOoyala IQ分析メトリックを表示できるようになりました。 サードパーティのビデオプレーヤーで再生されたビデオアセットのIQ分析メトリックスを有効にするための主な手順は、次のとおりです。
  1. Ooyala IQ JavaScript SDKを他社のプレーヤーと統合することで、サードパーティのプレーヤーがプレーヤーイベントをOoyala IQに伝達できるようにします。Ooyala Ooyala独自のアダプタを使用するか、独自のアダプタを構築することで、ビデオプレーヤーを構築できます。
  2. ビデオアセットが.mp4ファイルであること、および.mp4ファイルをエンコードできるように正しいエンコードプロファイルがあることを確認して、Ooyalaアセットがサードパーティのプレーヤーで再生可能であることを確認してください。
  3. 必ずあなたの.mp4ビデオアセット、アセットの埋め込みコードへのURLを参照し、アセットタイプをmp4として指定して、あなたのサードパーティプレーヤーを作成してください。
見る サードパーティのプレイヤーとOoyala Assetsを使ってみる 始める方法の詳細については。

Ooyala IQでの混合プレーヤーのサポート

Ooyalaプレーヤーとサードパーティプレーヤーを使用してOoyalaビデオアセット（Backlotでホストされている）を再生している場合は、1つのBacklotアカウントを使用してビデオをOoyalaプレーヤーとサードパーティプレーヤーにストリーミングできます。 両方のタイプのプレーヤーについて、1つのダッシュボードに（1つのユーザーアカウントを介して）統合されたIQ分析指標を表示できます。

サードパーティのアセット（サードパーティのCMSでホストされている）の場合は、アセットメタデータを取り込むために別のBacklotプロバイダーアカウントを設定する必要があります。

Ooyala IQでの無限ラベルスクロール

Ooyala IQのビジネスインテリジェンスページは、「ラベルによるフィルタ」機能を使用して、無限ラベルスクロールをサポートするようになりました。 多数のラベルがある場合のBusiness Intelligenceページの読み込み応答時間も向上しました。

Ooyala IQの新しいプリセット時間範囲

Ooyala IQは、以下の新しいプリセット時間範囲選択をサポートします。
  • 今年
  • 昨年
  • 最終12月間
注意： 2016の場合、今年はプロバイダのタイムゾーンで2016-01-01から今日までを意味し、昨年は2015-01-01から2015-12までを意味します。プロバイダのタイムゾーン 例えば、「今日」として31-12-12を使用して、今年は2015-02-10から2016-01-01を意味し、昨年は2016-02-10から2015-01-01を意味し、最後の2015は月を意味する-12-31から12-2015-02まで。

Ooyala Analytics Chrome拡張機能

Ooyala Analytics Chrome拡張機能は、現在Chromeウェブストアで入手できます。 https://chrome.google.com/webstore/detail/ooyala-analytics-plugin/kgkibeopmmkehaijmmfhldeclmgbmgnp。 このデバッグツールを使用すると、Ooyala IQで動作するように独自のサードパーティ製プレーヤアダプタを構築するのに役立ちます。

Chromecastデバイスに対するOoyala IQのサポート

Ooyala IQはChromecastデバイスの再生を追跡するようになりました。 browser = chrome、deviceOs = chromecast、およびdeviceType = settopのデータを調べることで、Ooyala IQ APIを使用してChromecast受信者分析を監視できます。 Ooyala IQのUIでは、これは[Device Type]> [settop]> [other - Chromecast]の下に表示されます。 [プレーヤー]パネルに、Chromecastプレーヤーの場合は "Chromecast_player"が表示されます。 分析結果は、エンドユーザーが開始したキャスト/プレイシナリオによって異なります。

この機能は1月にリリースされました。 見る Chromecast統合アナリティクス 詳細については。
