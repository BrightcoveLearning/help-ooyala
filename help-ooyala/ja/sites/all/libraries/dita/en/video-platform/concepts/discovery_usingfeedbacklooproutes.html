フィードバックループルートの使用

フィードバックループルートの使用方法を学びます。

Discovery APIリクエストを使用してビデオが表示され、ユーザーに推奨されている各インスタンスを追跡するには、 / v2 /発見/フィードバック/印象 次の例に示すように、route。
[POST] / v2 / discover / feedback /インプレッション{"device_id"： "device_id"、 "bucket_info"： "bucket_info"、 "ui_tag"： "tag1"}

ユーザーがDiscoveryの推奨動画を視聴し始めた各インスタンスを追跡するには、 / v2 /発見/フィードバック/再生 次のコード例に示すように、ルート。

注意： インプレッションは、バッチごとに1回だけではなく、ユーザーに表示されるビデオごとに記録する必要があります。
[POST] / v2 / discover / feedback / play {"device_id"： "device_id"、 "bucket_info"： "bucket_info"、 "ui_tag"： "tag1"}
次の表に、フィードバックループルートで使用するパラメータを示します。
パラメーター 説明
デバイスID Ooyalaが生成したデバイスID。これはユーザー、ブラウザ、またはデバイスに固有です。
bucket_info おすすめのバッチの生成方法、動画の位置、その他の便利なユーザーセンターの測定基準に関する情報を含むエンコードされた文字列。
ui_tag 推奨事項がデバイス画面のどこに表示されるかを説明するオプションのパラメータ。 値は最大80文字の英数字ストリングにすることができ、ストリングを使用することはできません デフォルト。 このパラメータの使用例は、推奨事項を含む要素のID値を追加することです。

Discoveryからのコンテンツに対するすべてのリクエストに対するJSONレスポンスには、で推奨されているblob（おすすめのリストに含まれる各ビデオに対する）が含まれます。 bucket_info。 たとえば、電話をかけたとき / v2 /ディスカバー/トレンド、応答の bucket_info blobは次の例のようになります（改行が追加されています）。
