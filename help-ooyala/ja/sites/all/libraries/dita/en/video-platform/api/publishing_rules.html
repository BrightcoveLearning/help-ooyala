公開ルール

公開ルールは、いつどこでコンテンツを再生できるかを指定します。 公開ルールを使用すると、コンテンツを特定の地域に限定し、特定のドメインにのみ埋め込まれるようにして、特定の時間にのみ表示できるようにすることができます。

注意： Backlot REST APIコマンドについて詳しくは、以下を参照してください。 バックスロットAPIリファレンス.

アセットがロードされると、再生前に公開ルールがチェックされます。 作成した各ルールには、これらの制限を任意に組み合わせて、複数のビデオまたはチャンネルに割り当てることができます。

注意： Backlot UIでは、公開ルールはシンジケーショングループと呼ばれます。

新しい公開ルールを追加

[POST] / v2 / publishing_rules {
   プロパティ

}

リスト公開ルール

[GET] / v2 / publishing_rules

公開ルールを取得

[GET] / v2 / publishing_rules /publishing_rule_id

公開ルールを変更する

[パッチ] / v2 / publishing_rules /publishing_rule_id{
   
   プロパティ

}

公開ルールを削除

[削除] / v2 / publishing_rules /publishing_rule_id

anonymizer_statuses

ブロックする匿名化機能の状況を指定します。

タイプ：文字列の配列

有効値：プライベート| アクティブ| 容疑者 非アクティブ corporate_vpn

OoyalaはNeustarのIP Intelligenceサービスを使用して匿名化プロキシを識別します。 Neustarは、プロキシとして検出されたIPアドレスに次のステータスを割り当てます。
  • プライベート：この名称のIPアドレスには、公にアクセスできない匿名プロキシが含まれているとされています。 そのため、自動化ツールで日常的にテストすることはできません。 これらのアドレスは通常、匿名サービスを一般に販売する商業的ベンチャーに属します。 この指定のアドレスは、所有権情報から派生するか、信頼できる信頼性の高いソースから取得されます。
  • アクティブ：アノニマイザーは過去6ヶ月以内に陽性反応を示した。
  • 容疑者：アノニマイザーは過去2年以内にポジティブテストを行いましたが、最後の6ヶ月はテストしませんでした。
  • 非アクティブな：アノニマイザーは過去2年以内に肯定的なテスト結果を出していません。
  • corporate_vpn：ユーザーは単一の国内の複数の州からのトラフィックをルーティングするプロキシを介して接続しています。 ユーザーがIPロケーションとは異なる州にいる可能性があります。 多くの場合、これらは地域の拠点または国際的な拠点からのトラフィックを中心地点、多くの場合は本社経由でルーティングする企業ネットワークです。

条件：proxy_restrictionsが指定されている場合は必須。

例：["private"、 "suspect"]

親：proxy_restrictions
