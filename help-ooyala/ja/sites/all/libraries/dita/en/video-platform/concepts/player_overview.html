プレイヤーV3（廃止予定）

重要： Ooyala Player V3は非推奨であり、予定されている予定です。 無効 2018-01-31について その日以降、Player V3はあなたのビデオまたはオーディオコンテンツを再生しなくなります。 まだPlayer V3を使用しているお客様は、Player V4に移行する必要があります（を参照）。 Player V3からの移行 そして プレイヤー移行に関するFAQ).
Ooyala Player V3は、次のような高度なメディアプレーヤーです。
  • モバイルデバイスとデスクトップデバイス間での再生用に最適化
  • カスタマイズされたユーザーエクスペリエンスを確実にするために高度に設定可能
  • 特に大規模ネットワークや放送局のためのスケーラビリティのために設計されています
  • Ooyalaの分析とサードパーティのサービスとの統合

Types of Players

Ooyala provides the following types of players:
  • Web players that run in the browser environment on desktops and mobile devices. There are two types of web players: HTML5プレーヤーV3（廃止予定） そして Flash Player V3（廃止予定）. To learn more about which player to use for your needs, see Player V3 - HTML5とFlash Playerの比較（廃止予定）.
  • Mobile players that run as native iOS or Android apps and are implemented using Ooyala mobile SDKs. For details, see モバイルプラットフォーム開発.

カスタム性

Ooyala provides many options for customizing the player to enhance the user's playback experience.

Design-time player configuration:

  • Backlot GUI (see Setting up Players Using the Backlot REST API (Deprecated))
  • REST API (Chrome) (see プレイヤー（Backlot REST API）)

Run-time programmatic control (web):

  • JavaScript/CSS (HTML5 Player only) (see HTML5プレーヤーV3の基本チュートリアル（非推奨）)
  • JavaScript API (see プレーヤV3 JavaScript APIリファレンス（非推奨）)
  • ActionScript (deprecated Flash Player v2 only) (see プレーヤActionScript API（プレーヤV2は廃止予定）, V2プレーヤAPI（プレーヤV2は廃止予定） そして Differences Between Player V3 and Player V2 (Deprecated)).

Run-time programmatic control (mobile):

  • Android用モバイルSDK
  • iOS用モバイルSDK

参照してください Use Cases for Customizing Player V3 (Deprecated) to help you decide which approach is best for your needs.

You can also customize the playback experience by building playlists using ThemeBuilder (see Theme Builder Customization for Player V3).

Integration and Extensibility

You can extend player functionality, such as adding monetization functionality, by integrating with third-party components (see Player V3 Integrations).
