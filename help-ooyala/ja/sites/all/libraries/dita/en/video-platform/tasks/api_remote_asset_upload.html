Backlot REST APIを使用したリモートアセットの追加

リモートアセットを管理する前に、それをBacklotに追加する必要があります。

注意： Backlot REST APIコマンドについて詳しくは、以下を参照してください。 バックスロットAPIリファレンス.

リモートアセットを追加するには：

使用 / v2 / assets ルート。 次の例では、 "My Remote Asset"リモートアセットを作成します。 
[POST] / v2 / assets {"name"： "私のリモートアセット"、 "asset_type"： "remote_asset"、 "duration"：120000、 "stream_urls"：{"flash"： "http://mydomain.com/ my_flash_file.flv "、" iphone "：" http://mydomain.com/iphone_compatible_file.mp4 "}}

バックスロット 次のような応答を返します。

{"asset_type"： "remote_asset"、 "期間"：120000、 "名前"： "私のリモートアセット"、 "preview_image_url"：null、 "created_at"： "2011-05-16T20：35：53：00：00" 、 "embed_code"： "Y1NWZnMjq-DPsM2"、 "stream_urls"：{"フラッシュ"： "http://mydomain.com/my_flash_file.flv"、 "iphone"： "http://mydomain.com/iphone_compatible_file.mp4 "}、" time_restrictions "：null、" updated_at "：" 2011-06-02：00 + 08：58 "、" external_id "：" myExternalId "、"元のファイル名 "：null、"説明 "：null、"ステータス "："ライブ "}
注意： Ooyala Scratchpadでアカウントの認証情報を使用してコードサンプルを試してみてください。 スクラッチパッドを起動するには、にアクセスしてください。 Ooyala APIスクラッチパッド。 スクラッチパッドの使用方法については、 スクラッチパッドを使った練習の依頼.

リモート資産が正常に追加されました。

注意： あなたはリモートアセットとしてVR 360ビデオを追加することができます。 そうするには、追加のクエリパラメータを含める必要があります（vr360type これは、VR 360ビデオのタイプを指定します（立体視または立体視）。 見る VR 360コンテンツのアップロード 詳細については。
