ライブストリームを追加する

ライブストリームを追加すると、からURLを受け取ります。 Ooyala あなたのエンコーダで設定すること。

注意： Backlot Liveストリーミングは推奨されておらず、無効にされる予定です。 詳細と代替案については、 OVPリリースノート.

最高品質のストリーミングを実現するには、ブロードキャスト元の場所でエンコーダIPを指定し、すべてのライブストリームをエンコードして配信するのに十分な専用帯域幅があることを確認してください。

注意： 新しくプロビジョニングされたライブストリームが使用可能になるまでに最大で30分かかります。

ライブストリームを追加するには：

  1. 使用 / v2 / assets ルート。 POST本文で、asset_typeをlive_streamに、is_flashをtrueに設定し、エンコードを指定します。
    注意： iPhoneおよびiPad（iOSデバイス）へのライブストリーミングの場合は、is_iosもtrueに設定します。
    次の例では、3つのFlashエンコーディングとiOSライブサポートを使用してライブストリームを作成します。 
    [POST] / v2 / assets {"name"： "FlashとiOSのライブストリーム"、 "asset_type"： "live_stream"、 "primary_encoder_ip"： "8.8.8.8"、 "password"： "encoder_password"、 "is_flash"：true、 "is_ios"：true、 "エンコーディング"：[{"width"：800、 "height"：600、 "bitrate"：600}、{"width"：800、 "height"：600、 "bitrate"：300} 、{"幅"：400、 "高さ"：300、 "ビットレート"：200}]}

    バックスロット 次のような応答を返します。

    {"名前"： "FlashおよびiOSライブストリーム"、 "preview_image_url"： "資産タイプ"： "ライブストリーム"、 "期間"： "0"、 "埋め込みコード"： "FlczhuMzocRFR3YwMD0bxxo7wOtQY8-wN"  -  "X"、 "X" 2012-03T02：00：42 + 13 "、"エンコーディング "：[{" bitrate "：00、" height "：00、" width "：200}、{" bitrate "：300、" height "：400 、 "width"：300}、{"bitrate"：600、 "height"：800、 "width"：600}]、 "is_ios"：true、 "time_restrictions"：null、 "updated_at"： "600-800- 2012T03：02：00 + 42：15 "、"osted_at"：null、 "original_file_name"：null、 "is_flash"：true、 "ios_live_upload_url"： "http：//post.ooyala-i"。 akamaihd.net/00/00dayarchive/ VpNDqXTEbTE_201184UgsKdpQcDjcJATg / FlczhuMzocRFR1YwMD5bxxo3wOtQY0-W / master.m7u8 " "ios_live_delivery_url"：" http://ooyala-i.akamaihd.net/hls/live/3/8dayarchive/ VpNDqXTEbTE_201184UgsKdpQcDjcJATg / FlczhuMzocRFR1YwMD5bxxo3wOtQY0-W / master.m7u8 "、" backup_entry_point "：" rtmp：/ /b.ep3.i.akamaientrypoint.net/EntryPoint "、" primary_entry_point "：" rtmp：//p.ep8.i.akamaientrypoint.net/EntryPoint "、"説明 "：null、"状態 "："ライブ "}
    注意： Ooyala Scratchpadでアカウントの認証情報を使用してコードサンプルを試してみてください。 スクラッチパッドを起動するには、にアクセスしてください。 Ooyala APIスクラッチパッド。 スクラッチパッドの使用方法については、 スクラッチパッドを使った練習の依頼.
  2. エンコーダの設定については、Flashについては、 primary_entry_point そして backup_entry_point.
  3. エンコーダの設定については、iOSの場合は、 ios_live_upload_url そして ios_live_delivery_url.
    これらのプロパティの値は以下の形式を取ります。 の pcode プロバイダコードまたはアカウントIDです（詳細については、 あなたのAPIクレデンシャル）。 ザ 埋め込みコード ライブストリームの埋め込みコードまたはコンテンツIDです。 Spinnakerの対応するフィールド名も表示されています。
