バーチャルアセット

仮想アセットは個々のプログラムまたはイベントを表します。

カスタムキーと値のペアのメタデータは、通常のアセットのカスタムメタデータと同じ規則に従って、仮想アセットに関連付けることができます。 見る カスタムメタデータ.

APIの仮想アセットは以下の構造を持ちます。
注意： external_id 仮想資産のIDを外部システムにマッピングするためのオプションのフィールドです。 
{「名前」：「ユタフットボール」、「external_id」：「PAC_CLF00011」、「id」：「999fffdc884841aeb05fc9e5c6fcf958」}

すべてのバーチャルアセットを見る

[GET] / v2 / virtual_assets

単一の仮想アセットを表示する

[GET] / v2 / virtual_asset /virtual_asset_id

仮想資産のトラックセグメントを表示する

[GET] / v2 / virtual_assets /virtual_asset_id/ track_segment_instances

仮想資産のインスタンスを表示する

[GET] / v2 / virtual_assets /virtual_asset_id/ asset_instances

仮想アセットを作成する

[POST] / v2 / virtual_assets {
   プロパティ

}

仮想資産を変更する

[パッチ] / v2 / virtual_assets /virtual_asset_id{"名前"： "ユタフットボール2012"}

仮想資産メタデータの設定/置換

[PUT] / v2 / virtual_assets /virtual_asset_id/ metadata {"name1「」value1「」name2「」value2「」name3「」value3「……」name100「」value100"}

カスタムメタデータを変更する

[パッチ] / v2 / virtual_assets /virtual_asset_id/ metadata {"name1「」value1「」name2「」value2「」name3「」value3「……」name100「」value100"}

仮想アセットのメタデータを取得する

[GET] / v2 / virtual_assets /virtual_asset_id/メタデータ

仮想資産を削除する

[削除] / v2 / virtual_assets /virtual_asset_id

ルート属性

次の表は、ルートを通じて表現できるすべての属性を示しています。

ルート属性 説明
virtual_asset_id

仮想資産のID。
