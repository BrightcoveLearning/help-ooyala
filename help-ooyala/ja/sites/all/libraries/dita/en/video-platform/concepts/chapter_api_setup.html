Backlot REST APIを使用する

Ooyala RESTベースのインターフェースを提供します。 バックスロット プラットフォームを提供します。

注意： Backlot REST APIコマンドについて詳しくは、以下を参照してください。 バックスロットAPIリファレンス.

ユーザーアカウントに対してAPIアクセスが有効になっている場合 Ooyala APIキーと秘密キーを提供します。 あなたが使用する場合 Ooyala SDKは、これらの認証情報に基づいて自動的に署名を計算します。 直接HTTP呼び出しをする場合は、署名を計算するためのコードを書く必要があります。

  • 共通配置アーキテクチャ：複数層
    Ooyalaは、サーバー配備には多層設計をお勧めします。
  • ベストプラクティス
    Ooyalaは、Webサイト上で豊富でユニークなエクスペリエンスを作成できるようにする一連のAPIを提供します。 従うべきベストプラクティスはここにあります。
  • Webプログラミングのためのアドバイス
    これを使用する開発者向けの推奨事項は次のとおりです。 Ooyala API、およびWebのプログラミングに関する推奨事項から恩恵を受けることができるユーザー。
  • RESTの概要
    内のすべてのリソース バックスロットAPI 独自のURLを持っています。 リソースのURLにリクエストを送信し、リクエストで標準のHTTPメソッドを使用することで、それらのリソースを表示、変更、削除できます。
  • 文字エンコードとMIMEタイプ
    でデータをエンコードする UTF-8 そして適切な設定 MIME ボディを含むリクエストを入力します。
  • APIサーバーエンドポイント
    これらのエンドポイントに要求を出してください。
  • あなたのAPIクレデンシャル
    Ooyala APIの認証情報には、APIキーとシークレットが含まれています。これらは、HTTPリクエストに署名するため、およびスクラッチパッドを使用するときに提供するものです。
  • 要求に署名するための一般的なアルゴリズム
    へのすべての要求 バックスロット 認証には、APIキー、リクエストの有効期限、および署名という3つのクエリ文字列パラメータが必要です。 このトピックでは、署名を生成する方法について説明します。
  • 要求の有効期限を設定する
    あなたが設定する必要があります 有効期限が切れます HTTPリクエストのパラメータ。
  • API ping URL
    API Ping URLは、ファイルトランスコードプロセスが成功したかどうかにかかわらず、ファイルトランスコードプロセスが終了するたびに呼び出されます。
  • スクラッチパッドを使った練習の依頼
    スクラッチパッドで練習要求をすることができます。
  • ラッパーSDK / APIライブラリ
    さまざまな言語用のSDKがいくつかあります。
  • APIレート制限
    Ooyala APIを通して、あなたは毎分300リクエストを許可されます。 毎分、カウンターはリセットされます。
  • 高性能APIエンドポイント
    高性能APIエンドポイントは、大容量Webサイトからの繰り返し要求をキャッシュします。
  • JSONP
    JavaScriptのコールバック関数の名前は のGET 高性能APIエンドポイントへの要求。
  • HTTP応答コード
    要求結果に応じて、Backlot REST APIはさまざまなHTTP応答コードを返します。
  • v2 Analytics APIを使用してデータをエクスポートする方法（廃止予定）
  • APIリファレンス
    Backlot API Referenceに完全なドキュメントがあります。
