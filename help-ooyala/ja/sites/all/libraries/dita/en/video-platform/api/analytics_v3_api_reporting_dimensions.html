レンディション

Here are the reference details for all predefined dimensions for the v3アナリティクス レポートAPI.

Dimensions are specified with the 寸法= query string parameter. Some considerations:
  • 最大1月の時間範囲では、無制限のデータ行を使用して、一度に最大3ディメンションで照会できます。
  • 1か月を超える時間範囲では、最大2行のデータを使用して、一度に最大10,000ディメンションで照会できます。
    注意： 1ディメンションは、 URL クエリ
  • 複数の値は、スペースを入れずにコンマで区切る必要があります。
  • 倍数の順序は重要ではありません。
    • ディメンション= device_type、dma と同じです。 ディメンション= dma、device_type

レポートGETの一般的な構文

ルートとクエリ文字列の基本的な構文は次のとおりです。 読みやすくするために、単一行の要求は複数の行に分割されています。 

[GET] / v3 / analytics / reports /？
            report_type=type
            ＆大きさ=大きさ
            ＆メトリクス=メトリクス
            ＆フィルター=filter_type == 'filter_value'
            ＆START_DATE=date
            ＆終了日=日付＆other_parms
            ＆api_key = your_api_key
  • 必須 クエリ文字列パラメータ（太字で表示）は report_type, START_DATE、そしてapi_key。
  • 次元が指定されていない場合は、すべての次元にわたる合計値が返されます。
  • 最大1月の時間範囲では、無制限のデータ行を使用して、一度に最大3ディメンションで照会できます。 1か月を超える時間範囲では、最大2行のデータを使用して、一度に最大10,000ディメンションで照会できます。
  • メトリックが指定されていない場合は、すべてのメトリックが返されます。
注意： APIキーはクエリパラメータとして、またはヘッダX-API-KEYを介して渡すことができます。
注意： 現時点では、 report_type is パフォーマンス.
注意： 1ディメンションのみを使用できます（URL）のための URL クエリ

ロングクエリのレポートの一般的な構文POST

HTTP GET仕様の230文字の制限を超えるクエリパラメータを持つクエリの場合は、POSTリクエストを使用してください。 一部のブラウザおよびhttpクライアントは、230文字を超える文字をサポートしている場合がありますが、HTTP GET仕様に違反するクエリについては正式なサポートは提供されません。 POSTリクエストの場合は、クエリ文字列パラメータの代わりにリクエストボディにJSONオブジェクトを渡します。

名前 意味 追加情報

資産

Video or other assets, including both Ooyala asset IDs (embed codes or content IDs) and external identifiers.
APIを使用して無制限の資産データをクエリするには、クエリで次の条件を満たす必要があります。
  1. 1月以下（<32日）の日付範囲を設定してください。
  2. 用途 ディメンション=資産 （追加寸法なし）。
  3. をセットする ページ パラメータ（ここで ページ= XおよびX <=最後のページ）
  4. をセットする 制限 パラメータ（ここで 制限<= 1000）。

To filter the asset dimension by label, include the filter: filters=label=='yourLabelId'

Response also includes: 
{ 
    
   埋め込みコード,名,status,is_live_stream,type

}
  • 名 is the title of the video.
  • 埋め込みコード is the video identifier (content ID or asset ID).
  • status is the video's status in ウーヤラバックスロット: live or paused.
  • is_live_stream indicates whether the stream is ライブ or video-on-demand (VOD).

国

Country code. See 国と場所のコード.

Response includes full country name.

In the Analytics UI, part of Geography.

地域

Geographic region. Our geographic reporting uses the definition of region provided by Quova.

You cannot specify 地域 alone; you must also specify its 国.

In the Analytics UI, part of Geography.

DMA

Defined marketing area (a US-centric concept not prominent in other parts of the world), specified by DMA ID.

Response includes DMA ID and a descriptive name. In the Analytics UI, part of Geography.
状態 State name. For example, if you drill down by Geo on the United States, you will see California, Arizona, etc. Our geographic reporting uses the definition of state provided by Quova (state is "the first-level administrative division" of a country).

You cannot specify 状態 alone; you must also specify its 国.

In the Analytics UI, part of Geography.

デバイスタイプ

Device type.

See possible values in プラットフォーム、デバイス、およびオペレーティングシステムのコード.

ドメイン

Internet domain (fully qualified domain name).

 In the Analytics UI, part of Traffic Source.

URL

Uniform resource locator (URL).

注意： 1ディメンションのみを使用できます（URL）のための URL queries. You can only apply URL or ドメイン filters for URL クエリ
In the Analytics UI, part of Traffic Source.

os

Operating system of the user's device. Must be exact string; partial matches not supported.

See possible values in プラットフォーム、デバイス、およびオペレーティングシステムのコード.

ブラウザ

 User's web browser.

注意： ブラウザ dimension applies only when デバイスタイプ is デスクトップ.

See possible values in プラットフォーム、デバイス、およびオペレーティングシステムのコード.

pcode

Ooyala-supplied provider ID.

player_id

The identifiers of Ooyala or other named video players used during playback.

注意： ウーヤラモバイルSDK （アンドロイドとiOS）は現在送信しません player_idので、 player_id for the mobile SDKs is attributed as "unknown".
