使い方 ウーヤラIQ with Third-Party Players

あなたは今統合することができます Ooyala サードパーティ製システム（非Ooyala ビデオプレーヤー）。

Ooyala IQは以下の指標を提供できます。
  • Ooyalaアセット（Backlotでホストされている）をプレイしているOoyalaプレーヤー。
  • （新）Ooyalaアセットをプレイしているサードパーティ製プレイヤー（Backlotで開催）.
  • サードパーティの資産（サードパーティのCMSでホストされている）をプレイしているサードパーティのプレイヤー。
このリリースでは統合することができます ウーヤラIQ サードパーティのプレイヤーと一緒に：
  • Webベースのプラットフォームとオペレーティングシステム Ooyala IQ JavaScript SDK.
  • JavaScriptをサポートしていないデバイスは、 Ooyala IQ JSON API.
  • Rokuデバイスは、独自のBrightScriptを使用しています。 Roku用Ooyala IQ SDK.

このリリースでは、すぐに使用可能なSDKまたはアダプタを使用して、またはOoyala IQ JSON APIまたはJS SDKと直接統合することで、Ooyala IQをあらゆるプレーヤまたはデバイスと統合できます。 JavaScriptをサポートするすべてのプラットフォームにJavaScript SDKを使用し、ご使用のプラットフォームがJavaScriptをサポートしていない場合、またはRokuを使用している場合はJSON APIを使用することをお勧めします。

次の図は、上記の統合がOoyala IQとどのように統合されるかを示しています。 ライトブルーのアイテムはあなたが管理し、ダークブルーのアイテムはOoyalaが管理します。

新着情報

サードパーティ（Ooyala以外）のプレーヤーで、OoyalaおよびOoyala以外のビデオアセットの統計情報を表示できます。

ウーヤラIQ サードパーティのプレーヤー分析に合わせてUIが調整されました。 調整内容は次のとおりです。
  • プレーヤー サードパーティのプレイヤーからイベントが発生した場合、パネルにサードパーティのプレイヤーの名前が表示されます。 たとえば、JWプレーヤーからのイベントを分析すると、「JWプレーヤー」が表示されます。 プレーヤー パネル。
  • コンテンツのメタデータがBacklotに取り込まれない場合、Ooyala IQは適切なUIの低下をもたらします。 つまり、Ooyala IQは、サードパーティのCMSシステムからのメタデータの取り込みを必要とせずに、サードパーティのプレイヤーイベントに関する分析を提供できます。 たとえば、Ooyala IQがビデオアセットのタイトルメタデータを取得しない場合は、タイトルの代わりにビデオアセットの外部IDが使用されます（例については下の画像を参照）。
  • Ooyalaプレーヤーとサードパーティプレーヤーを使用してOoyalaビデオアセット（Backlotでホストされている）を再生している場合は、1つのBacklotアカウントを使用してビデオをOoyalaプレーヤーとサードパーティプレーヤーにストリーミングできます。 両方のタイプのプレーヤーについて、1つのダッシュボードに（1つのユーザーアカウントを介して）統合されたIQ分析指標を表示できます。プレーヤーパネルとデバイスパネルを見ることで、Ooyalaプレーヤーとサードパーティプレーヤーのさまざまな指標を見ることができます。 サードパーティのアセット（サードパーティのCMSでホストされている）の場合は、アセットメタデータを取り込むために別のBacklotプロバイダーアカウントを設定する必要があります。

サードパーティのプレイヤーとOoyala Assetsを使ってみる

Ooyalaアセット（Ooyala主催）をサードパーティのプレイヤーでプレイしている場合は、以下の手順に従ってサードパーティのプレイヤーイベントをOoyala IQと統合してください。

  1. Ooyala IQ JavaScript SDKまたはJSON APIを他社のプレーヤーと統合することで、サードパーティプレーヤーがプレーヤーイベントをOoyala IQに伝達できるようにします。Ooyala Ooyala独自のアダプタを使用するか、独自のアダプタを構築することで、ビデオプレーヤーを構築できます。 見る JS SDKを使用したサードパーティ製プレイヤーとIQの統合 JavaScript SDKの詳細については Ooyala IQ JSON API JSON APIの詳細については。
  2. あなたのOoyalaの資産がサードパーティのプレイヤーで再生可能であることを確認してください。
    1. サードパーティのプレイヤーが再生できるストリームを作成するようにアカウント処理プロファイルが設定されていることを確認してください。 サポートしているストリームの種類については、サードパーティのプレーヤーに確認してください。 処理プロファイルの確認については、カスタマーサクセスマネージャーに問い合わせてください。 サードパーティのプレイヤーは、Ooyalaの再生認証APIを使用して、使用可能なストリームにアクセスできます（サードパーティのプレイヤーによって異なります）。
      注意： アセットライブラリをBacklotに取り込む前に、Ooyalaによってトランスコードされたさまざまなストリームをターゲットデバイスでテストして、ストリームが正しく再生されることを確認することをお勧めします。
    2. エンコードタイプを有効にするために処理プロファイルを変更する必要がある場合は、新しく生成されたファイルを取得するために新しいビデオファイルをアップロードするか、ビデオファイルを再エンコードする必要があります。 コンテンツの置き換え（アセットの置き換え）はこのステップでは機能しません。 同じ埋め込みコードを維持するために手動でアップロードせずにビデオを再処理するには、新しいファイルをアップロードするか、Brightcoveテクニカルサポートでサポートチケットを開く必要があります。
    3. Backlot APIを使用して/ assets / <ASSET EMBED CODE> / streamsをクエリし、目的のエンコーディングのURLをコピーします。
  3. 動画へのファイルパスとして、ステップ2cから収集したエンコードURLを指定して、通常どおりにサードパーティプレーヤーを作成します。
  4. 使用 Ooyala Analytics Chromeプラグイン アナリティクスpingがOoyalaバックエンドに正しく送信されていることを確認するか、または Ooyala Backlotおよびイベントpingで資産メタデータを検証するためのCustomer Success Manager。

サードパーティプレーヤーがOoyalaアセットを再生できるようにした後、Ooyalaアセットに対して分析イベントが発生すると、IQアカウントは自動的にサードパーティプレーヤーのメトリクスの表示を開始します。

サードパーティ製プレイヤーとサードパーティ製アセットを使ってみる

サードパーティのプレイヤーを使用してサードパーティのアセット（Ooyalaがホストしていない）をプレイしている場合は、以下の手順に従ってサードパーティのプレイヤーイベントをOoyala IQと統合してください。

  1. 試用版のBacklotアカウントを作成して、そのアカウントを使用してBacklotでサードパーティの資産とメタデータを表示できるようにします。 無関係なBacklot機能を取り除いたBacklotの簡単なバージョンを見るでしょう。
  2. アクセス ウーヤラIQ at http://analytics.ooyala.com またはBacklot ANALYZEタブから。
  3. （オプション）コンテンツメタデータを他のコンテンツメタデータと同期します。Ooyala CMSと Ooyala あなたがIQスタンドアロンであなたのビデオを識別することができるようにBacklot CMS。 メタデータを同期しないと、IQスタンドアロンユーザインタフェースでビデオタイトルを表示できなくなるため、これを行うことを強くお勧めします。 見る Ooyala IQとサードパーティのメタデータを同期させる コンテンツメタデータを取り込む方法の詳細については。 一緒に仕事をしたい場合は、カスタマーサクセスマネージャーに相談してください。 Ooyala このステップのプロフェッショナルサービス
  4. 関連のあるSDKをあなた以外の人と一緒に実装してください。Ooyala ビデオプレーヤー 一緒に仕事をしたい場合は、カスタマーサクセスマネージャーに相談してください。 Ooyala このステップのプロフェッショナルサービス
  5. 使用 Ooyala Analytics Chromeプラグイン アナリティクスpingがOoyalaバックエンドに正しく送信されていることを確認するか、または Ooyala Backlotおよびイベントpingで資産メタデータを検証するためのCustomer Success Manager。
