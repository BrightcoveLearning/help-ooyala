カスタム分析を構成するには、基本的に2つのステップがあります。
  1. Analytics REST APIを使用したカスタムタグの作成
  2. プレーヤーの作成時にカスタムタグを送信するようにプレーヤー作成コードを変更する

この例では3つのカスタムタグを扱います。 すべてのタグにはオプションの表示名があり、APIを介して管理できます。 Backlotでカスタム分析を表示すると、次のフォーマットが表示されます。[表示名]（タグID：[タグ]）

例えば：
  • 標準（タグID：標準）
  • カスタムウィジェット（タグID：カスタム）
  • ツールバー（タグID：ツールバー）
  1. 次の例では、Ooyala REST APIを使用して「標準」、「カスタム」、および「ツールバー」タグを作成します。 
    [PUT] / v2 / analytics / tags / standard {"display_name"： "Standard"} [PUT] / v2 / analytics / tags / custom {"display_name"： "Custom"} [PUT] / v2 / analytics / tags / toolbar {“ display_name”：“ツールバー”}
  2. JavaScriptで、タグ識別子を割り当てます。 
    var tags = new Array（）; / *ウィジェットの種類に応じて正しい値を割り当てます。* / if（widgetType ==“ standard”）tags [0] =“ standard”; if（widgetType ==“ custom”）tags [0] =“ custom”; if（widgetType ==“ toolbar”）tags [0] =“ toolbar”;
  3. 送信するようにあなたのプレイの埋め込みを変更します タグ プレーヤーが作成されたときの配列。 以下は、プレーヤV3作成の基本的な例です。 
     window.player = OO.Player.create（playerID、embedCode、{"分析"：{"タグ"：タグ}}）;

カスタム分析が構成されています。 あなたはを通して結果を見始めることができます バックスロットUI or バックスロットAPI.
