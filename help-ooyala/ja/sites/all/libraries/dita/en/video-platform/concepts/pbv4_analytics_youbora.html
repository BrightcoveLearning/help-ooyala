YOUBORA Analyticsプラグイン

プレーヤV4を使う ニースの人々の仕事 （NPAW）YOUBORAプラグイン（youbora.min.js）で、Ooyala Player V4のNPAW YOUBORAサービス品質（QoS）分析を追跡します。 YOUBORA統合に関する一般的な情報については、 NPAWのYOUBORAを搭載したOoyala QoSソリューション.

再生中の実行時の流れ

再生中、再生イベントはYOUBORA Analyticsへのイベントに関する通知をトリガーします。

対応プレイヤー

Player V4 YOUBORA Analyticsプラグインは、YOUBORA AnalyticsとOoyala HTML5 Webプレーヤーのバージョン4.6.9以上の統合をサポートします。

必須条件

YOUBORAを搭載したOoyala QoS Solutionを有効にする前に、以下との顧客関係が必要です。 NPAW YOUBORAのアカウントで。

要件

分析目的でビットレート情報を追跡する場合は、次のいずれかのビデオプラグインを使用する必要があります。
  • DASHおよびHLS用のBitmovinプラグイン（bit_wrapper.min.js（SafariのHLS HTML5を除く）
  • AkamaiパッケージHDS用のAkamai HDビデオプラグイン（akamaiHD_flash.min.js)
    注意： プレーヤーV4用のAkamai Packaged HDS用Akamai HDビデオプラグインは推奨されておらず、無効にする予定です。 詳細と代替案については、 OVPリリースノート.
  • HDS用OSMF Flashプラグイン（osmf_flash.min.js)
    注意： Player V4用HDS用OSMF Flash Videoプラグインは推奨されておらず、無効にする予定です。 詳細と代替案については、 OVPリリースノート.
HLSとMP4メインビデオプラグイン（main_html5.min.js）ビットレート情報は報告されません。

ステップ1：YOUBORAプラグインをロードする

プレーヤーにYOUBORAプラグインを追加するには、（プレーヤーファイルに加えて）次のいずれかのURLを使用してプラグインをロードします。
  • http://smartplugin.youbora.com/v5/javascript/ooyalav4/stable/sp.min.js
  • https://smartplugin.youbora.com/v5/javascript/ooyalav4/stable/sp.min.js
NPAWはPlayer V4用のYOUBORA Analyticsプラグインを維持しているので、最新バージョンをロードするためにこれらのリンクの1つを使用してください。
注意： 他のPlayer V4プラグインとは異なり、このプラグインを自分でホストしようとすることはお勧めできません。
これをロードする方法のいくつかの例はここにあります <HEAD> あなたのWebページのセクション：
ロード方法 例
ハードコードされたHTTP 
<script src = "http://smartplugin.youbora.com/v6/js/adapters/ooyala/6.2.3/sp.min.js"> </script>
ハードコードされたHTTPS 
<script src = "https://smartplugin.youbora.com/v6/js/adapters/ooyala/6.2.3/sp.min.js"> </script>
呼び出し元のページのプロトコル 
<script src = "// smartplugin.youbora.com/v6/js/adapters/ooyala/6.2.3/sp.min.js"> </script>

ステップ2：ページレベルパラメータの指定

他のPlayer V4プラグインと同様に、YOUBORAパラメータを指定する必要があります。 playerparams プレーヤーの作成中に渡される変数（を参照） プレーヤV4のページレベルパラメータ):
var playerParams = {"youbora"：{//アカウントコードは必須です "accountCode"： "myCode"、//その他のオプションYOUBORA params}} OO.ready（function（）{window.pp = OO.Player.create（ 'playerDiv） '、' myVideoEmbedCode '、playerParams）;}
以下のパラメーターを指定できます。 を参照してください ユブラ 各パラメーターに関する追加情報については、製品資料を参照してください。
パラメーター データ型 説明
口座番号 {文字列} 必須。 NicePeopleAtWorkによって提供されています。 データの送信先の顧客アカウントをYOUBORA Analyticsに指定します。
ユーザ名 {文字列} システムに関連付けられているユーザーアカウント。 YOUBORA Analytics内でユーザーを識別するために使用されます。 詳しくは、 ユブラ 製品ドキュメント
content.transactionCode {文字列} YOUBORA Analyticsにトランザクションコードを指定します。 これは、YOUBORA Analyticsプラットフォームで収集されたデータをソートするときにユーザーをフィルタリングするために使用できる汎用文字列です。 通常、NPAWは、顧客がそのコンテンツ管理システム（CMS）または有料取引と相関できるパラメーターを使用することをお勧めします。 例：purchaseCode、sessionCodeなど。 詳しくは、 ユブラ 製品ドキュメント
parse.HLS {ブール} YOUBORAの解析HLSアルゴリズムを設定します。 以下のいずれかの値
  • parse.HLS = true 解析HLSアルゴリズムを有効にします。
  • parse.HLS = false 解析HLSアルゴリズムを無効にします。
詳細については、 ユブラ 製品ドキュメント
parse.cdnNode {ブール} YOUBORAのCDNノード検出を設定します。 以下のいずれかの値
  • parse.cdnNode = true CDNノード検出を有効にします。
  • parse.cdnNode = false CDNノード検出を無効にします
詳細については、 ユブラ 製品ドキュメント
余計 {オブジェクト} YOUBORA Analyticsで収集されたデータをフィルタリングするのに役立つ、プラグインに渡す追加のカスタムパラメータ。 キー名パターンparamNを使用して、最大10個のパラメーターを指定できます。ここで、Nは数字1 - 20です。 値は、あなたがYOUBORAに送信したいデータを表します。 YOUBORAのサイトでは、各キーの意味をマッピングします（たとえば、extraparam1は1つのことを意味し、extraparam2は他のものを意味する、など）。 
extraparam.1:'param1',
extraparam.2:'param2',
extraparam.3:'param3',
//Up to 20
詳細については、 ユブラ 製品ドキュメント
