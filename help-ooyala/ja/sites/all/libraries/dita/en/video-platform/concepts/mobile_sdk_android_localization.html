ユーザインタフェースのローカライズ

UIテキストにカスタムローカライズ文字列を設定できます。

Mobile SDKのユーザーインターフェイスでテキスト文字列の言語を変更するには、次のキーと値のペアをリソースファイルに追加します。 res / values / strings.xml。 その後、各値を好きなようにカスタマイズできます。 
<string name = "oo_live">ライブ</string> <string name = "oo_subtitles_title">字幕</string> <string name = "oo_subtitles_none">なし</string> <string name = "oo_subtitles_cc">字幕<< / string> <string name = "oo_subtitles_en">英語</string> <string name = "oo_subtitles_es">スペイン語</string>
希望する言語ごとに、2文字のISO 639-1言語コードを oo_subtitles_ キー。 たとえば、上記のスペイン語のように： oo_subtitles_es.

デバッグを支援するために、Mobile SDKはクラスによってリソースファイルから値を検索します。 getContext（）。getResources（）。getIdentifier（name、 "string"、getContext（）。getPackageName（））.

デバイス言語に基づくローカライズの有効化

あなたの中で アクティビティ初期化する前に、次のコード行を追加します。 OoyalaPlayer。 アプリケーションごとに一度だけこれを実行する必要があります。
String localeString = getResources（）。getConfiguration（）。locale.toString（）; LocalizationSupport.useLocalizedStrings（LocalizationSupport.loadLocalizedStrings（localeString））;

これはActivityの設定からロケールを取得し、現在英語、日本語、スペイン語をサポートするSDK内でローカライズを実行するためにそれを使用します。
