2015-04-2リリース

このリリースでの注目すべき変更には、モバイルSDKリリースとv3プレーヤーのアップデートが含まれます。

ソフトウェア機能リリース

1.3.1をGoogle Playストアにフックします。

このリリースには、以下のアップデートが含まれています。

  • Widevine共有オブジェクトが削除されました。
  • リクエストされたビデオが利用できなかったり、Hook互換エンコーディングを持っていなかったりした場合にアプリがクラッシュする問題は解決されました。
  • このリリースには、Google Analyticsイベントのログ記録の改善が含まれています。

V3プレーヤーのアップデート

AndroidのGoogle IMAプリロール広告でコンテンツの履歴書が送信されなかった問題が解決されました。

Google IMAポストロール広告が一時停止状態から再開せず、代わりに広告が閉じて動画が最初から再開されるというHTML5の問題が解決されました。

Android用モバイルSDKリリース4.0.0

このリリースには、以下のアップデートおよび機能強化が含まれています。

4.0.0をリリース4 / 2 / 15をリリース

アップデートと機能強化

このリリースには、以下の更新および機能拡張が含まれています。

  • Androidサンプルアプリ用のOoyala Mobile SDK GitHubリポジトリが公開されました。 このポータルを使用して、最新のサンプルアプリを入手してください。 詳細については Android Mobile SDKのサンプルアプリケーション.
  • 新しいDevice Managementサンプルアプリは、次のWebサイトで入手できます。 Androidサンプルアプリ用のOoyala Mobile SDK GitHubリポジトリ
  • Googleが推奨するプラクティスに基づいて、Ooyalaは次のように切り替えました。 Androidのメーカー。 すべての顧客アプリケーションはまだ私たちのライブラリと互換性がありますが、私たちのサンプルアプリケーションは Androidのメーカー。 これには、Core SDKとFreewheel、IMA、その他の統合が含まれます。 詳細については iOS Mobile SDKのサンプルアプリケーション.
  • OOOoyalaPlayerViewController APIが変更されました。 このリリースの時点では、インスタンス化する必要があります OooyalaPlayer このプレイヤーを使って OoyalaPlayerLayoutController。 これは、次のコード例に示されています。 
    OoyalaPlayerプレーヤー=新しいOoyalaPlayer（pcode、domain、options）OoyalaPlayerLayoutControllerコントローラー=新しいOoyalaPlayerLayoutController（playerLayout、player）
  • カスタムコントロールフォルダの内容は、削除されたレガシファイルと不要なファイルに更新されます。 詳細については Android用のカスタムコントロールの実装.
  • ClosedCaptionsStyle そして ClosedCaptionsView クラスは com.ooyala.android へのパッケージ com.ooyala.android.captions パッケージ。
  • デバッグモード, OrderedMap, OrderedMapValue他のユーティリティクラスは、 com.ooyala.android へのパッケージ com.ooyala.android.utils パッケージ。
  • クリックスルー広告のアプリ内レンダリングのサポートが利用可能になりました。
  • 次のファイルの可視性がパブリックからパッケージプライベートに変更されました。
    • VASTAd
    • VASTAdPlayer
    • VASTAdSpot
    • VASTLinearAd
    • VastSequenceItem
    • VASTStream
    • VASTUtils
    • VASTWrapperAd

解決済みの問題

このリリースでは、以下の問題が解決されています。

  • アプリがエラーをスローする問題 履歴書（ ） 以前に呼ばれた setEmbedCode 解決されました。
  • SDKは現在、コンソールにエラーイベントを記録します。
  • VisualOnビデオに空白の画面が表示され、再生イベントが開始されたときにビデオが開始されるまでに長い時間がかかるという問題が解決されました。

iOS用モバイルSDKリリース4.0.0

このリリースには、以下のアップデートおよび機能強化が含まれています。

4.0.0をリリース4 / 2 / 15をリリース

アップデートと機能強化

このリリースには、以下の更新および機能拡張が含まれています。

  • iOS用Ooyala Mobile SDKサンプルアプリ GitHubリポジトリが公開されました。 このポータルを使用して、最新のサンプルアプリを入手してください。 詳細については iOS Mobile SDKのサンプルアプリケーション.
    注意： Omnitureサンプルアプリはこのリポジトリには含まれていません。 Omnitureの詳細については iOS上のOmnitureとの統合.
  • Apple Watch用のサンプルアプリは現在、 iOS用Ooyala Mobile SDKサンプルアプリ GitHubリポジトリ
  • OOOoyalaPlayerViewController APIが変更されました。 このリリースの時点では、インスタンス化する必要があります OOOoyalaPlayer このプレイヤーを使って OOOPlayerViewController。 これは、次のコード例に示されています。 
    OOOoyalaPlayer * player = [[OOOoyalaPlayer alloc] pcode：PCODE domain：domain]; OOOoyalaPlayerViewController * controller = [[OOOoyalaPlayerViewController割り当て] initWithPlayer：player];

  • SDKの更新および統合タスクを改善するために、iOS SDKは静的フレームワークを使用するように更新され、静的ライブラリは推奨されなくなりました。 この変更には、コアSDKとFreewheel、IMA、その他の統合が含まれます。

    このリリースのアプリを更新するには、すべて削除しました OoyalaSDK.a ヘッダ参照 OoyalaSDK.framework すべてのヘッダー参照を参照してフレームワーク参照に変更します。 例えば：

    置換：

    #import "OOPlayerDomain.h"

    を使用して：

    #import <OoyalaSDK / OOPlayerDomain.h>
  • [カスタムコントロール]フォルダの内容が更新され、従来の不要なファイルが削除され、次の不足しているファイルが追加されます。
    • OOClosedCaptionsSelectorBackgroundViewController
    • OOClosedCaptionsSelectorViewController

    詳細は iOSのカスタムコントロールについて.

  • クリックスルー広告のアプリ内レンダリングのサポートが利用可能になりました。

解決済みの問題

このリリースでは、以下の問題が解決されています。

  • FreeWheel統合アプリは、広告が利用できない場合、キューポイントイベントで空白の画面を表示します。
  • Google IMA広告がiOSアプリで機能していませんでした。
  • シークイベント後、ビデオの時間とコンテンツが同期していませんでした。
  • シークイベントはライブストリームでアプリをクラッシュさせます。
  • ビデオがGoogle IMA広告のテキストと重なっています。
  • Google IMAの広告終了イベントが失敗しました。
