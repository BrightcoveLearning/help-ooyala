Player V4の基本チュートリアル

このチュートリアルでは、Ooyalaプレーヤーを使ってWebページにビデオを追加する手順について説明します。 プレーヤーとビデオをBacklotで設定し、プレーヤーをWebページに埋め込み、ビデオを再生して設定を確認してから、追加のBacklot設定を設定/テストすることから始めます。
注意： このチュートリアルでは、純粋なSAAS（Software as a Service）モデルのデフォルトのOoyala設定を使用します。Ooyalaでは、すべてのリソース（ソフトウェアパス、プラグイン、およびスキンリソース）をホストして管理します。 これは、Ooyalaプレーヤーを埋め込んで配置する最も簡単な方法を提供します。 この方法が使用されている場合、Ooyalaはすべてのプレーヤーリソースを束ね、最小化し、そしてバグ修正とアップデートを自動的に提供します。

始める前に/前提条件

このチュートリアルを始める前に、次のことを行ってください。
  • Webページ上のHTMLコードを編集する方法を知っています。
  • Backlotユーザーアカウントを持っています。
  • 使用したいビデオをBacklotにアップロードします。
  • ビデオアセットを実行するためのプレーヤーを追加するWebページを用意します。
背景情報については、 プレイヤーV4 そして Player V4を始めよう.

ステップ1：バックスロットにログインする

Backlotにログインします。 https://backlot.ooyala.com/

ステップ2：新しいプレイヤーを作成する

注意： プレーヤーのAPIは OO.ready プレーヤコアが完全にロードされると実行されるコールバック関数を登録することを可能にするメソッド。 プレーヤの作成（つまり呼び出し）は、APIの要件です。 OO.Player.create（））前には起こらないはず OO.ready と呼ばれています。 例えば： 
OO.ready（function（）{player = OO.Player.create（ "コンテナ""YOUR_ASSET_ID"、playerParam）;}）;
  1. [プレーヤーのブランド]タブに移動します（[公開]> [プレーヤーのブランド]の順にクリックします）。
  2. プレイヤー選択サイドバーに行きます。
  3. それを確認する プレイヤー - V4 選択したプレーヤーの種類です。
    注意： 5月以降にアカウントを作成した場合1st 2017、Player V4のみがオプションです。 何らかの理由でPlayer V3にアクセスする必要がある場合は、Ooyalaテクニカルサポートに連絡してください。
  4. ストリーミングを停止する場合は上部の +新しいボタン。 Backlotは新しいプレイヤーをリストに追加します。
  5. [プレーヤ]タブで、[プレーヤ名]を[新しいプレーヤ]からわかりやすいものに変更します（例： myPlayer).
一般情報については、 Backlotでプレイヤーを作成する.

ステップ：3：プレーヤーをビデオアセットに割り当てる

  1. [管理]タブに移動します。
  2. リストからビデオを選択してください。
  3. 埋め込みサブタブに移動します。
  4. [カスタムプレーヤー]の下の下向き矢印をクリックし、作成したばかりのプレーヤーを選択します。
    注意：
    • コンテンツIDは、選択した動画にOoyalaが割り当てたIDです。
    • プレイヤーIDは、選択したプレイヤーにOoyalaが割り当てたIDです。
  5. 詳細サブタブに行き、プレビューウィンドウの再生ボタンをクリックして、ビデオが正しく動作することを確認します。

ステップ5：再生テスト

  1. ブラウザでページをロードしてください。
  2. ビデオをクリックして実行します。
あなたのビデオはページ内で再生されるはずです。 そうでない場合は、私たちのを確認してください プレイヤーV4に関するよくある質問.

ステップ6：Backlotの他の設定を構成しテストする

  1. Backlotで、PUBLISH> Player Branding> Playerに移動します。
  2. [プレーヤー]タブで、再生中に現在のビットレートを表示するには、[ビットレート]オプションを有効（チェック）にします。
  3. [プレーヤー]タブで、[常にスクラバーバーを表示する]オプションを有効（チェック）にして、スクラバーバーが常に表示されるようにします（自動的に非表示にはなりません）。
  4. ブラウザでページをロードしてください。
  5. ビデオをクリックして実行します。
  6. 再生中に、ビットレートコントロールが見えるようになったことを確認します。
  7. 他の設定を確認して、どのプレーヤーのUIコントロールがWebサイトに最適かを判断します。
注意： プレーヤーの設定はGoogleのCDNを介して配布されるため、埋め込みプレーヤーに適用された変更はすぐにはわかりません。 すべての地域に適用するには最大で1時間かかります。

一般情報については、 バックスロットでのプレイヤーの設定。 あなたがあなたがあなたのBacklot設定で見るものよりあなたのプレーヤーのユーザーインターフェース上でより多くのコントロールを必要とするならば、他のオプションを見てください。 プレーヤV4外観のカスタマイズ.

ステップ7：再生終了時の動作を設定してテストする（オプション）

再生がビデオで終了したときの動作を設定できます。 Ooyalaアカウントの設定方法に応じて、検出設定または画面終了設定のいずれかを設定できます。

どのオプションを使用するべきかを決定するために：

  1. Backlotで、PUBLISH> Player Brandingに進みます。
  2. 一番右側のサブタブを見てください。
    • Discovery アカウントに対して検出が有効になっていることを意味します。
    • 終了画面 アカウントで検出が有効になっていないことを意味します。
  3. アカウントに適用される説明（下記）を完了してください。

ステップ7.1 - 終了画面設定の構成（オプション）

  1. PUBLISH>プレーヤーのブランド設定>画面の終了に進みます。
  2. [画面の終了]タブで、[タイトル]と[説明]を有効にします。
  3. [管理]> [詳細]に移動して、ビデオのタイトルと説明を確認または指定します。
  4. 再生が終了したら、ビデオのタイトルと説明が表示されることを確認します。
重要： 画面上のテキスト（タイトル/説明）を使用すると、プレーヤーはプレーヤーの「一時停止」イベントでビデオ画像をぼかします。 これはテキストが読みやすくなるようにするためです。 一時停止中にプレーヤーがビデオ画像をぼかしたくない場合は、Title要素とDescription要素をオフにします。
一般情報については、 バックスロットでのプレイヤーの設定.

ステップ7.2：検出設定の設定（オプション）

アカウントで検出が有効になっている場合は、[検出]タブでプレーヤー設定を構成できます。
  1. PUBLISH> Player Branding> Discoveryに移動します。
  2. まだ選択されていない場合は、プレーヤー選択リストからプレーヤーを選択します。
  3. [探索]タブで、[このプレーヤーの探索を有効にする]を選択します。
  4. ブラウザでページをロードしてください。
  5. ビデオをクリックして実行します。
  6. 再生が終了したら、ビデオのおすすめが表示されることを確認します。
一般情報については、 Player V4でコンテンツを発見する.

ステップ8：Ooyala IQ Analyticsでアクティビティを表示する

  1. 分析>ダッシュボードに移動します。
  2. 動画が再生されたことを示す統計を探します（ 過去の時間の傾向, 今人気の、および トップ10の国).
一般情報については、 Ooyala IQアナリティクスユーザーガイド.
