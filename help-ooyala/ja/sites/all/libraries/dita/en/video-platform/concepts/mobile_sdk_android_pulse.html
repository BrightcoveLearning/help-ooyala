AndroidでのOoyala Pulseとの統合

あなたが必要なもの

Ooyala Pulseを統合するためのOoyala SDKを使用すると、デスクトップで作成したのと同じ広告エクスペリエンスを、サポートされているモバイルデバイスでも作成できます。

Ooyala Player V4とAndroid上のOoyala Pulseからの広告配信を統合するには、次のものが必要です。

  • Android用Ooyala Mobile SDK（ Ooyala ダウンロード)
  • Android用Ooyala Skin SDK詳細については、を参照してください。 Android用プレイヤースキン
  • Ooyala Video Advertising Android 2.x SDK（ Ooyala ダウンロード)
  • Ooyala Pulse Android用Ooyala SDK（ Ooyala ダウンロード)
  • Androidのメーカー

のために Android用Ooyala Mobile SDK, Android用Ooyala Skin SDK、および Ooyala Pulse Android用Ooyala SDK同じ番号のバージョンを使う必要があります。 のために ウーヤラビデオ広告アンドロイド2.x SDK常に最新版を使用してください。

サンプルアプリケーション

Ooyala Pulseサンプルアプリは、次のWebサイトからダウンロードできます。 OoyalaサンプルアプリGit Hub。 この場所には、プロジェクトの設定方法とサンプルアプリの動作に関する情報も含まれています。

使ってみよう

次の手順では、Androidでの広告配信のためのOoyala PlayerとOoyala Pulseの統合を開始する方法の基本的な概要を説明します。

  1. Ooyala Video Advertising Android 2.x SDKおよびOoyala Pulse Android用Ooyala SDKをプロジェクトにインポートします。
    import com.ooyala.pulse; import com.ooyala.android.pulseintegration;
  2. OoyalaPulseManagerオブジェクトを作成し、それをOoyalaPlayerと関連付けます。
    this.manager = new OoyalaPulseManager（this.ooyalaPlayer）; this.manager.setListener（this）;
  3. 必要に応じて、OoyalaPulseManagerのPulsePlayerOptionsオブジェクトを通じて広告プレーヤーのオプションを設定します。 可能なオプションは以下のとおりです。
    • displayAdTitle：広告再生中に広告タイトルを表示するかどうかを示すブール値。 デフォルト値はfalseで、広告タイトルを表示しないことを示します。
    this.manager.setOptions（new PulsePlayerOptions（true））;
    注意： 広告プレーヤーのオプションは、ビデオコンテンツプレーヤーには影響しません。
  4. デリゲートにOoyalaPulseManager.Listenerメソッドを実装します。 このメソッドのすべてのパラメータの初期値は、Backlotアカウントの設定から自動的に取得されますが、必要に応じて、セッションを作成する前に、Ooyala Pulseアカウントのホストの場所、永続ID、コンテンツメタデータ、リクエスト設定などを変更できます。 。 すべての可能なパラメータについては、 AndroidのOoyala Pulseのパラメータ .
    @Override public PulseSession createPulseSession（OoyalaPulseManager ooyalaPulseManager、String video、String pulseHost、ContentMetadata contentMetadata、RequestSettings requestSettings）{//正しいパルスホストとオプションを設定します。Pulse.setPulseHost（pulseHost、nil、nil）; //幅と高さを広告が含まれるビューの幅と高さに設定します。 requestSettings.width = containerView.getWidth（）; requestSettings.height = containerView.getHeight（）; //要求する広告の最大ビットレートを決定する方法を実装します。 requestSettings.setMaxBitRate（BandwidthChecker.getMaxBitRate（））; Pulse.createSession（contentMetadata、requestSettings）を返します。 }

必須パラメータ、カスタムメタデータ、およびビデオプラザ広告セット

Mobile SDKでOoyala Pulse広告配信を利用するには、次のタイプの広告セットを作成する必要があります。 ビデオプラザ Backlotで。 詳細はBacklotユーザーガイドを参照してください。 Ooyalaでは、Ooyala Pulse関連のパラメータをさまざまな場所に保存できます。 優先順位の高い順に、Pulseパラメータとその値は次のように定義できます。

  1. あなたのアプリ自体。
  2. バックスロットで 管理 タブで定義されたカスタムメタデータとしての各ビデオアセット用 カスタムメタデータ をクリックします。
  3. バックスロットで マネー タブ、 広告セット のサブタブ ビデオプラザ 広告セットの種類。
注意： アプリで直接パラメータを設定する場合は、Ooyala Pulse Sessionが作成される前に設定する必要があります。
次のパラメータは、アプリで直接設定するか、Backlotでアセットのカスタムメタデータとして設定するか、またはアセットに関連付けられたVideoplaza広告セット（推奨）で設定する必要があります。
  • パルスホスト：あなたのOoyala PulseアカウントへのURL。 アプリでは、このパラメータを Pulse.setPulseHost（pulseHost、deviceContainer、persistentId） メソッドを設定することもできます。 deviceContainer と persistentId。 設定することをお勧めします deviceContainer 〜へ ゼロ。 例えば：
    Pulse.setPulseHost（ "https://pulse-demo.videoplaza.tv"、nil、nil）;
他の重要なパラメータは次のオブジェクトを通して設定されます。
  • リクエスト設定オブジェクト：Ooyala Pulseから要求する広告の掲載順位と広告の品質に関する情報。 このオブジェクトのプロパティは RequestSettings.
  • コンテンツメタデータオブジェクト：アプリでの再生用に選択された動画コンテンツに関する情報。OoyalaPulseからのターゲット広告のリクエストに役立ちます。 このオブジェクトのプロパティは ContentMetadata.
各パラメータの概要、設定可能な場所、および基本値の場所については、を参照してください。 AndroidのOoyala Pulseのパラメータ.
