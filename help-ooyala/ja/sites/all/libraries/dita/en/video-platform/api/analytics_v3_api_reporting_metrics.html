メトリクスは メトリックス= クエリ文字列パラメータ。 複数の値は、スペースを入れずにコンマで区切る必要があります。

注意： 含めないことで、メトリックのセット全体を取得できます。 メトリクス あなたのクエリで。
警告： 測定基準を指定しない場合 URL 次元、エラーメッセージが返されます。 サポートされている指標 URL 寸法が含まれます：
  • ディスプレイ
  • plays_requested
  • video_starts
  • time_watched
  • player_loads
  • リプレイ
  • uniq_plays_requested
  • uniq_displays
  • uniq_video_starts
  • playthrough_25
  • playthrough_50
  • playthrough_75
  • playthrough_100

特定のメトリックを取得するには、次の例のように指定します。

  • metrics = plays_requested
  • metrics = uniq_plays_requested
  • metrics = Displays、uniq_displays

レポートGETの一般的な構文

ルートとクエリ文字列の基本的な構文は次のとおりです。 読みやすくするために、単一行の要求は複数の行に分割されています。 

[GET] / v3 / analytics / reports /？
            report_type=type
            ＆大きさ=大きさ
            ＆メトリクス=メトリクス
            ＆フィルター=filter_type == 'filter_value'
            ＆START_DATE=date
            ＆終了日=日付＆other_parms
            ＆api_key = your_api_key
  • 必須 クエリ文字列パラメータ（太字で表示）は report_type, START_DATE、そしてapi_key。
  • 次元が指定されていない場合は、すべての次元にわたる合計値が返されます。
  • 最大1月の時間範囲では、無制限のデータ行を使用して、一度に最大3ディメンションで照会できます。 1か月を超える時間範囲では、最大2行のデータを使用して、一度に最大10,000ディメンションで照会できます。
  • メトリックが指定されていない場合は、すべてのメトリックが返されます。
注意： APIキーはクエリパラメータとして、またはヘッダX-API-KEYを介して渡すことができます。
注意： 現時点では、 report_type is パフォーマンス.
注意： 1ディメンションのみを使用できます（URL）のための URL クエリ

ロングクエリのレポートの一般的な構文POST

HTTP GET仕様の230文字の制限を超えるクエリパラメータを持つクエリの場合は、POSTリクエストを使用してください。 一部のブラウザおよびhttpクライアントは、230文字を超える文字をサポートしている場合がありますが、HTTP GET仕様に違反するクエリについては正式なサポートは提供されません。 POSTリクエストの場合は、クエリ文字列パラメータの代わりにリクエストボディにJSONオブジェクトを渡します。 

[POST] / v3 / analytics / reports {
    
    "report_type「」type「」大きさ「」大きさ「」メトリクス「」メトリクス「」フィルター"："（filter_type == \ "filter_value \"）「」START_DATE「」date「」終了日"："日付 "、"other_parms"：" other_param_value "、
    "api_key"： "your_api_key"}

一意のユーザー

あなたはユニークなユーザーの数を測定したいかもしれません。 一意のユーザーを測定するには、 uniq_plays_requested 下記のメトリック の Ooyala モバイルSDKは、アプリケーション固有のランダムな一意のIDを生成して保存します。 他の環境では、一意のユーザーはローカルストレージまたはCookieによって識別されます。 ユーザーがブラウザのキャッシュをクリアすると、そのユーザー/デバイスのIDは次回ビデオを見るときに再生成されます。 生成されたIDは完全にランダムであり、ユーザーを識別できる情報は含まれていません。 そのような情報がユーザに利用可能でないとき、新しいユニークな識別子はそのユーザのために作成されます。 時間の経過とともにユニークユーザーの数を計算するときに重複排除を行います。 たとえば、1日にはユーザーA、B、Cがいます。 2日にユーザーB、Eがいます。次に、日付範囲= XXUMX日と1日を選択すると、合計一意ユーザー数= 2（A、B、C、E）、Daily Avgとなります。 Unique Users =（4 + X NUMX）/ X NUMX = X NUM X（X NUM Xは閉整数に変換されます）。
