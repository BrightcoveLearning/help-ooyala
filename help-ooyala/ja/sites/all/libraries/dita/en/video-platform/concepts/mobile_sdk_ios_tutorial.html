はじめに：iOSサンプルアプリケーション

一般的なXcode開発環境でSDKを設定するために必要なのは、いくつかの短いステップだけです。

SDKを使い始めるには、次のものが必要です。
  • iOS用Mobile SDK自体

    Mobile SDKはからダウンロードできます。 Ooyala ダウンロード。 適切なバージョンをクリックして、Mobile SDKパッケージのzipファイルをダウンロードします。

  • iOSアプリケーションおよびApple iOS SDK 6用のXcode 5以降の開発環境 http://developer.apple.com/xcode/
  • Backlotアカウントにログインして、に移動します。 アカウント > 開発者。 あなたに APIキーピリオドの前の英数字（ "。"）は、 PCODE チュートリアルで使用する定数です。 たとえば、APIキーが xxxxxxxxxxxxxxxxxxxxxxxxxxxx.0000その後、最初の28文字（xxxxxxxxxxxxxxxxxxxxxxxxxxxx） のために PCODE。 詳しくは、 あなたのAPIクレデンシャル.
  • 再生したい動画の埋め込みコード（コンテンツIDまたはアセットID）
iOS用Mobile SDKに慣れるための最善の方法は、サンプルアプリケーションの1つを実行することです。
  1. ダウンロードしたファイルを解凍する OoyalaSDK-iOS.zip ファイル。
  2. ファイルをオープンする OoyalaSDK-iOS / SampleApps / GettingStartedSampleApp / GettingStartedSampleApp.xcodeproj ソースコードはXcodeで表示されています。 左上で、階層表示に切り替えて展開します ViewControllerをを選択し、 〜viewDidLoad メンバー。
  3. 適切な行の各定数または変数の引用符の間に、プロバイダコード（pcode）と埋め込みコードを貼り付けます。 適切な行は次のようになります。 
    NSString * const PLAYERDOMAIN = @ "http // www.ooyala.com"; 
NSString * const EMBED_CODE = @ "ここにあなたの埋め込みコード"; NSString * const PCODE = @"あなたのpcodeはこちら"; ... [ooyalaPlayerViewController.player setEmbedCode：@" EMBED_CODE "];
    • 埋め込みコード：あなたのビデオアセットの一つの識別子。
    • PCODE：あなたのOoyala提供のプロバイダコード
    • プレイヤードメイン：変更せずに http://www.ooyala.com。 この定数は以下と組み合わせて機能します。 シンジケーション制御 （公開ルール） バックスロット。 ビデオにインターネットドメイン制限を設定している場合 バックスロット （参照 バックスロット ユーザーガイド）、ここでの定数はそれらの許可されたドメインの1つに設定することができます。 設定していない場合 シンジケーション制御定数は効果がありません。
    • setEmbedCode（） function：引数として再生したいビデオの埋め込みコード。 この機能は、SDKの主要な「主役」です。
  4. 変更を保存します。
  5. 左上で、 iPhone 6.0シミュレータ.
  6. 左上のをクリックします ラン.
iPhoneシミュレータが表示され、ビデオがロードされます。 再生ボタンをクリックしてビデオを表示します。
注意：

ビデオの表示に問題がある場合は、シンジケーション設定がiPhoneおよびiPadデバイスを許可するように設定されていることを確認してください。 詳細については、Ooyalaの「Publishing Rules」セクションを参照してください。 バックスロット ユーザーガイド.

ビデオがドロップアウトしてもオーディオが再生され続ける場合は、低帯域幅の接続がある可能性があります。 これにより、視聴者はコンテンツを引き続き体験でき、Appleによって設計された期待される動作となります。
