HTTPレスポンスコードとメッセージ

すべての応答の先頭には、次のメッセージ形式が含まれています。 可能な値 ステータスメッセージ そして コード 次の表に説明があります。 特に明記しない限り、これらのコードは ウーヤラIQ レポートAPI エラーの場合、応答にはエラーメッセージと説明メッセージが含まれます。
{ "状態"： "ステータスメッセージ、「statusCode」： コード
  "エラー"： "説明メッセージ"}
例： 
 {"status"： "Invalid parameter"、 "statusCode"： "400"、 "errors"：["フィルタの構文が無効です：無効な文字。 位置4”]}で
HTTPレスポンスコード 一般的な意味 詳細な説明
200 ご注文完了 要求は成功しました。
204コンテンツがありません 正常に受信しました Event Ping APIによって返されます。 レスポンスボディは返されません。
400 不正な要求 要求が正しくありません。 たとえば、必須のクエリ文字列パラメータは含まれていません。
401 許可されていません どちらかの資格情報（Ooyala提供されたAPIキーとシークレット）が見つからないか、それらの値が無効です。
403 不十分な許可 要求内の資格情報に、要求されたデータに対する十分な許可がありません。
429 クォータを超えました リクエスタのAPIコールのクォータを超えました。 クォータについては、 APIレート制限
500 システムエラー で考えられる問題 Ooyala システム。
503 サービスは利用できません 一時的なサービス停止があります。
