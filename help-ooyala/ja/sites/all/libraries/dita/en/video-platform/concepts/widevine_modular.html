Widevineモジュラーコンテンツ保護

Ooyala provides support for Widevine Modular to meet the content protection requirements for high-quality content online in on-demand and live streaming formats. Widevine’s multi-platform DRM provides the capability to license, securely distribute and protect playback of multimedia content on any consumer device. Content owners and digital media providers can use Widevine’s solutions to ensure revenue generating services keep flowing to whatever device consumers desire.

注意： If you are currently using Widevine Classic, please contact Technical Support or your account manager to help you transition to Widevine Modular.
注意： Widevine Classic DRMとのOoyala統合は推奨されておらず、無効にされる予定です。 詳細と代替案については、 OVPリリースノート.

To give you the capacity to protect your content using Ooyala and Widevine, you need to understand how Widevine works and how to use Widevine with your Android or Connected TV device. This document describes how to do this.

Widevine Modular is part of a set of comprehensive content protection features that work together to provide you with the ability to secure your content. These features include:

  • User Authentication through token-based options such as the Ooyala Player Token. For information about setting up and using this feature, see プレイヤーV4のOoyalaプレイヤートークン.
  • Widevineなどのメカニズムによるコンテンツ認証。 Widevineに関する情報はこの文書で提供されています。
  • Authorization API, which is a mechanism that handles all authorization requests. For more information about the Authorization API, see プレーヤV4用プレーヤ認可API.
Ooyalaが提供するコンテンツ保護は、スタンドアロンまたは組み合わせて複数レベルのコンテンツ保護を提供します。 Ooyalaでは、これらの機能を組み合わせてコンテンツ保護戦略を作成することができます。 追加のコンテンツ保護機能（Adobe Passなど）については、営業、カスタマーサクセスマネージャー、またはテクニカルサポートにお問い合わせください。

OoyalaとWidevine DRMのワークフロー

Customers who want Widevine need to work with Ooyala Customer Success Managers or professional services to enable Widevine Modular support. When you use Ooyala’s Widevine server, the licenses exist in the Widevine cloud.

Widevine DRMの実装では、コンテンツが暗号化されていてライセンスなしでは役に立たないため、各コンテンツビューアには個別のライセンスが必要です。 Widevineは暗号化を実行しません、CDNからのコンテンツはすでに暗号化されています。 Widevineは、ライセンスキーの保存と取得のための安全なクラウドを提供するだけです。

Widevineと統合する場合は、コンテンツの保護がOoyalaのコンテンツ保護機能のさまざまな部分とどのように機能するかを理解する必要があります。 次の図は、WidevineがOoyalaプレーヤートークン（オプションですが推奨されるユーザー認証機能）と、ユーザー認証要求を処理するOoyala Authorization APIとの関係を示しています。

Ooyala SDKベースのアプリケーションでWidevineを使用する

To use Widevine with our Android player, you need to download and use our native SDKs to create the client-side pages. To download the Ooyala Android SDK, go to Ooyala ダウンロード. To learn about implementing Android, see Android用モバイルSDK . That’s all you need, as once you have the Widevine Modular feature enabled, the Ooyala SDK-based apps should work seamlessly with Widevine.

Widevineを接続テレビで使用する

Widevineを接続されたテレビで使用したい場合は、次の作業が必要です。
  1. Implement Widevine Modular in accordance with the device specific SDKs. You pre-configure your device SDK and Widevine setup according to the SDK instructions.
  2. WidevineおよびデバイスSDKをSDK固有の言語で初期化します（これはデバイスによって異なります）。 該当するプラットフォームプログラミングガイドに従ってアプリを設定します。
  3. You will make a call to the 認証API （これはJSON RESTful API呼び出しです）
  4. Widevine URLを取得するか、データをストリーミングするために電話をかけます。
  5. 取得したURLの要素をWidevineフィールドの対応する値にマッピングする必要があります。 下記の値を差し込みます。

デバイス登録

When using Widevine Modular, the デバイスID that is cached in the browser might get removed in the following situations:

  1. clearing the browser privacy settings/history/cache
  2. uninstalling or reinstalling the browser

Device registration will fail if the device limit is exceeded because the same device would then have a new デバイスID and be tracked as a new device, see デバイス登録API.
