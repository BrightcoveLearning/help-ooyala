Ooyala Backlot APIリファレンスV1（非推奨）

このバージョンのBacklot APIは非推奨です。 APIのバージョン2で新しい開発作業を行うことを強くお勧めします。 V2 APIの使用方法については、Ooyalaサポートセンターの開発者ドキュメントセクションを参照してください。

このドキュメントはプログラム的な方法でOoyala Backlotシステムと対話するために利用可能なインターフェースに関する情報を含みます。 すべてのBacklot APIはRESTベースであり、リクエストごとに署名が必要です。

  • API V1（非推奨）とAPI V2の違い
    Backlot APIのバージョン1とバージョン2の間には多くの違いがあります。
  • クエリAPI（廃止予定）
    Backlotアカウントのコンテンツに関する情報を要求するために、柔軟なクエリAPIを提供しています。
  • サムネイルクエリAPI（廃止予定）
    Backlotアカウントにアップロードされたすべてのビデオについて、Ooyalaトランスコーディングシステムはおよそ15-30秒ごとにサムネイル画像を作成します。
  • 属性更新API（廃止予定）
    属性APIには、アセットとチャンネルの属性を更新する機能が含まれています。
  • カスタムメタデータAPI（廃止予定）
    カスタムメタデータAPIを使用すると、任意の名前と値のペアを任意のアセットに関連付けることができます。 メタデータは、署名付きのRESTスタイルのGET呼び出しを使用して操作されます。 http://api.ooyala.com/partner/set_metadata.
  • ラベル管理API（廃止予定）
    Backlot Label APIはラベルのプログラムによる制御を可能にします。 APIには、ラベルの作成と削除、ラベルとコンテンツの関連付けと削除、およびラベルの名前変更の呼び出しが含まれています。
  • プレーヤ管理API（廃止予定）
    プレーヤ管理ＡＰＩは、利用可能なプレーヤのリストおよび特定の埋め込みコードへのプレーヤの割り当てを可能にする。
  • チャネルAPI（廃止予定）
    Channel APIはチャンネルラインナップの一覧表示と設定を可能にします。
  • チャネルセットAPI（廃止予定）
    チャンネルセットAPIはチャンネルセットラインナップの一覧表示と設定を可能にします。
  • 動的チャネルAPI（廃止予定）
    動的チャンネルは、一連のビデオを表示するという点で標準のOoyalaチャンネルと似ていますが、明示的なリストではなく、どのビデオを再生するかをプレイヤーに指示しているという点で異なります。
  • 高性能API（廃止予定）
    標準のAPIに加えて、OoyalaはAPIリクエスト用の高性能キャッシングレイヤも提供しています。
  • バックスロットAPIのサンプルコード（非推奨）
    私たちはPHPとRubyでサンプルコードを提供します。
  • 署名生成（廃止予定）
    アカウントのセキュリティを確保するために、APIクエリに署名する必要があります。
