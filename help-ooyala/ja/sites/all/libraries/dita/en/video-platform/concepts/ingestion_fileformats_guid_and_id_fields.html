CSVまたはXMLマニフェストファイルを使用したアセットメタデータの更新

使用 id CSVマニフェストファイルの列または GUID 取り込まれたコンテンツの独自の識別子を指定する要素（XMLマニフェストファイル内）。 後でそれらの識別子を使用して、メタデータを含むアセットを更新できます。

id 列（ CSVマニフェストファイル）と GUID 要素（ XMLマニフェストファイル次のように動作します。

  • ファイルが最初に取り込まれるときにフィールドを明示的に設定しない場合、IDはシステムによって割り当てられた埋め込みコード（コンテンツIDまたは資産ID）です。
  • 最初のアップロードでこのフィールドを設定した場合は、以前にアップロードしたファイルを後で更新するために、CSVまたはXMLマニフェストファイルでそのフィールドを指定する必要があります。
  • CSVまたはXMLマニフェストファイルに、以前にアップロードされたビデオファイルの外部IDまたは埋め込みコード（コンテンツIDまたはアセットID）を含むフィールドが含まれている場合、システムはアセットのメタデータを更新します。
  • CSVまたはXMLマニフェストファイルがビデオファイルを参照している場合、システムは顧客のディレクトリでそのビデオファイルをローカルに検索します。 システムがそれを見つけることができない場合は、エントリはスキップされます。
注意： 外部識別子を含めない場合は、 Studio上ではサポートされていません。 これらのフィールドをNULL値で組み込みます。 NULL値は後で検索できません。 マニフェストファイルからフィールドを省略します。
