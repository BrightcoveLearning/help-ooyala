ウォッチリストにアイテムを追加する

このエンドポイントは、指定されたユーザーの指定されたウォッチリストにアイテムを追加します。

ウォッチリスト内のアイテムの位置

  • 位置の索引付け（番号付け）は0から始まります。
  • 項目はスタック順に挿入されます。 最後に追加された項目は位置0を占めます。 最も最近に追加されたアイテムは位置nを占めます（ユーザーのウォッチリストにn + 1個のアイテムがあるとします）。 例えば、項目Aを追加した場合、Aは位置0です。 アイテムBを追加した場合、Bは位置0、Aは位置1です。
  • 挿入時に、他のすべての項目の位置は自動的に更新されます。

HTTPリクエスト

POST https://player.ooyala.com/watchlist/v3/<watchlist_id>/items?account_token= <account_token> {"item_id"： "<12345>"}

クエリパラメータ

パラメーター 説明 タイプ 必須
api_version でなければなりません v3. 文字列 はい（URL）
watchlist_id アイテムを追加するウォッチリストのID。 ユーザーのアカウントに関連付けられている既存のウォッチリストにリンクされている有効なUUIDである必要があります。 指定しない場合、アイテムはユーザーのデフォルトのウォッチリストに追加されます。 UUIDv4または文字列 いいえ（URL）
account_token 特定のエンドユーザに関連付けられているウォッチリストを参照するためにウォッチリストサービスによって使用されます（UUIDによって識別されます）。 詳しくは、 OoyalaアカウントトークンAPI. 文字列 はい（URL本体）
item_id 追加するアイテム（アセット）のID。
注意： 指定した項目がターゲットのウォッチリストにすでに存在する場合、変更は行われませんがエラーはスローされません。 システムはの有効性をチェックしません。 item_id ウォッチリストに
文字列 はい（ボディ）

リターンコード

コード 理由 説明
201 OK POST要求は成功しました。 新しいアイテムは正常に作成されました。
400 リクエストエラー 次の問題が原因で失敗しました。
  • パラメーターが欠落しているかフォーマットが正しくありません。
  • 指定されたアイテムは既にウォッチリストに存在します。
  • 指定された watchlist_id 空ではなく、存在しません。
401 無断 指定された account_token 無効です。
404 見つかりませんでした サーバーが停止しています。
500 内部サーバーエラー 内部サーバーエラーが発生しました。
