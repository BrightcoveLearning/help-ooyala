トラックセグメント

トラックセグメントは、トラック内のスケジュールされた開始/終了時刻における仮想資産のインスタンスとして定義されます。

各トラックセグメントは、（トラック内だけではなく）スケジュール内のすべてのトラックセグメントにわたって一意の識別子が割り当てられます。 同じトラック内のトラックセグメントは時間的に重なることはできませんが、異なるトラック内のトラックセグメントは時間的に重なることができます。 トラックセグメントは次のような構造をしています。
注意： external_id そして override_stream_asset_id プロパティはオプションです。
{ "ID"： "4fc59badbf9a4f26be1df179082c5459"、 "track_id"： "4fcb0f981d70459a9693472d6d05d7b7"、 "START_TIME"： "2013-08-13T11：42：14Z"、 "END_TIME"： "2013-05-13T13：49：14Z"、 " status "：" online "、" virtual_asset_id "：" 9a6519637be3446e920228c36da18315 "、" external_id "：" PAC12_447355812 "、" override_stream_asset_id "："0yMTVoMzpqSXNXXXXXXXXXXXXXXXXXXXXXXXXXXXXCXXCXXXXXXXXXXXXCX性MXXXXXXXXXXXXXXXXXXXXXX-職のコンバーターの数」

トラックセグメントを作成する

[POST] / v2 / track_segments {
   プロパティ

}

すべてのトラックセグメントを見る

[GET] / v2 / track_segments

1つのトラックセグメントを見る

ユーザーは、Ooyalaによって生成されたパブリックIDを使用することも、外部IDを使用してトラックセグメントを照会することもできます。 
[GET] / v2 / track_segments /track_segment_id

仮想資産情報を含める

クエリ文字列パラメータを指定して、仮想アセットIDだけでなく仮想アセット情報を取得することもできます。 include = virtual_asset. 
[GET] / v2 / track_segments？include = virtual_asset

トラックセグメントへのバッチタイムウィンドウアクセス

スケジュールのスタイルで「今日の最新情報」のレンダリングビューを有効にするには、トラックセグメントに特定の時間枠で一括してアクセスできます（たとえば、「水曜日に始まるすべてのトラックセグメント」）。 次のクエリは、8月の15番目のすべてのトラックセグメント2013を返します。

  • 一方または両方を指定できます。 from_time そして 時間に.
  • これらのパラメータの値はURIエンコードされている必要があります。
  • クエリは、その間のすべてのトラックセグメントを返します。 from_time そして 時間に.
[GET]/v2/track_segments?from_time=2013-08-13-00:00:00Z&to_time=2013-08-14-00:00:00Z

トラックセグメントを削除する

[削除] / v2 / track_segments /track_segment_id

トラックセグメントを修正する

[パッチ] / v2 / track_segments /track_segment_id
 {"start_time"： "2012-08-13T11：30：00Z"}
