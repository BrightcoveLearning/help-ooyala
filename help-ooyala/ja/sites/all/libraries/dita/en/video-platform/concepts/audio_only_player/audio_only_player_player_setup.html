オーディオ専用プレーヤー：プレーヤーの設定

デフォルトのオーディオ専用プレーヤー

Ooyala provides a default Audio-Only Player in addition to the existing default Video Player:

Any new Audio-Only Player you create based on the default one, initially has the following settings enabled by default:

  • ボリュームバー - enabled by default from the プレーヤー configuration sub-tab.
  • Skip Forward Button - enabled by default from the プレーヤー configuration sub-tab.
  • Skip Backward Button - enabled by default from the プレーヤー configuration sub-tab.
  • リプレイ - visible in the Player skin after at least one full play, and enabled by default from the 終了画面 configuration sub-tab.

For Audio-Only Player, the Player setup process is identical to the one used before for Video Players (see Backlotでプレイヤーを管理する). However, there are some differences, as follows:

  • An audio-specific Player is created, which gets identified as audio-only type.
  • アセットは音声のみの出力用に処理されます。
注意： Customers cannot create default Audio-Only Players from the UI or using Player APIs.

  4. Configure the Audio Player's properties in the Player Branding tab - it contains the プレーヤー, ブランディング, 起動画面、および 終了画面 sub-tabs. These naturally differ from the properties available for a Video Player type. The default Audio-Only Player comes with default settings so you can do this configuration at any time later.

    注意：

    Ooyala currently only supports two script-type embed code options - Standard and Advanced HTML5 Player embed code. iFrame embed code is not yet available for Audio-Only Player. The Standard and Advanced embed codes for Audio-Only Player have the same settings as for the Video Player, and we recommend using the Standard embed unless the customer needs broader configuration options on how the Player embed is constructed. The Standard embed option provides all the basic features such as CORE and Skin; the Advanced embed option contains all the scripts for additional configuration. For both options, in the embed code script, ensure that the playerParam に設定されています "playerType":"audio" (var playerParam = {"playerType":"audio"};).
