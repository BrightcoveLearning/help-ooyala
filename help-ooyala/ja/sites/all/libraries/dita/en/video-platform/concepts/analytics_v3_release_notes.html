2018-12-17

このOoyala IQリリースでは、以下の新機能と機能強化が導入されています。

  • ビジネスインテリジェンスページの新しい資産フィルタ

    • カスタムメタデータに基づいてアセットをフィルタリングできるようになりました。
      これを行うには、アセットのフィルタアイコンをクリックして、 メタデータ別 最初のドロップダウンメニューから、カスタムメタデータのキーと値のペアを入力します。
      注意： コンマを使用して、カスタムメタデータの複数のキーと値のペアを追加できます。
    • 事前定義された期間に基づいて資産をフィルタリングできるようになりました。

      定義済みの期間は次のとおりです。

      • 小資産0 - 期間中の5分
      • 中資産5 - 期間中の20分
      • 長資産20 +期間（分）

      資産のフィルタアイコンをクリックし、期間ドロップダウンメニューから期間を選択してこれを行うことができます。

      注意： この期間フィルタは、リリース日以降のトラフィックのある資産に対してのみ機能します。
    • 公開日に基づいてアセットをフィルタリングできるようになりました。

      資産のフィルタアイコンをクリックしてから、 公開日で絞り込む ボタンを押して日付範囲を選択します。

  • マスターアカウントにログインしたときに、ビジネスアカウントページ内のさまざまなサブアカウントからデータをフィルタリングするためのマスターアカウントのユーザーのサポート。

新しいOoyala IQ JavaScript SDK

新しいOoyala IQ JavaScript SDKは、サードパーティのプレイヤーをOoyala IQと統合するために利用可能です。 新しいライブラリーは、Thriftエンドポイントを経由せずに直接イベントをJSON APIエンドポイントに送信します。これにより、既に使用可能だったものに加えて、以下のメトリックを報告することができます。
  • イベントを探す：視聴者がビデオを通してシークする（進行インジケータを前方または後方にドラッグする）ときはいつでも。
  • 選手情報：一意の識別子、プレイヤーの名前、およびプレイヤーのバージョン。
注意： たとえば、Kaltura Player、FlowPlayerなどのためにOoyala IQ JavaScript SDKの上に構築されている現在の標準アダプタは、これらの追加のメトリックをサポートしていません。

  • Ooyalaアセット（Backlotでホストされている）をプレイしているOoyalaプレーヤー。

サードパーティのプレーヤーを使用してBacklotアセットを再生するのか、サードパーティのCMSホストアセットを再生するのかを設定できます。 次の例は、Ooyala-JW Playerアダプタを使用してコンテンツタイプを指定する方法を示しています。
  • OOYALA_CONTENT： MEDIAID はOoyalaの埋め込みコードです。 Ooyalaアセットを指定するために必要な情報を入手するには、Backlot APIクエリを作成します。 / assets / <資産コード> / streams アセットの場合は、目的のエンコーディングの結果をコピーします。 以下の例は、 OOYALA_CONTENT JW Playerアダプタで指定します。
    <script type = "text / javascript"> var jwPlayer = jwplayer（ "video"）。setup（{ファイル： 'pathToYourOoyalaMp4VideoAsset'、メディアID： 'yourOoyalaEmbedCode'、タイプ： 'mp4'}）; var jwOoyalaReporter = Ooyala.Analytics.JWReporter（ "<pcode>"、jwPlayer、{contentType：Ooyala.Analytics.MediaContentType.OOYALA_CONTENT}）; </script>
  • EXTERNAL_CONTENT： MEDIAID これは、Ooyalaの埋め込みコードではありません（そして、Ooyala以外のCMSからの顧客定義のメディアIDです）。 以下の例は、 EXTERNAL_CONTENT JW Playerアダプタで指定します。
    <script type = "text / javascript"> var jwPlayer = jwplayer（ "video"）。setup（{ファイル： 'pathToYourVideoAsset'、メディアID： 'yourMediaID'}）; ;））; var jwOoyalaReporter = Ooyala.Analytics.JWReporter（ "<pcode>"、jwPlayer、{contentType：Ooyala.Analytics.MediaContentType.EXTERNAL_CONTENT}）; </script>

    • start_conversion_rate ようになりました video_conversion_rate.

    • unique_plays ようになりました uniq_plays_requested.

  • player_loads メトリックはUIに表示されなくなりました。 このメトリクスにはAPIを介してのみアクセスできます。
