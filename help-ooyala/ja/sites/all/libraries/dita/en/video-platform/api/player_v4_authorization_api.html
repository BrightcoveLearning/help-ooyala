リクエスト構文

Authorization APIへの要求の構文は以下のとおりです。

[GET] / sas / player_api / v2 / authorization / embed_code / {pcode} / {embed_codes}？{query_params}

例：

http://player.ooyala.com/sas/player_api/v2/authorization/embed_code/Z5Mm06XeZlcDlfU_1R9v_L2KwYG6/44azdwNDpSWUvfd8F30d55tXY0YH9njH?codecPriority=avc&device=html5&domain=debug.ooyala.com&supported_formats=m3u8,hds&player_type=video

レスポンスの要素

レスポンスで返された要素の中で：

  • debug_data データのデバッグにのみ使用されます。 いつでも変更できます。
  • 署名 応答には、応答が3rdパーティーによって改ざんされていないことを確認するために使用されます。
  • ストリーム これらのストリームにアクセスするためのbase64エンコードURLを含む、埋め込みコードが承認された場合は含まれます。 各ストリームは承認結果を返すことができます。 認証エラー.

視聴者ごとの同時ストリーム制限については、で説明されているように、追加のプロパティが返されます。 ビューアごとの同時ストリーム数の制限.
