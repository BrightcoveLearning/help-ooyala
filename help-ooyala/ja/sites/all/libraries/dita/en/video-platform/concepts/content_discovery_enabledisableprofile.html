プレーヤーの検出プロファイルの有効化/無効化

ディスカバリAPIを使用すると、以前に作成したディスカバリプロファイルを有効または無効にできます。 これは、 このプレイヤーの発見をオンにする Backlot UIでの設定（ Backlot UIを使用したプロファイルの作成と管理について).

検出プロファイルを有効にするには、次のようにします。 {"discovery_enabled"： "true"} 上記のPUTルートを持つリクエストボディのプロパティ。 
[PATCH] / v2 / players / e18ab1da1813483499554ea2d8e67fbd / discovery / profile {"discovery_enabled"： "true"}

可能なHTTPレスポンスコードとボディ：

  • 200：成功
  • 401：一般的な失敗です。 このコードは、既に有効になっている検出プロファイルを有効にしようとした場合、または既に無効になっている検出プロファイルを無効にしようとした場合に返されます。 この場合、レスポンスボディには、次のエラーメッセージを含めることができます。
    {"メッセージ"： "検出が有効になっていません"} {"メッセージ"： "検出は既に有効になっています"} {"メッセージ"： "\"有効にする\ "パラメータがないか無効、有効な値：true、false"} {"メッセージ" "："モジュールが無効です。プレーヤーにモジュールを追加/削除できません。 "} {"メッセージ "："プレーヤーが無効です。プレーヤーにモジュールを追加/削除できません。 "}
検出プロファイルを無効にするには、次のようにします。 {"discovery_enabled"： "false"} PUTルートを持つリクエストボディのプロパティ。 
[PATCH]/v2/players/e18ab1da1813483499554ea2d8e67fbd/discovery/profile{
   "discovery_enabled":"false"
}

上記の例の可能なレスポンスボディとHTTPレスポンスコードを参照してください。

注意： Ooyala Scratchpadでアカウントの認証情報を使用してコードサンプルを試してみてください。 スクラッチパッドを起動するには、にアクセスしてください。 Ooyala APIスクラッチパッド。 スクラッチパッドの使用方法については、 スクラッチパッドを使った練習の依頼.
