はじめに：Xcodeで環境を設定する

iOSおよびXcode用のOoyala Mobile SDKを使用して基本プロジェクトを設定する方法を学びます。

このチュートリアルを完了するには：
  • ダウンロード iOS用Ooyala Mobile SDK （バージョン4.3.0以降）そしてそれをあなたのシステムの都合の良いディレクトリに解凍してください。
  • システムにXcode 6（またはそれ以降）をインストールしてください。
  • Backlotアカウントにログインして、に移動します。 アカウント > 開発者。 ザ APIキー ピリオド（ "。"）の前の英数字と同じであり、指定する値です。 PCODE チュートリアルでは定数です。 たとえば、APIキーが xxxxxxxxxxxxxxxxxxxxxxxxxxxx.0000その後、最初の28文字（xxxxxxxxxxxxxxxxxxxxxxxxxxxx） のために PCODE。 詳しくは、 あなたのAPIクレデンシャル.
  • Backlotアカウントにログインし、以下の手順で 埋め込みコード:
    1. 案内する 管 理.
    2. 資産を選択してください。
    3. 選択します 埋め込む.
    4. を入手 コンテンツID.
  • プレーヤードメインのURLを持っています。
XcodeでOoyala Mobileアプリスケルトンアプリを設定する方法を学ぶには、このチュートリアルを完了してください。
  1. Xcodeを起動して選択 フィレット > 新しい > プロジェクト.
  2. 選択します シングルビューアプリケーション をクリックして 次. 新しいプロジェクトオプションダイアログボックスが表示されます。
  3. 次のプロジェクトオプションを設定して、[]をクリックします。 次.
    オプション 説明
    名前 プロジェクトの名前を入力してください。
    組織名 組織の名前を入力してください。
    組織識別子 組織の識別子を入力してください。
    Language このチュートリアルではObjective-Cを選択してください。
    デバイス このチュートリアルではユニバーサルを選択してください。
  4. プロジェクトの場所を選択してクリック 次. プロジェクトが作成されました。
  5. 選択します フィレット > にファイルを追加...次のようにします。
    1. OoyalaSDK-iOSフォルダを開き、 OoyalaSDK.framework.
    2. アクティブ化 保存先>必要に応じてアイテムをコピー チェックボックスをオンにしてクリック Add をクリックします。.
  6. 残りの必要なフレームワークをプロジェクトに追加します。
    1. ストリーミングを停止する場合は上部の 構築フェーズ をクリックします。
    2. 展開 バイナリとライブラリのリンク.
    3. プラス記号をクリックします（+）。
  7. 以下のフレームワークを選択してクリック Add をクリックします。:
    • AVFoundation.framework
    • AVKit.framework
    • CFNetwork.framework
    • CoreGraphics.framework
    • CoreMedia.framework
    • CoreText.framework
    • Foundation.framework
    • libxml2.dylib
    • libc ++。dylib
    • MediaAccessibility.framework
    • MediaPlayer.framework
    • QuartzCore.framework
    • Security.framework
    • SystemConfiguration.framework
    • UIKit.framework

    このステップが完了すると、リンカーライブラリのリストは次のスクリーンショットのようになります。

  16. ナビゲーションヘッダーの ラン. シミュレートされたiPhoneが埋め込みビデオプレーヤーと共に表示されます。 再生ボタンを選択して、で呼び出されたコンテンツを再生します。 埋め込みコード 値。
ノート:
  • ビデオの表示に問題がある場合は、シンジケーション設定がiPhoneおよびiPadデバイスを許可するように設定されていることを確認してください。 詳細については、Ooyala Backlot User GuideのPublishing Rulesセクションを参照してください。
  • ビデオがドロップアウトしてもオーディオが再生され続ける場合は、低帯域幅の接続がある可能性があります。 これにより、視聴者は引き続きコンテンツを体験できます。 これはAppleが設計した予想される動作です。
