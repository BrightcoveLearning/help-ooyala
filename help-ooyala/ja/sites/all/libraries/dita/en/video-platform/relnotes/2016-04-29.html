2016-04-29リリース

このリリースの注目すべき内容には、Ooyala IQをサードパーティのプレイヤーやRokuデバイスと統合するための新しいJSON APIが含まれています。

ソフトウェア機能リリース

サードパーティプレーヤー用の新しいOoyala IQ JSON API

JSON APIエンドポイントをサポートするようにOoyala IQバックエンドを更新しました。 JSONは、言語に依存しない軽量のオープンスタンダードデータ交換フォーマットで、読み書きが容易です。

このJSON APIエンドポイントは、既存のIQでのサードパーティ製プレーヤサポートを強化して、JavaScriptベースではないサードパーティ製プレーヤを含む、任意のデバイスまたはプレーヤからOoyala IQへのプレーヤイベントおよびデータの送信を可能にします。

新しいを使う ウーヤラIQ サードパーティ製プレイヤーのイベントをOoyala IQに伝達するためのJSON API。 このエンドポイントを使用してJSONデータをOoyala IQに送信することも、すぐに使用可能なSDKのいずれかを使用することもできます。 このリリースでは、Roku用のすぐに使えるSDKを提供しています。

Ooyala IQ JSON APIの詳細については、 JSON APIを使用したサードパーティ製プレイヤーとIQの統合.

Roku統合用の新しいOoyala IQ SDK

Roku用のOoyala IQ SDKを使用して、RokuチャンネルとOoyala IQを統合できるようになりました。 Roku用Ooyala IQ SDKは、JSONデータをOoyala IQ JSON APIエンドポイントに送信し、どのイベントをいつ送信するかについてのベストプラクティスに自動的に従い、すべての必須フィールドが存在することを確認する、すぐに使えるSDKです。 。

Roku用Ooyala IQ SDKは、Roku OS v7 +をサポートしています。

Ooyala IQ SDK for Rokuを使用している場合は、Rokuの[デバイス]パネルにRokuがOSとして表示されます。 Ooyala IQビジネスインテリジェンス インタフェース。

Rokuプレーヤーもプレーヤーパネルに表示されます。

Roku用Ooyala IQ SDKの詳細については、 Roku統合のためのOoyala IQ SDK。 Roku APIドキュメントのOoyala IQ SDKについては、を参照してください。 Ooyala IQ RokuライブラリAPIドキュメント.

Ooyala Ingestion Service（OIS）の機能強化

Ooyala Ingestion Service（OIS）v2.5は、ビデオとメタデータをBacklotにアップロードするための大きな前進です。 OIS v2.5を使用すると、ユーザーはFTP、Aspera、またはMRSSを使用してファイルをBacklotに取り込むことが容易になります。

OIS v2.5の新機能

  • さまざまなグローバル取り込みエンドポイント。 OIS v2.5は次の取り込みエンドポイントを使用します。transfer.ooyala.com）、米国（transfer-us.ooyala.com）、 EU （transfer.ooyala-eu.com）、およびAPAC（transfer-apac.ooyala.com） メインエンドポイントは、ユーザーを最も近い地域に自動的にリダイレクトします。 地域固有のエンドポイントに直接アクセスすることもできます。
  • アップロード専用のユーザーアカウント。 Backlot GUI（[アカウント]> [ユーザー管理]）で、[アップロードのみ]権限を持つユーザーを自分のアカウントに追加できるようになりました（を参照）。 ユーザーの管理） FTPまたはAsperaクライアントを使用している場合は、このユーザーに関連付けられている電子メールとパスワードを使用して、transfer.ooyala.comで新しい取り込みエンドポイントにログインできます。
  • マニフェスト摂取の促進 マニフェストベースの取り込みを有効にするプロセスが合理化され、その結果、有効化要求のターンアラウンドが速くなりました。 あなたの要求は自動的にXMLとCSV両方のマニフェストのサポートを含みます（あなたはもうどちらか一方だけを要求することはありません）。
  • FTPまたはAsperaを介した取り込み アップロード専用ユーザーを作成することで、FTPまたはAsperaの特別な資格情報を取得するためにOoyalaに連絡する必要がなくなりました。 ファイルは処理後すぐに削除されるようになりました。 取り込みワークフロー全体を通過しなかった（たとえば、マニフェストのアップロードを待っている）送信済みファイルは、7日後にサーバーから削除されます。
  • MRSSを使用したリモート資産の更新 MRSSフィードに新しいメタデータ、サムネイル、クローズドキャプションファイルを追加して、Backlotのリモートアセットを更新できるようになりました。
  • CSVを使用してリモートアセットを作成する CSV取り込みを使用してリモート資産を作成できるようになりました。 これにより、XMLとMRSSの取り込みを使用したリモートアセット作成に対する既存のサポートに3番目のオプションが追加されます。
OIS v2.5の確認済みの制限事項
  • 不適合ワークフロー OIS 2.5の場合、マニフェストベースの取り込みワークフローは、OoyalaのDTD標準に準拠する必要があります。 Backlotは不適合ワークフローを拒否しますが、検証エラーをまだユーザーに通知しません。
  • 検証を取り込みます。 取り込まれたファイルを検証するためのmd1ハッシュアルゴリズムのOIS v5使用は、OIS v2.5ではまだ利用できません。
