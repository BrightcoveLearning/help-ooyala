クエリ文字列パラメータの報告

これはのためのすべてのクエリ文字列パラメータの完全なリファレンス詳細です。 v3アナリティクス レポートAPI.

すべてのパラメータ名は大文字と小文字が区別され、小文字です。

クエリ文字列全体をURLエンコードする必要があります。 パラメーター 制限 そして ページ 正しい合計カウント値を取得するには、API呼び出しで使用する必要があります。

レポートGETの一般的な構文

ルートとクエリ文字列の基本的な構文は次のとおりです。 読みやすくするために、単一行の要求は複数の行に分割されています。 

[GET] / v3 / analytics / reports /？
            report_type=type
            ＆大きさ=大きさ
            ＆メトリクス=メトリクス
            ＆フィルター=filter_type == 'filter_value'
            ＆START_DATE=date
            ＆終了日=日付＆other_parms
            ＆api_key = your_api_key
  • 必須 クエリ文字列パラメータ（太字で表示）は report_type, START_DATE、そしてapi_key。
  • 次元が指定されていない場合は、すべての次元にわたる合計値が返されます。
  • 最大1月の時間範囲では、無制限のデータ行を使用して、一度に最大3ディメンションで照会できます。 1か月を超える時間範囲では、最大2行のデータを使用して、一度に最大10,000ディメンションで照会できます。
  • メトリックが指定されていない場合は、すべてのメトリックが返されます。
注意： APIキーはクエリパラメータとして、またはヘッダX-API-KEYを介して渡すことができます。
注意： 現時点では、 report_type is パフォーマンス.
注意： 1ディメンションのみを使用できます（URL）のための URL クエリ

並べ替えの基準となるコンマ区切りのメトリックのリスト。 複数の指標の場合は、指標がクエリに配置されている順序で指標によって並べ替えます。 （メトリックがあると仮定して）任意の数のソートメトリックを明示的に使用できますが、デフォルトのソートには2つのメトリックの制限があります。

time_segmentとデータの持続性について

時間範囲は、要求を満たすために自動的に最小時間範囲に拡張されます。 たとえば、今日だけの開始日と終了日を指定しますが、 time_segment of 週間 指定された開始/終了日を今週全体（プロバイダのタイムゾーン内）に拡大します。 指定がない場合 time_segment指定したディメンションの集計合計が返されます。 どちらもない場合 time_segment 指定されたディメンションも、時間範囲の総計も返されません。
