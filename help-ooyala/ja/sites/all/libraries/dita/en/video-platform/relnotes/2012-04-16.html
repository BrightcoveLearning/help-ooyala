2012-04-16リリース

アップデートには、新しいBacklot SDK、新しいモバイルSDK、5.1チャンネルオーディオの正しい処理、ネガティブタイムスタンプのサポート、複数の場所にオーディオアセットを保存するための修正、およびオーディオのみのHLSストリームを作成するための修正が含まれます。

リリース項目は以下のとおりです。

  • 新しいBacklot SDK - 5つのプログラミング言語（Java、PHP、Python、Ruby、およびC＃）のBacklot SDKがあります。
  • 新しいMobile SDK - Android、iOS、およびAdobe AIR SDKが利用可能になりました。
  • 5.1チャンネルオーディオが正しく処理されるようになりました - 以前は、5.1チャンネルオーディオが正しく処理されませんでした。 これで、5.1チャンネルのビデオファイルとオーディオファイルをアップロードできます。これらのファイルは自動的にステレオに変換されます。 これは自動的な変更であり、あなたの側では何もする必要はありません。
  • ネガティブタイムスタンプのサポート - 以前は、ネガティブタイムスタンプを持つネガティブビデオ（通常はiOSデバイス）はトランスコード中に拒否されていました。 現在、タイムスタンプがマイナスの動画をサポートしています。 これは自動的な変更であり、あなたの側では何もする必要はありません。
  • オーディオアセットを複数の場所に保存するための修正 - Ooyala以外のストレージを使用していた顧客にとって、オーディオのみのアセットが正しく処理されなかった。 現在、音声のみのアセットがすべての顧客にサポートされています。 これは自動的な変更であり、あなたの側では何もする必要はありません。
  • オーディオのみのHLSストリームを作成するための修正 - オーディオトラックのないビデオアセットがHLSストリームで失敗することがありました。 これらの資産は正しく処理されるようになりました。 これは自動的な変更であり、あなたの側では何もする必要はありません。
