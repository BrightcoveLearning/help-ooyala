Adobe Pass統合リファレンス

Adobe Passは現在Ooyala Player V4と統合されていません。 Adobe Passの統合に関連するプレーヤーパラメータ、API、およびイベント。

プレイヤーパラメータ

パラメーター 説明
staticResourceId Adobe Pass Serverに渡されるプレーヤーパラメータ。 BacklotラベルのresourceIdと一致させます。
showLoginAtStart 起動時にログインを表示するかどうか。

TRUE - エンドユーザーがまだ認証されていない場合は、自動的にMVPDを表示してログインします。

FALSE（デフォルト） - ログインにMPVDを表示しません。

APIリファレンス

OO.Pass API
API呼び出し 説明
OO.Pass.login（） 認証とログインを確認してください。 すでにログインしている場合は、OO.EVENTS.ADOBE_PASS_AUTH_STATUSを聞くことができます。それ以外の場合は、ログインダイアログが表示されます。
OO.Pass.logout（） 現在のユーザーをログアウトしようとしました。
OO.Pass.checkAuthentication（） 認証されたらフルチェックを行います。 現在の認証ステータスでOO.EVENTS.ADOBE_PASS_AUTH_STATUSを聞くことができます。
OO.Pass.isAuthenticated（） 認証された場合はtrueを返します。
OO.Pass.selectProvider（イベント） useProviderDialog（）を参照してください。 この関数を使用してプロバイダを解析し、Adobe PassにMVPDの選択を伝えることができます。
OO.Pass.getSelectedProvider（） 現在ログインしているプロバイダを返します。
OO.Pass.useProviderDialog（ダイアログ） カスタムMVPDダイアログコールバック関数を使用してください。 サンプルWebページを参照してください。
OO.Pass.checkAuthorization（resourceId） エンドユーザーが認証されている場合は、指定されたresourceIdに対して承認を得ているかどうかを確認します。

成功：OO.EVENTS.ADOBE_PASS_TOKEN_FETCHEDとsetTokenを発行します（Adobe Pass内で特定）。

失敗：OO.ERROR.ADOBE_PASS_TOKENとtokenRequestFailedを発行します（Adobe Pass内で固有）。
OO.Pass.getAuthorization（resourceId） 提供されたresourceIdの許可を得ようとしました。 エンドユーザーが認証されていない場合は、最初にログインを試みます。

成功：OO.EVENTS.ADOBE_PASS_TOKEN_FETCHEDとsetTokenを発行します（Adobe Pass内で特定）。

失敗：OO.ERROR.ADOBE_PASS_TOKENとtokenRequestFailedを発行します（Adobe Pass内で固有）。
OO.Pass.isEntitlementLoaded（） Access Enablerがロードされ初期化されている場合はtrueを返します。

OO.PASS CALLBACK API

API呼び出し 説明
OO.Pass.onSetToken（func） Adobe PassがsetTokenイベントを送出したときにコールバックするために、関数func（event、resId、token）の形式で関数を受け取ります。
OO.Pass.onTokenRequestFailed（func） Adobe PassがtokenRequestFailedイベントを送出したときにコールバックする関数func（event、stat、errorCode）の形式の関数を使用します。
OO.Pass.onSetAuthenticationStatus（func） Adobe PassがsetAuthenticationStatusイベントを送出したときにコールバックするために、関数func（event、stat、errorCode）の形式の関数を呼び出します。
OO.Pass.onEntitlementLoaded（func） Adobe PassがentitlementLoadedイベントを送出したときにコールバックするために、関数func（event、stat、errorCode）の形式の関数を呼び出します。

イベント紹介

イベント 説明
OO.EVENTS.ADOBE_PASS_AUTH_STATUS 誰かが認証を確認したとき。
OO.EVENTS.ADOBE_PASS_TOKEN_FETCHED 認証に成功したトークンが取得されたとき。
OO.EVENTS.ERROR エラーが発生したとき Adobe Passには、一部のAdobe Passエラーに対してリスナーに渡される追加のパラメーターがあります。

注意： MVPDからのより詳細なエラーは、params.passDetailsにあります。

最も一般的なAdobe Passイベント

次のリストには、最も一般的なAdobe Passイベントが含まれています。

  • setToken
  • tokenRequestFailed
  • setAuthenticationStatus
  • entitlementLoaded

これらのイベントについて詳しくは、Adobe Passの資料を参照してください。
