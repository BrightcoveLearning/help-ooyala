資産のタイプと/ v2 / assetsとの相関関係

/ v2 / assets あらゆる種類の資産との取引をルーティングします。

注意： Backlot REST APIコマンドについて詳しくは、以下を参照してください。 バックスロットAPIリファレンス.

When you create a new asset using the Backlot REST API, the asset_type property in the request body can specify any of the following values.

  • ビデオ
  • 広告：広告は単なる動画であり、広告では バックスロットUI.
  • オーディオ
  • remote_asset
  • youtube：YouTube向けのビデオ
  • live_stream (deprecated)
  • live_audio (deprecated)
  • channel (deprecated)
  • channel_set (deprecated)

で資産を作成した後 [POST] / v2 / assets （に記載されているように 資産の作成：POST), in general you can update it in a variety of ways with [パッチ] / v2 / assets。 一般化されたプロセスを見てください。 資産情報の編集：パッチ.

加えて、 / v2 / assets route has qualifiers for working with assets in general or specific types of assets, such as associating labels, publishing rules, ad sets, and more. Exact use of these qualifiers is detailed in other sections.
