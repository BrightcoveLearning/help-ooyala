v3 Analytics APIの相違点（廃止予定）

これらの間のプログラミングインタフェースの主な違いは次のとおりです。 v2アナリティクス とv3 Analytics（ウーヤラIQ).

警告： v2 Analytics APIは非推奨になりました。 見る ウーヤラIQアナリティクス 新しいv3 Analytics APIの詳細については、
  1. APIの主な違いは v2アナリティクス そして v3アナリティクス つまり v3 以下の理由により、より豊富な調査が可能です。 多次元分析。 以下に説明するように、これは構文にある種の影響を及ぼします。
  2. v2アナリティクス APIは、経路自体に直接多くのオプションやフィルターを位置的に表現しているため、経路の解析が困難になることがよくあります。 また、で v2 ルート上で要求できる次元は1つだけです。 の中に v3アナリティクス ただし、API クエリ文字列パラメータ（名前/値ペア） より明確にするために使用され、最大3次元を要求に含めることができます。 たとえば、パフォーマンスレポートの場合：
    • In v2アナリティクス: / v2 / analytics / reports /レベル/パフォーマンス/合計/日付範囲
    • In v3アナリティクス: / v3 / analytics / reports /？report_type = performance＆start_date =開始日
  3. In v3アナリティクスHTTP GET仕様の230文字の制限を超えるクエリパラメータを持つクエリでは、POSTリクエストを使用してください。 一部のブラウザおよびhttpクライアントは、230文字を超える文字をサポートしている場合がありますが、HTTP GET仕様に違反するクエリについては正式なサポートは提供されません。
  4. 一般に v2アナリティクスたとえば、データをフィルタ処理する日付範囲をルートの最後に指定することができますが、これは混乱を招きます。 代わりに v3アナリティクスには、Live モジュールで提供された START_DATE クエリ文字列パラメータと（必要に応じて） 終了日 パラメータ。
  5. In v2アナリティクス、 breakdown_by パラメータを指定すると、レスポンスは日、週、または月ごとに値を表示します。 に v3アナリティクス 直接の同等物はありません。 すべての値が返されます。 で返される時間セグメントを制限することができます。 time_segment パラメータ。
  6. v3アナリティクス に相当するものがない機能 v2: filters =, メトリックス=, 寸法=、および他の クエリ文字列パラメータ.
  7. データキューブのサイズを管理するために、 市、タグ、 そして URL 寸法
  8. APIレポート 次の表は、の一般的な構文をまとめたものです。 v2アナリティクス レポートタイプ v3アナリティクス 一般的な同等またはほぼ同等。 これは、すべての構文上の可能性を徹底的に扱うことを目的としているのではなく、違いの一般的な見方です。

    注意： エンゲージメント、シェアリング、そして独自のレポートが統合され、パフォーマンスレポートに含まれています。
