Create an Integrated Receiver Application

Use the following steps to create an integrated receiver application.
注意： If you want to use a receiver as is (that will have Ooyala branding and playerId), go to https://github.com/ooyala/chromecast-sample-receiver, and clone or download the project,.
  1. Host the content of the プロジェクト (receiver.html and images folder) on your server.
  2. Register your receiver HTML page in the Google Cast SDK developer console at Google Cast SDK Developer Console.
  3. Retrieve the receiver application ID from the Google Cast SDK developer console.
  4. 受信側アプリケーションIDを送信側アプリケーションコードに入れてください（ iOS用のSenderアプリケーションを構築する or Android用の送信者アプリケーションを構築する for how to add this to the sender application).

イベントリスニング

You can customize the custom receiver UI to react to player events by listening to the events published by the Player V4 player (see プレーヤV4のイベントモデル) or by modifying the CSS (see CSSを使用したプレーヤV4外観のカスタマイズ). This customization can occur in the onCreate function or after the Player V4 script has been loaded.

For example, you can display a splash screen when the content finishes. The user interface that you customize in your customized receiver should abide by the design checklist that Google has in place.

The event handling is located at the following path: src/app/player.js. This module has a method named as ‘onCreateHandlers’ that contains the event listeners for all the player event used.

エラー処理

For general Ooyala Player V4 errors, see JavaScriptを使用したPlayer V4のエラー処理. Since the receiver uses the same player V4 used in the web version, the error handling needs to be set by listening to the error events triggered by the player API.
