カスタムメタデータAPI（廃止予定）

カスタムメタデータAPIを使用すると、任意の名前と値のペアを任意のアセットに関連付けることができます。 メタデータは、署名付きのRESTスタイルのGET呼び出しを使用して操作されます。 http://api.ooyala.com/partner/set_metadata.

注意： このソフトウェアは非推奨です。 最新版を使用してください。

必須パラメータ

  • pcode - プロバイダコード
  • expires - エポックからの経過秒数での照会有効期限（00：00：00 1 / 1 / 1970 GMT）
  • embedCode - メタデータを追加または削除するための埋め込みコード
  • signature - リクエストのデジタル署名 署名は、署名生成規則とともにSHA-256を使用して計算する必要があります。

オプションのパラメータ

  • delete - 削除する名前のリスト これらの名前は、ヌル文字（ "\ 0"、 "％00"としてURIエンコード）で区切る必要があります。
  • name = value pairs - 埋め込みコードに関連付ける一連のname = valueメタデータペア。

制限

  • 資産ごとに最大100の名前/値ペアを設定できます。
  • 名前は最大128文字の長さ、値は最大2048文字の長さにすることができます。

応答は次の構造を持つXMLです

<？xml version = "1.0" encoding = "UTF-8"？>> <result code = "success"> ok </result>

リクエストが成功すると、 "code"は "success"になります。 要求が失敗した場合、 "code"は失敗の理由を示すHTTPステータスコードになります。

注意として、すべてのキーと値は適切にURIエスケープする必要があります。

例：

http://api.ooyala.com/partner/set_metadata?pcode=<pcode>&signature=<signature>&expires=<expiration>&embedCode=<embedCode>&director=Francis%20Ford%20Coppola&actor=Marlon%20Brando&delete=writer%00copyright

このクエリは、 "director"に "Francis Ford Coppola"の値を、 "actor"に "Marlon Brando"の値を割り当て、 "writer"と "copyright"という名前を削除します。
