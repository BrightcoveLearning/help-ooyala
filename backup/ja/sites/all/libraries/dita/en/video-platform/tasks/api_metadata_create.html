カスタムメタデータの追加

資産には最大100の名前/値ペアを追加できます。

注意： Backlot REST APIコマンドについて詳しくは、以下を参照してください。 バックスロットAPIリファレンス.
名前は128文字、値は2,048文字に制限されています。

カスタムメタデータをアセットに追加するには：

  1. 現在のメタデータを置き換えずにメタデータを追加するには、PATCHを / v2 / assets ルート。 次の例では、directorとcopyrightのカスタムメタデータを JxbzdkMjqBEsO 資産。 
    [PATCH] / v2 / assets / JxbzdkMjqBEsO / metadata {"監督"： "ピータージャクソン"、 "著作権"： "2002"}

    バックスロット 次のような応答を返します。

    {"監督"： "ピータージャクソン"、 "著作権"： "2002"、 "カテゴリ"： "アドベンチャー"、 "custom_id"： "54678934"}
    注意： Ooyala Scratchpadでアカウントの認証情報を使用してコードサンプルを試してみてください。 スクラッチパッドを起動するには、にアクセスしてください。 Ooyala APIスクラッチパッド。 スクラッチパッドの使用方法については、 スクラッチパッドを使った練習の依頼.
  2. 現在のメタデータを完全に置き換えるメタデータを追加するには、PUTを / v2 / assets ルート。 次の例は、のカスタムメタデータを置き換えます。 JxbzdkMjqBEsO 資産。 
    [PUT] / v2 / assets / JxbzdkMjqBEsO / metadata {"監督"： "ピータージャクソンではない"、}

    バックスロット 次のような応答を返します。

    {"監督"： "本当にピータージャクソンではない"、}
    注意： Ooyala Scratchpadでアカウントの認証情報を使用してコードサンプルを試してみてください。 スクラッチパッドを起動するには、にアクセスしてください。 Ooyala APIスクラッチパッド。 スクラッチパッドの使用方法については、 スクラッチパッドを使った練習の依頼.
  3. 現在のカスタムメタデータを取得するには、いつでもGETを実行できます。
