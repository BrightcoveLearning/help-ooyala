RESTの概要

内のすべてのリソース バックスロットAPI 独自のURLを持っています。 リソースのURLにリクエストを送信し、リクエストで標準のHTTPメソッドを使用することで、それらのリソースを表示、変更、削除できます。

リクエストを行うときは、任意のHTTPライブラリまたはツールを使用できます。 の バックスロットAPI 次のHTTPメソッドをサポートしています。各メソッドには、次に説明する標準化された機能があります。

  • GET：リソースを見る
  • POST：新しいリソースを作成する
  • DELETE：リソースを削除します
  • PUT：既存のリソースを置き換えます。 リソースを作成する
  • PATCH：既存のリソースを更新または修正する
注意：

すべての要求はHTTPSを使用して行われる必要があります。

すべての経路が上記のすべての要求をサポートするわけではありません。

一般に、POSTリクエストはそのリソースへのPATCHと同じパラメータを受け取ります。

新しいリソースを作成するときは、POSTリクエストを使用します。 リソースの要件に応じて、リクエストの本文にオブジェクトのJSON表現を含めることによってその設定を構成します。 次の例では、名前だけを指定してラベルを作成します。 

[POST] / v2 / labels {"名前"： "レジャー"}

名前を設定しただけですが、 バックスロット デフォルトを使用して残りの設定を自動的に構成します。

要求が成功すると、 バックスロットAPI 次のような応答を返します。

{"名前"： "レジャー"、 "ID"： "9d765ce4b3884f8"、 "フルネーム"： "/レジャー"、 "parent_id"：null}

GET要求を発行することによって、いつでもこの情報を入手できます。 たとえば、次のリクエストはラベル付きのラベルの設定を返します。 9d765ce4b3884f8 IDを指定します。

[GET]/v2/labels/9d765ce4b3884f8

この情報を使用して、PATCH要求を使用してリソースを変更できます。 たとえば、次のリクエストはラベルの名前をに変更します。 レクリエーション:

[PATCH] / v2 / labels / 9d765ce4b3884f8 {"名前"： "レクリエーション"}

要求が成功すると、 バックスロットAPI 次のような応答を返します。

{"名前"： "レクリエーション"、 "ID"： "9d765ce4b3884f8"、 "フルネーム"： "/ Recreation"、 "parent_id"：null}
