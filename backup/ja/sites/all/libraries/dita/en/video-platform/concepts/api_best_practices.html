ベストプラクティス

Ooyalaは、Webサイト上で豊富でユニークなエクスペリエンスを作成できるようにする一連のAPIを提供します。 従うべきベストプラクティスはここにあります。

APIキーのセキュリティ

あなたのOoyala APIキー、シークレット、そしてプロバイダーコードを安全かつ安全に保ってください。 アプリケーションにキーやコードを直接埋め込むことは避けてください。 安全なプロセスだけがあなたの秘密と鍵を読むことを許可しなさい。

Brightcoveテクニカルサポートまたは他の担当者はセキュリティキーを要求しません。

フロントエンドサーバーのキャッシュまたはデータベース

  • 顧客に迅速な対応を提供するには、サイトとのユーザーの対話中にBacklot APIを何度も呼び出さないように、Webサイトにキャッシュまたはデータベースを設定します。 詳細については、 Webプログラミングのためのアドバイス.

  • さらに、Ooyalaの高性能APIエンドポイント（cdn-api.ooyala.com）を利用してください。 大量のWebサイトからの繰り返し要求をキャッシュします。 コンテンツの種類に応じて、リクエストの有効期限を次の1時間、4時間、8時間、または1日に丸めます。 たとえば、データが頻繁に変わる場合は、次の時間に切り上げます。 データが頻繁に変更されない場合は、次の日に切り上げてください。 詳細については、 高性能APIエンドポイント

エラー処理

コードでエラーを処理し、問題が発生した場合にWebサイトでコンテンツを適切に表示できることを確認してください。

レート制限管理

レート制限要求は、レート制限がリセットされるまでの秒数を返します。 指定した秒数だけ待機してから再試行するようにコードを構成してください。 詳細については、 APIレート制限.
