フィルター

Here are the complete reference details for filters for the v3アナリティクス レポートAPI.

The filter names are essentially the same as dimension names. Filters are specified with the filters = query string parameter. You can further refine the results with boolean operators. Surround filter values with single quotations (e.g. filters=asset=='asset_id'). Filter values are case-sensitive.

For examples of how to use filters, see 使用できるフィルタ.

Date Range Limitation

注意： For up to 2 filters you can set a date range of up to 1 year (366 days). For 3 filters you can set a date range of up to 1 month (31 days).

HTTP GET仕様の230文字の制限を超えるクエリパラメータを持つクエリの場合は、 POSTリクエスト.

レポートGETの一般的な構文

ルートとクエリ文字列の基本的な構文は次のとおりです。 読みやすくするために、単一行の要求は複数の行に分割されています。 

[GET] / v3 / analytics / reports /？
            report_type=type
            ＆大きさ=大きさ
            ＆メトリクス=メトリクス
            ＆フィルター=filter_type == 'filter_value'
            ＆START_DATE=date
            ＆終了日=日付＆other_parms
            ＆api_key = your_api_key
  • 必須 クエリ文字列パラメータ（太字で表示）は report_type, START_DATE、そしてapi_key。
  • 次元が指定されていない場合は、すべての次元にわたる合計値が返されます。
  • 最大1月の時間範囲では、無制限のデータ行を使用して、一度に最大3ディメンションで照会できます。 1か月を超える時間範囲では、最大2行のデータを使用して、一度に最大10,000ディメンションで照会できます。
  • メトリックが指定されていない場合は、すべてのメトリックが返されます。
注意： APIキーはクエリパラメータとして、またはヘッダX-API-KEYを介して渡すことができます。
注意： 現時点では、 report_type is パフォーマンス.
注意： 1ディメンションのみを使用できます（URL）のための URL クエリ

ラベル

Filter data by label using the label id.

Ooyala IQでラベルを操作するときは、次の点に注意してください。
  • ラベルと関連付けられている動画間の相関関係は、事前集計または事前集計（ロールアップ）されませんが、APIリクエスト時に解決されます。
  • Ooyalaシステムはラベルの親階層に基づいて集約します。 たとえば、2つのラベル、親 "Cycle World"とその子ラベル "Sport Bikes"を想像してください。 「Sport Bikes」という子供のラベルが付いたビデオは、「Cycle World」という親のラベルにロールアップされます。 ラベルのデータをリクエストした場合、返される指標はそのラベルとその子ラベルのものです。
注意： This filter is not valid with the URL dimension. You can only use ドメイン そして URL filters with the URL 寸法。
 You can use any defined label in Backlot. You can filter by multiple labels at a time. However, you can not mix the label filter with the asset type フィルタ。

asset_type

 Filter data by asset type. The asset type indicates whether the asset is a live channel ("live") or video-on-demand ("vod"). An asset will be considered of type ライブ only if it was created using Ooyala's live platform. Live assets that got converted to video-on-demand before the end date of your reporting period will be counted as VOD only, otherwise they will be counted as live.
注意： This filter is not valid with the URL dimension. You can only use ドメイン そして URL filters with the URL dimension. Also, this filter cannot be combined with the label filter.

The asset type can be "live" or "vod". Note that the asset type is case-sensitive and should be entered in all lowercase letters.

Boolean Operators; (AND) and | (OR)

The formal syntax of a filter is shown below. 
filters = フィルタ  [boolOp フィルタ]*
シンボル 拡張
フィルタ dimension-filter or composite-filter
dimension-filter 寸法 op 値

どこ：

  • 寸法 is a dimension name listed in レンディション or a custom dimension name.
  • op is == (equal to) or != (not equal to).
  • 値 is a single-quoted dimension value, like os!='android' (data with operating system Android).
composite-filter ( フィルタ [boolOp フィルタ ]*)

どこ：

  • フィルタ is defined above.
  • boolOp is ; (AND) or | (OR). The boolean operator and second filter are optional and can be repeated without limit.
Grouping with Parentheses

Use parentheses to logically group components of the filter value. For an example, see 使用できるフィルタ.
