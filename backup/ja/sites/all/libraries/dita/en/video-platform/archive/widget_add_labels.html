ラベルを追加する（廃止予定）

ユーザーがアップロードした動画を安全に整理するために、静的動画を宣言できます。 ラベル または正規表現 ダイナミック パラメータ。

これらのパラメータで渡されたラベルは署名されているので、HTMLインジェクション攻撃はこれらのラベルを変更できません。 A ラベル ウィジェットを介してアップロードされたすべてのビデオに適用されますが、dynamicは、を使用して動的に入力されたラベルの正規表現ルールです addDynamicLabel

注意： <id> 部分には、任意の英数字を含めることができます。 ラベル[<id>] そして 動的[<id>]この文字列はあなたの便宜のためです。

例：

ラベル[<id>]（文字列）

ファイルをアップロードしている顧客を識別するラベルを追加するには、次のようにします。

label [by_user] = / user / u12345 label [by_classification] = / user /種別/ trusted

この例では、アップロードされたビデオはすべてu12345（ユーザーのサブラベル）、および信頼される（ユーザーのサブラベルである分類のサブラベル）というラベルが付けられます。

使用されるラベル名は、先頭のスラッシュから始めて完全修飾名である必要があり、英数字を含むことができます。 ラベルが存在しない場合は作成されます。

動的[<id>]（文字列）

ユーザーがコンテンツに追加できるようにすることができる特定のラベルを設定するには： 

dynamic [by_genre] = ^ / by_genre /（アクション|コメディ|ドラマ）$

この例では、ユーザーは3つのジャンル（アクション、コメディー、ドラマ）のうちの1つに限定されているため、ラベル付けに一貫性があります。 の中に 動的[<id>]パラメータ、行の先頭に '^'文字、行の末尾に '$'文字を使用すると、値のセットが制限されます。 それらを省略すると、より広い一致が可能になります。 例えば、 

動的[カスタム] = ^ /カスタム/

ユーザーが名前で始まる任意のラベルを作成および追加できるようにします。 /カスタム/ 終了文字がないためです。 すべて ダイナミック 値は正規表現です。

注意： パートナーがパラメータを広く設定しすぎると、動的ラベルがコードインジェクション攻撃の危険にさらされる可能性があります。
