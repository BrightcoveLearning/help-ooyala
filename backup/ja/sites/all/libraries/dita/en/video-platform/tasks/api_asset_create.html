資産の作成：POST

資産を作成するには、 POST / v2 / assets リクエストの本文にプロパティがあります。

注意： Backlot REST APIコマンドについて詳しくは、以下を参照してください。 バックスロットAPIリファレンス.
注意： 以下のステップは、一般資産の作成に関するものです。
  • リモートアセットについては、 リモートアセット.
  • ライブストリームについては、 ライブストリーム（非推奨）.
  • チャンネルセットについては、 チャンネルとチャンネルセット（廃止予定）.
  • YouTubeの動画については、 YouTubeビデオをアップロードする
使用 POST メソッドと / v2 / assets リクエストボディのルートとプロパティ。
注意： それらのアセットを作成することに加えて、ビデオおよびオーディオファイルもアップロードする必要があります。 見る ビデオまたはオーディオアセットをアップロードする：4つのステップ.
注意： POST本体のプロパティはアセットの種類によって異なります。 
[POST] / v2 / assets {"asset_type"： "ビデオ"、 "期間"：0、 "名前"： "マイビデオ"、 "preview_image_url"：null、 "created_at"： "2011-07-22-18：54： 19 + 00：00 "、" embed_code "：" Y1dTdvMjq9QtOMGrP-H59OIgiZ6-_Mrl "、"時間制限 "：null、"更新された_at "：" 2011-XNXXXXXXXXXXXXXXXXXXXXXXXXXXXXX "。 、 "説明"：null、 "ステータス"： "アップロード中"、}

バックスロット 次のような応答を返します。 に注意してください 埋め込みコード 財産。 これが資産への鍵です。 asset_id また、一般的な資産の更新、削除、または作業に必要です。
