資産の管理

Backlotでは、アセットはビデオ、オーディオ、ライブストリームなどのさまざまな種類のコンテンツの一部です。

注意： Backlot REST APIコマンドについて詳しくは、以下を参照してください。 バックスロットAPIリファレンス.

これは一意の識別子によって識別されます。 埋め込みコード （API内） コンテンツID （Backlot UIの場合）、または asset_id （ドキュメント内）

  • 資産のタイプと/ v2 / assetsとの相関関係
    / v2 / assets あらゆる種類の資産との取引をルーティングします。
  • 資産の作成：POST
    資産を作成するには、 POST / v2 / assets リクエストの本文にプロパティがあります。
  • 資産情報の表示：GET
    資産の情報を表示するには、 GET / v2 / assets /asset_id。 すべての資産に関する情報を表示するには、 asset_id.
  • 資産情報の編集：パッチ
    資産の情報を変更するには、 パッチ/ v2 /アセット/asset_id.
  • 資産の削除：DELETE
    資産を永久に削除するには、 DELETE / v2 / assets /asset_id.
  • すべての資産に共通のタスク：PUTまたはPATCHと修飾子
    一部の機能は、その種類にかかわらず、すべての資産に共通です。
  • 動画
    に動画をアップロードしたとき Ooyala, Ooyala アカウントに指定された設定を使用して自動的にトランスコードされ、プレーヤで動画を再生できるようになります。
  • ライブストリーム（非推奨）
    ライブストリーミングは、リアルタイムでのビデオコンテンツの配信です。
  • リモートアセット
    リモートアセットは、Backlotの外部でホストされるコンテンツの一部です。 コンテンツをアップロード、トランスコーディング、およびBacklotに保存する代わりに、必要に応じてBacklotがコンテンツを見つけることができるように、そのURLやその他の説明情報をアップロードするだけです。
  • YouTubeのビデオ
    YouTubeビデオは、YouTubeによってホストされ、Ooyalaプレーヤーを通じて再生されたビデオです。
  • チャンネルとチャンネルセット（廃止予定）
    チャンネルとチャンネルセットを使用すると、コンテンツを顧客に配信するためにグループ化できます。
  • 代替ルート
    Backlot APIには、同じタスクを実行するためのいくつかの方法があります。
