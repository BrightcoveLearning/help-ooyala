2014-07-02リリース

注目すべき変更には、iOSおよびAndroidモバイルSDKのv.2.5.0のリリース、およびプレーヤAPIのdestroy（）関数の新しいオプションパラメータ（HTML5のみ）が含まれます。

ソフトウェア機能リリース

以下は新しいソフトウェアリリース項目です。

iOS SDK v.2.5.0

iOS用Ooyala Mobile SDKのバージョン2.5.0がリリースされました。 バージョン2.5.0はクローズドキャプションとAirplayサポートに関するバグ修正を含みます。 バージョン2.5.0にはVAST広告機能への変更も含まれており、アプリケーションをAdSupport.frameworkとリンクする必要があります。
  • モバイルアプリでLiveRailを介した広告の再生を可能にするために、VAST URLに[LR_DEVICEID] 'マクロ'拡張を実装しました。 これにはAdSupport.frameworkが必要です。
  • ログをカスタマイズするためのOODebugModeを追加しました。
  • HLS再生用のビットレート変更通知を追加しました。
iOS SDK v.2.5.0のフルリリースノート

Android SDK v.2.5.0

Android用Ooyala Mobile SDKのバージョン2.5.0がリリースされました。 バージョン2.5.0には、ほとんどのOoyala SDKクラスのネームスペースに対する変更が含まれています。 アプリケーションのインポートを更新する必要があるかもしれません。
  • Data Model、Player、VAST、およびUIクラスを独自のパッケージに移動しました。 クラスは、簡単に消費できるパッケージを作成し、混乱を減らすために分離されました。 クラスが移動された場所を確認するには、SDK zipファイル内のAPIドキュメントを参照してください。 これはあなたの輸入のいくつかへの変更を必要とするかもしれません。
  • モバイルアプリでLiveRailを介した広告の再生を可能にするために、VAST広告のLR_DEVICEIDの代わりにbeginFetchingAdvertisingIdを追加しました。 これにはGoogle Play Services Libraryが必要です。
  • サードパーティの認証トークンにsetCustomDrmDataを追加しました。
  • 一貫性のあるロギングとランタイムアサーションのためのDebugModeクラスを追加しました。
  • フルスクリーンコントローラにクローズドキャプションを追加しました。
  • Playready DRMのサポートを追加しました。
Android SDK v.2.5.0のフルリリースノート

フリーホイール広告セットフィールド "fw_video_asset_id"

広告設定フィールド「fw_video_asset_id」は、ページレベルで設定することも、映画のメタデータ（Backlot UIまたはBacklot APIを介して設定）によって設定することもできます。 この新機能により、シンジケートコンテンツおよびプレーヤでのターゲット広告が可能になります。 どちらも設定されていない場合、このフィールドの値はexternalIDまたはembed_code（プレーヤーの埋め込みに使用されている方）のいずれかにフォールバックします。
