バックスロットREST API

プレーヤーのクロム（デザイン要素）をカスタマイズするには、Backlot REST APIを使用してください。 これには、フォント、色、スクラバーバーなどのUI要素が含まれます。

重要： このセクションのトピックは、 Ooyala APIドキュメント のページをご覧ください。
  • Backlot REST APIについて
    Backlot REST APIへようこそ。
  • HTTP応答コード
    要求結果に応じて、Backlot REST APIはさまざまなHTTP応答コードを返します。
  • 資産
    / v2 / assets あらゆる種類の資産との取引をルーティングします。
  • ラベル
    外部発行ターゲットに追加すると、ラベルは外部に発行するビデオ、チャンネル、およびチャンネルセットを指定します。 さらに、ラベルは、ビデオライブラリの整理、ビデオの検索、およびターゲット分析の取得にも役立ちます。
  • プレイヤー（Backlot REST API）
  • 広告セット
    広告セットは、再生中のアセットの広告エクスペリエンスを制御する広告設定です。
  • 公開ルール
    公開ルールは、いつどこでコンテンツを再生できるかを指定します。 公開ルールを使用すると、コンテンツを特定の地域に限定し、特定のドメインにのみ埋め込まれるようにして、特定の時間にのみ表示できるようにすることができます。
  • 設定可能なDRM
    デジタル著作権管理（DRM）を使用すると、視聴者がDRMポリシーを介して高価値コンテンツをどのように使用してアクセスするかを制御できます。 DRMポリシーはアカウントまたは資産レベルで設定できます。
  • シンジケーション
    シンジケーションは、コンテンツのリストと、コンテンツをシンジケートしたい宛先から構成されます。
  • 国と場所のコード
    これらの国コードはさまざまな分野で使用されています。 Ooyala 機能。
