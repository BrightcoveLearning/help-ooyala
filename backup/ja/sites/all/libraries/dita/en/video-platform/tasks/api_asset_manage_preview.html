プレビュー画像の管理

にアセットを追加した後 バックスロット視聴者が再生ボタンをクリックする前に表示する画像を指定できます。 画像はGIF、PNG、またはJPG形式にすることができます。

注意： Backlot REST APIコマンドについて詳しくは、以下を参照してください。 バックスロットAPIリファレンス.

ご希望のサムネイルまたはプレビュー画像を選択するには、次の選択肢があります。

  • 自動生成された画像の1つを使用してください。 ディスカッションを見る 自動生成プレビュー画像（サムネイル）.
  • あなた自身のカスタム画像をアップロードしてください。 Backlotは1つのプレビュー画像のアップロードのみをサポートしています。 それ以降のアップロードは単に前回のアップロードを置き換えます。
  • 目的の画像がある場所のURLを指定します。

プレビュー画像を管理するには：

  1. トランスコーディングによって自動的に生成されたプレビュー画像を表示するには、 / v2 / assets ルート、アセットID、および / generated_preview_images 修飾子 次の例では、アセット用にプレビューで自動生成された画像を返します。 しぶき: 
    [GET] / v2 / assets / JxbzdkMjqB / generated_preview_images

    バックスロット 次のような応答を返します。

    [{"url"： "http://ak.c.ooyala.com/JxbzdkMjqB/Ut_HKthATH4"、 "時間"：0}、{"url"： "http://ak.c.ooyala.com/JxbzdkMjqB / q78XH7vLqJJ "、"時間 "：4512}]
    注意： Ooyala Scratchpadでアカウントの認証情報を使用してコードサンプルを試してみてください。 スクラッチパッドを起動するには、にアクセスしてください。 Ooyala APIスクラッチパッド。 スクラッチパッドの使用方法については、 スクラッチパッドを使った練習の依頼.
  2. カスタムプレビュー画像を作成するには、ファイルをアップロードするか、画像がある場所のURLを指定します。
    • 注意： A POST requestはプレビュー画像ファイルを現在のファイルのセットに追加します。 A PUT requestは現在のプレビュー画像ファイルを置き換えます。

      カスタムプレビュー画像をアップロードするには、 POST、 / v2 / assets リクエストボディのroute、アセットID、および画像データ。

      次の例では、プレビュー画像をアセットに追加します。 Y1dTdvMjq9QtOM:

      [POST] / v2 / assets / Y1dTdvMjq9QtOM / preview_image_files <file_contents>

      バックスロット 次のような応答を返します。

      {"幅"：800、 "高さ"：600、 "URL"： "http://ak.c.ooyala.com/Y1dTdvMjq9QtOM/promo136839624"、 "id"： "promo136839624"}

      追加する画像のサイズごとにこの手順を繰り返します。

      注意： Ooyala Scratchpadでアカウントの認証情報を使用してコードサンプルを試してみてください。 スクラッチパッドを起動するには、にアクセスしてください。 Ooyala APIスクラッチパッド。 スクラッチパッドの使用方法については、 スクラッチパッドを使った練習の依頼.

    • カスタムプレビュー画像をURLで指定するには、 PUT、 / v2 / assets リクエストボディのルート、アセットID、サイズと場所の詳細。

      次の例では、複数のサイズのプレビュー画像をアセットに追加します。 Y1dTdvMjq9QtOM:

      [POST] / v2 / assets / Y1dTdvMjq9QtOM / preview_image_urls [{"幅"：1920、 "高さ"：1080、 "url"： "http://www.mysite.com/my_big_image.png"}、{"幅" ：1280、 "高さ"：720、 "URL"： "http://www.mysite.com/my_medium_image.png"}、{"幅"：640、 "高さ"：480、 "URL"： "http： //www.mysite.com/my_small_image.png "}]

      バックスロット 次のような応答を返します。

      [{"url"： "http://www.mysite.com/my_big_image.png"、 "幅"：1920、 "高さ"：1080}、{"url"： "http://www.mysite.com /my_medium_image.png "、"幅 "：1280、"高さ "：720}、{" url "：" http://www.mysite.com/my_small_image.png "、"幅 "：640、"高さ "： 480}]
      注意： Ooyala Scratchpadでアカウントの認証情報を使用してコードサンプルを試してみてください。 スクラッチパッドを起動するには、にアクセスしてください。 Ooyala APIスクラッチパッド。 スクラッチパッドの使用方法については、 スクラッチパッドを使った練習の依頼.
  3. プライマリプレビュー画像のアセットには、生成された画像またはアップロードされた/カスタムの画像を使用できます。

    • 自動的に生成された画像の1つを選択するには、 type プロパティ、指定 生成された そして時間。 次の例では、1次プレビュー・イメージは15秒のマークで生成されたイメージに設定されています。

      [PUT] / v2 / assets / Y1dTdvMjq9QtOM / primary_preview_image {"タイプ"： "生成"、 "時間"：15}
      注意： Ooyala Scratchpadでアカウントの認証情報を使用してコードサンプルを試してみてください。 スクラッチパッドを起動するには、にアクセスしてください。 Ooyala APIスクラッチパッド。 スクラッチパッドの使用方法については、 スクラッチパッドを使った練習の依頼.

    • アップロードしたファイルを使用するには、リクエストボディに type プロパティ、指定 upload_file。 次の例では、プライマリプレビュー画像はアップロードされた画像の1つを使用するように設定されています。 埋め込み時に、Ooyalaは自動的に埋め込みサイズに最も近い画像を選択します。

      [PUT] / v2 / assets / Y1dTdvMjq9QtOM / primary_preview_image {"type"： "upload_file"}
      注意： Ooyala Scratchpadでアカウントの認証情報を使用してコードサンプルを試してみてください。 スクラッチパッドを起動するには、にアクセスしてください。 Ooyala APIスクラッチパッド。 スクラッチパッドの使用方法については、 スクラッチパッドを使った練習の依頼.

    • プライマリプレビューイメージに関する情報を取得要求に含めるには、次のように指定します。 include = primary_preview_image。 次の例では、プライマリプレビューイメージがレスポンスで返されます。

      [GET] / v2 / assets / Y1dTdvMjq9QtOM？include = primary_preview_image
      注意： Ooyala Scratchpadでアカウントの認証情報を使用してコードサンプルを試してみてください。 スクラッチパッドを起動するには、にアクセスしてください。 Ooyala APIスクラッチパッド。 スクラッチパッドの使用方法については、 スクラッチパッドを使った練習の依頼.

    • リモートURLのファイルを使用するには、リクエストボディの type プロパティ、指定 remote_url。 次の例では、プライマリプレビュー画像はいずれかのリモートURLを使用するように設定されています。 埋め込み時に、Ooyalaは自動的に埋め込みサイズに最も近い画像を選択します。

      [PUT] / v2 / assets / Y1dTdvMjq9QtOM / primary_preview_image {"type"： "remote_url"}
      注意：

      FlashベースのOoyalaプレーヤーでリモートURLタイプのプレビュー画像を表示するには、Flash必須のクロスドメインセキュリティファイルを必ず設定してください。 crossdomain.xml * .ooyala.comドメインと、画像にアクセスする必要があるその他のドメイン（Webサイト）のエントリを含めること。 そうでないと、Ooyalaプレーヤーはリモートでホストされている画像を表示できません。

      ファイルはで記述されています HTTPストリーミング用のcrossdomain.xmlファイルの設定.

      追加する必要があるエントリは次のようになります。 さらに、画像にアクセスする必要があるすべてのドメイン（Webサイト）のエントリを含める必要があります。
      。 。 。 <allow-access-from domain = "*。ooyala.com" /> 。 。
