トラック

トラックはウーヤラスケジュールの基本です。

トラックは、1つ以上の直線的にスケジュールされたトラックセグメントで構成されています。
トラックは次のような構造をしています。
注意： live_stream_asset_id プロパティはオプションです。
{"id"： "8ffbfcda77fb40b0af6425ae24752254"、 "名前"： "P12N"、 "live_stream_asset_id"： "0yMTVoMzpqSB_nCyAJwvaAXSePizNX"

トラックを作成する

[POST] / v2 / tracks {"名前"： "新しいトラック"}

すべてのトラックを見る

[GET] / v2 / tracks

1曲見る

[GET] / v2 / track /track_id

トラックを削除する

[削除] / v2 / track /track_id

トラックを修正する

[PATCH] / v2 / track /track_id{"名前"： "新規トラック"}

ルート属性

次の表は、ルートを通じて表現できるすべての属性を示しています。

ルート属性 説明
track_id

トラックのID

タイプ：文字列

デフォルト：なし

Example: /v2/tracks/8ffbfcda77fb40b0af6425ae24752254

プロパティ

次の表は、トラックに関連付けることができるすべてのプロパティについて説明しています。

プロパティ 説明 必須？
live_stream_asset_id

ライブストリームまたはリモートライブストリームの識別子。

タイプ：文字列

例： "0yMTVoMzpqSB_nCyAJwvaAXSePizm6I3"

いいえ
名

このトラックのわかりやすい名前。

タイプ：文字列

有効値：256文字の制限。 改行（char 10）またはキャリッジリターン（char 13）を除き、制御文字を含めることはできません。

例： "私のお気に入りトラック"

はい

例

[POST] / v2 / tracks {"名前"： "私のお気に入りのトラック"、}
