2012-11-16リリース

アップデートには、Ooyala Discoveryの改良、コンテンツの保護と再生の改良と追加、Backlotの新しいMANAGEページ、コンテンツの置き換え、APIを介した広告セットの作成と更新、アップローダーウィジェットの強化、Live Assetシンジケーションの追加が含まれます。

リリース項目は以下のとおりです。

  • ウーヤラディスカバリー

    • アルゴリズムのパフォーマンスの向上 - ディスカバリ画面のクリック率（CTR）を向上させ、ユーザーエクスペリエンスとAPIを通じて推奨品質/完了率を向上させるパフォーマンスの向上を提供します。 これは自動的な変更であり、あなたの側では何もする必要はありません。
    • VOD + Live Recommendations - ライブプログラミングOoyala Discoveryの推奨を提供します。 Live to VODとLive to VODの推奨を生成できます。 これはOoyala Discoveryのお客様向けの自動アップデートです。
    • Twitterカード - Twitterカードを使用して、Twitter内でコンテンツを直接再生する簡単な方法を提供します。 私たちは組み込みプレーヤーをホストし、あなたの開発努力を最小にして、必要なTwitterメタデータを生成するための単一のAPIリクエストを提供します。 詳細については、 Twitterプレーヤーカードとの統合.
    • クロスデバイス連続再生 - 視聴者は、あるデバイスでビデオの視聴を開始し、同じデバイスまたは別のデバイスで後でそのビデオを視聴し続けることができます。 コンテンツは中断したところから自動的に再生を再開します。 詳細については、 クロスデバイスレジューム（XDR） ドキュメンテーション。
    • Electronic Programming Guide（限定版） - Backlot V2 API拡張機能を介してマルチトラックのリニアプログラミングを提供することで、独自のマルチチャンネルTV局を作成することができます。 Backlot搭載の電子プログラミングガイドは、簡単なXMLベースのアダプタを介してあらゆるリニア放送スケジューリングシステムと同期させることができ、同時視聴者数を含む、ライブ番組ごとの視聴者数メトリックスを詳述するAudience Insightsレポートを受信できます。 詳細については、営業、カスタマーサクセスマネージャー、またはテクニカルサポートにお問い合わせください。
    • BacklotによるOoyalaディスカバリー構成 - Backlot UIを通してOoyalaディスカバリーの完全構成を提供します。 コンテンツフィルタルールを指定し、コンテンツを宣伝し、順次コンテンツ用のルールを生成し、画面の一時停止と終了を有効にすることができます。 さらに、このリリースでは、ディスカバリ設定を個々のプレーヤプロファイルごとに簡単に関連付けることができ、異なるクライアントプレーヤプロファイル間で異なるディスカバリ動作のセットを使用できます。 この機能は、30、2013の11月にすべてのOoyala Discoveryのお客様にご利用いただけるようになります。

  • コンテンツ保護と再生

    • Google Widevine DRM - GoogleのWidevine DRMを使用したコンテンツのパッケージ化とクライアント統合を提供します。 Widevineは、当社のiOSおよびAndroid SDKでサポートされています。 さらに、Samsung、LG、PanasonicなどのConnected TVに配信されるコンテンツを保護することもできます。 詳細については、営業、カスタマーサクセスマネージャー、またはテクニカルサポートにお問い合わせください。
    • 強化されたポリシー管理 - Adob​​e AccessやGoogle Widevineなど、さまざまなDRMシステムに対する複数のDRMポリシーのサポート。
    • Flash Playerのパフォーマンスの向上 - 読み込み時間を最大で10％短縮する改善を行いました。 これは自動的な変更であり、あなたの側では何もする必要はありません。
    • Windows 8アプリケーションのサポート - V8 HTML3プレーヤーを使用して、デスクトップとタブレット（Microsoft Surfaceなど）のWindows 5アプリを作成できるようになりました。 詳細については、 Microsoft Windows用HTML5 Player V3アプリ8（非推奨） ドキュメンテーション。

  • バックスロットと分析

    • 新しいHTMLベースの[管理]タブ - すべての顧客に対応したBacklotは、FlashではなくHTML 5およびAjaxで実行されます。 主な利点は、キャッシュの高速化、パフォーマンスの向上、チャネル管理の向上、および暗室の置き換えです。 これは自動的な変更であり、あなたの側では何もする必要はありません。
    • コンテンツの置き換え - 動画アセットをいつでも置き換えることができます。 コンテンツ置換は、同じビデオアセットのコンテンツを絶え間なく急速に変化させている大規模なニュースおよびメディア組織にとって最も価値があります。 これにより、貴重なエンゲージメント指標やシンジケーションの機会を失うことなく、一日を通してストーリーを新鮮なコンテンツで更新できます。 詳細については、 ../tasks/api_asset_replacement.html.
    • APIを使用した広告セットの作成と管理 - GoogleのAPIを使用して広告と広告セットを作成および管理できます。 Backlot UIによる設定に加えて、広告ルールと広告戦略をプログラムで管理および自動化できるようになりました。 詳細については、 広告セット.
    • V2 API上に構築された強化されたアップローダウィジェット - 顧客が所有するWebサイトまたはパッケージ化されたアップローダライブラリを使用するアプリケーションを介して、コンテンツをBacklotにシームレスにアップロードできます。
    • ライブアセットシンジケート - VODアセットと同様に、Backlot内でプロビジョニングされたライブストリームを外部の宛先にシンジケートすることができます。 Live Asset Syndicationは、11月にすべてのお客様が利用できるようになります21、2013。
