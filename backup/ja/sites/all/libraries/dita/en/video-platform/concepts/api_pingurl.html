API ping URL

API Ping URLは、ファイルトランスコードプロセスが成功したかどうかにかかわらず、ファイルトランスコードプロセスが終了するたびに呼び出されます。

Backlot UIで、「アカウント」タブの「開発者」サブタブに移動して、API ping URLを設定できます。 API Ping URLは、次のものを含むGETリクエストを受け取るためのものです。 埋め込みコード APIでアップロードされた映画のトランスコーディングが完了するたびに（コンテンツIDまたはビデオまたは他のアセットに割り当てられたアセットID）。

トランスコーディングが失敗した場合でも、通知を受け取ります。 ただし、この場合は埋め込みコードは受け取りません。

コンテンツが再処理または置き換えられない限り、通知は通常1ムービーにつき1回だけ発生します。 複数のURLをコンマで区切って指定できます。 最大文字長は4,096バイトです。 入力したURLは、入力した時点で到達可能でなければなりません。 システムはそれが有効なURLかどうかを確認します。

たとえば、次のようにAPI Ping URLを設定した場合、処理が成功すると、Backlotはクエリ文字列内の埋め込みコードを使用してそれを実行します。

   http://foobar.com?foo=bar
   [GET] http://foobar.com?foo=bar&embedCode=ReturnedEmbedCodeHere

一般的な名前のAPI Ping URLの例：

http://example.com/ooyalapostback
