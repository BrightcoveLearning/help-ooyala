全般

Q：Ooyala IQとは何ですか？

A: Ooyala IQは、信頼性が高く、堅牢でスケーラブルなビッグデータアーキテクチャに基づいて構築されたOoyalaの新しいビデオ分析製品で、リアルタイム分析および多次元分析機能を提供して、ビデオ消費およびエンゲージメントパターンの完全な360程度のビューを提供します。 これは既存のOoyala Analytics製品へのメジャーアップグレードであり、追加料金なしですべてのOoyalaのお客様にご利用いただけます。

Q：以前のAnalytics製品とどう違うのですか？

A: 主な違いは、Ooyala IQがレポートの事前設定を必要とせずに多次元分析を可能にすることです。 一度に複数のデータセットを見ることができます。 たとえば、コンテンツプログラミングチームは、ラベルごととデバイスタイプごとにフィルタ処理されたデータを同時に表示して、モバイルデバイスとデスクトップで消費されるビデオグループ（スポーツコンテンツなど）の消費パターンを理解することができます。 以前の業界をリードする分析製品では、一度に1セットのデータしか調査できませんでした。

この強力な機能は、新しいv3 Analytics APIとOoyala IQ UIのビジネスインテリジェンスページに反映されています。 ビジネスインテリジェンスページでは、資産またはラベル、デバイス（プラットフォームとOSおよびブラウザ）、地域（国、地域、州/県、DMA）、プレーヤー、およびトラフィックソースに関するメトリックを1つの統合コンソールに表示し、あらゆる側面の調査を集中管理します。ビデオの消費量を1ページにまとめ、必要に応じて詳細にドリルダウンできるようにします。 製品の違いの詳細については、下の表を参照してください。

Q：気をつけるべき新機能は何ですか？またその理由は？

A: 次の機能が新しくなりました。
  1. HTML上に構築された新しく設計されたユーザーインターフェース：

    新しいHTML UIは、Flashプラグインへの依存を取り除き、さまざまなデバイス（iPadなど）上のIQにアクセスする機能を提供し、パフォーマンスを向上させます。

  2. 新しい多次元分析機能

    Ooyalaの新しい多次元クエリ機能を使用すると、デバイスタイプ（モバイル、タブレット、デスクトップ、セットトップボックスなど）やOS /ブラウザ、地理（国、地域、州など）など、複数の次元にわたって分析を実行できます。州、DMA）、ドメイン、プレイヤー、資産、またはラベル。 これは、「先月中にTablet - iOSで消費されるビデオのサブセットに対して、米国とカナダからのユニーク視聴者は何人いますか」など、より複雑なビジネス上の質問に対処するのに役立ちます。

  3. 新しい動画の詳細ページ：

    新しい[Video Details]ページでは、資産のパフォーマンスに関する全体的で詳細なビューが表示されます。 一意性およびエンゲージメントデータとは別に、主要な資産メタデータ、資産の主要業績評価指標、および各ディメンション別に分類された資産消費量を表示できるようになりました。 アセットをクリックするだけで、ダッシュボードページまたはビジネスインテリジェンスページから直接[ビデオの詳細]ページにアクセスできます。 この増加したアクセシビリティは、特定の資産の詳細なパフォーマンスとともに、その資産の消費とエンゲージメントのパターンをすばやく識別することを可能にします。

  4. トップ100ビデオのリアルタイム統計：

    Ooyala IQの新しいリアルタイムダッシュボードには、上位の100動画とその視聴データがリアルタイムで表示されますが、古い分析製品には上位3つの動画しか表示されません。

  5. 改善された報告：
    • メトリックはより正確に定義され計算されます。
    • 地理情報とデバイスレポートは、より正確なテクノロジによって提供されます。
    • ユニークユーザーを数えるためのアルゴリズムが改良され、ユニークユーザーの報告がより長い期間にわたって行われるようになりました。
    • より堅牢でスケーラブルで高性能なビッグデータアーキテクチャ上に構築されています。
  6. 新しい測定基準と新しい次元

Q：Ooyala IQの新しいメトリクスとディメンションは何ですか？

A: 次の新しいメトリクスとディメンションが追加されました。
  • ビデオが始まります。 「Video Starts」は、実際のビデオコンテンツが再生を開始する時期を示します。 プレロール広告が再生される場合、「ビデオ開始」は、すべてのプレロール広告が完了して実際のビデオコンテンツの再生が開始されたときにのみカウントされます。 要求された再生回数（以前のv2分析では再生回数と呼ばれていました）とVideo Startsを比較することで、プレロール広告による視聴者の減少を見ることができます。
  • プレイヤーの次元 Playerディメンションは複数の方法で使用できます。 プレイヤーは以下のことを報告するために使用できます。
    1. シンジケーションパートナーの再生指標（追跡したい各シンジケートパートナーのプレーヤーを作成することによって）
    2. Webサイトのさまざまなセクションでのパフォーマンス（各セクションにプレーヤーを作成することによって）
    3. プレーヤの変更によるパフォーマンス（さまざまな広告ロード、Discoveryコンテンツ推奨ツールの有効化、単にプレーヤUIの変更など、プレーヤ構成の有無にかかわらずプレーヤを作成することによる）

Q：広告プロバイダから取得した数が、Ooyala Analyticsの再生数と表示数と一致しません。 なんでこんなことが起こっているの？

A: グーグルまたはあなたの広告プロバイダーからの統計は、ウーヤラによって提供された統計と直接の関係を持ちません。 基礎となるデータにばらつきがあり、これらの計算やデータポイントに関する業界全体の標準がないため、すべてのシステムが独自の方法で統計を計算するため、それらを直接比較することはできません。 これは、ことわざりんごとことわざオレンジとを比較するようなものです。 統計の説明については、広告プロバイダに直接連絡する必要があります。 さらに、WebページでGoogleアナリティクスタグを2回指定するなど、実装の誤りがあると、Ooyalaのデータと明らかに矛盾する可能性があります。 お客様のご厚意により、Ooyalaは広告プロバイダの統計をBacklotに直接表示します。 ただし、統計情報はOoyalaではなく広告プロバイダによって計算されます。

サードパーティ（Ooyala以外）のプレーヤーでOoyala IQを使用する

Q：Ooyala IQをサードパーティのプレイヤーと一緒に使用できますか？ 統合できるプレーヤーに制限はありますか？

A: このリリースでは、IQはあらゆるストリーミングデバイスおよびあらゆるプレーヤから分析を受け取ることができます。 WebベースのプラットフォームおよびJavaScriptをサポートするOS用に、Ooyala IQをサードパーティのプレーヤーと統合することができます。 Ooyala IQ JavaScript SDK。 JavaScriptをサポートしていないデバイスと統合することもできます。 Ooyala IQ JSON API。 Ooyala IQとRokuデバイスを統合する Ooyala IQ Roku SDK.

Q：Ooyala IQは、どのようなサードパーティ製のプレーヤプラットフォームで、すぐに使える統合で現在サポートしていますか？

A: Ooyala IQは現在、以下のプラットフォームおよびサードパーティ製プレイヤーのための独創的な統合（アダプタとSDK）をサポートしています。
  • 六（OSv7 +）
  • Xbox One
  • Brightcove Player
  • FlowPlayerプレーヤー
  • HTML5ビデオタグプレーヤー
  • JWプレーヤー
  • カルチュラ奏者
  • YouTubeプレーヤー
AppleTVのためのすぐに使える統合はもうすぐです。 Ooyalaの次期tvOS SDKを使用している場合、Ooyala IQはそのままでAppleTV分析をサポートします。 今後のOoyala tvOS SDKを使用しない場合（たとえば、Apple TV用に開発されたアプリケーションで、OoyalaのものではなくネイティブのtvOSを使用する場合）、Ooyala IQ Javascript JSDKに接続する必要があります。 Ooyala PSGは有料でこの統合を実行できます。

JavaScript対応プラットフォーム用のOoyala IQ JSON APIまたはJavaScript SDKと統合することで、Ooyala IQをあらゆるデバイスまたはプレーヤー（FireTV、Sony Playstation、その他のストリーミングデバイスなど）と統合することができます。

Q：Ooyalaプレイヤーとサードパーティプレイヤーを同じプロバイダーアカウントで使用できますか？

A: Ooyalaの資産の場合、1つのBacklotアカウントを使用してOoyalaとサードパーティのプレーヤーの統計情報を表示できるようになりました。 1つのBacklotアカウントを使用して、1つのIQダッシュボードに統合型混合プレーヤー分析を表示できます。 サードパーティの資産の場合は、別のプロバイダアカウントが必要になります。

Q：マスターアカウントのIQデータが両方の総計となるような、他のサードパーティのプレーヤー用のサブアカウントを作成できますか？

A: はい。 Rokuプレーヤーの資産（または資産のメタデータのみ）をホストするBacklotアカウントとJW Playerの資産（または資産のメタデータのみ）をホストするBacklotアカウントが1つある場合、1つのマスターアカウントと2つのサブアカウントを持つことができます。それらを制御します。

Q：Unique Usersの値はどのように計算されますか？

A: 一部のプロパティのOVPとしてOoyalaを既に使用している場合は、デバイスIDまたはサブアカウントプロパティと同等のものを使用します（サードパーティのOVPを使用している可能性があります）。 ユーザーが匿名（ログインしていない）の場合は、ユーザーに割り当てられた追跡IDを使用します。 ログインユーザーを表すユーザーUUIDを使用することはお勧めできません。

Q：Ooyala以外のプレーヤーでDiscoveryを使うことはできますか？

A: いいえ。Discoveryは現在、Ooyala以外のプレイヤーをサポートしていません。 これは、プレーヤー内のDiscoveryエクスペリエンスとDiscovery APIの両方に当てはまります。 複数のサイト/プロパティを管理していて、それらのサイトの1つ以上にOoyalaプレーヤーを使用している場合、Discoveryはそれらのサイトだけで使用できます。

Q：IQ UIのBIページにある "Player"ディメンションの場合、 "Player"ディメンションには何が表示されますか？

A: プレーヤー名がプレーヤーディメンションとして表示されます（たとえばJWプレーヤー）。

Q：アセットの詳細ページには、アセットを再生したり、セグメントの進行状況をリアルタイムでグラフで確認したりできる領域があります。 第三者（Ooyala以外）のプレイヤーでそのデータをキャプチャすることはできますか？

A: Asset Detailsページでは、Ooyalaプレイヤーのみをロードします。 そのページに第三者のプレーヤーを表示することはありません。 ただし、Ooyalaプレーヤーを使用してリモートアセットを再生する方法と同様に、アセットURLを取得している限り、Ooyalaプレーヤーを使用したビデオアセットの再生はサポートされています。 セグメントがリアルタイムで進行状況を監視しているのがわかります。 アセットURLを取り込まないと再生できません。

私は何をホストしなければならず、Ooyalaは何をホストしますか？

A: JavaScript SDKおよびJavaScript対応プレーヤー用のアダプタのために、OoyalaはJavaScriptライブラリをホストしています。 Roku用Ooyala IQ SDKおよびXbox用SDKの場合、OoyalaはGitHubでRokuおよびXbox用のOoyala IQ SDKをホストします（ここでライブラリをダウンロードできます）。 Roku用Ooyala IQ SDKおよびXbox用SDKは、RokuチャンネルパッケージおよびXboxアプリパッケージに不可欠なものである必要があります。これは、Webページ上のJavaScript SDKの場合のようにRokuボックスまたはXboxにその場でダウンロードできないためです。

Q：Ooyala以外のプレイヤーがアップデートされるとどうなりますか？ IQ SDKの実装方法を変更する必要がありますか？

A: プレーヤーが更新または変更された場合は、何も壊れていないことを確認するためにSDK統合をテストして検証する必要があります。

Ooyala IQ JSON API

Q：IQデータ取り込みAPIでJSONデータ形式を使用することの主な利点は何ですか？

A: JSONは言語に依存しないオープンスタンダードのデータフォーマットで、読み書きが簡単です。 これにより、IQは任意のデバイス/プレーヤーからプレーヤーイベントとデータを「受信」することができます。 さらに、JSONの属性と値のペアは、カスタムディメンション、カスタムイベントなどの追加データフィールドの取り込みをサポートするように簡単に拡張できます。

Q：Ooyala IQ JSON APIの使用を開始できるユーザーに制限はありますか？

A: いいえ、誰でもこのAPIを使用できます（既存のOoyala CMSのお客様、新しいOoyala CMSのお客様、スタンドアロンのお客様など）。

Q：JSON APIはいつPlayer V4に統合されますか？

A: より多くの顧客がPlayer V4を使い始めるにつれて、JSON APIは2016の後半でPlayer V4と統合されます。

Q：Ooyala IQ JSON APIドキュメントには、GeoInfo（国、地域、都道府県など）を設定できると記載されています。 これが設定されていない場合、Ooyalaはまだこの情報について報告しますか？ これが設定されている場合、これはOoyalaの標準的なジオレポートを上書きしますか？

A: はい、はい。 Ooyalaはipアドレスから地理情報を取得するためにQuovaを使用しますが、これが手動で設定されている場合、手動で設定された値はQuova値を上書きします。

Q：Ooyala IQ SDK for RokuではどのバージョンのRokuがサポートされていますか？

A: Roku用Ooyala IQ SDKはRoku OS v7 +をサポートします

注意： Roku用Ooyala IQ SDKは現在Roku Scene Graph APIをサポートしていません。

Q：Ooyala IQ SDK for XboxではどのバージョンのXboxがサポートされていますか？

A: Xbox用Ooyala IQ SDKはXbox Oneをサポートしています。

Q：Roku用Ooyala IQ SDKはイベントマッピングを担当しますか？

A: はい、RokuイベントループでIQクラスを適切に使用すれば、イベントマッピングは正しくなります。

Ooyala IQ JavaScript SDK

Q：IQ JavaScript SDKとプレーヤ固有のアダプタの違いは何ですか？

A: IQ JavaScript SDKは、Ooyala IQのバックエンドと対話するSDKです。 プレーヤ固有のアダプタは、サードパーティのプレーヤとIQ JavaScript SDKを橋渡しするアダプタです。

Q：標準のサードパーティ製プレーヤ（JavaScriptベース）用のOoyala製アダプタは、プレーヤAPIの変更とバージョンのアップグレードを継続的にサポートしますか？

A: はい、OoyalaはプレーヤAPIの更新をサポートし、アダプタは標準のサードパーティプレーヤへの変更をサポートするように改訂されます。

Q：アダプタを書くためにサードパーティのプレーヤーについて何を知る必要がありますか？

A: サードパーティ製のプレーヤー用のアダプタを作成するには、サードパーティ製のプレーヤーのイベントAPIまたはイベントメカニズムを理解する必要があります。 これにより、プレーヤーのイベントをOoyala IQ JavaScript SDKまたはJSON SDKにマッピングできます。

Q：Ooyala IQ JavaScript SDKのドキュメントには、動画の再生が要求されたときにはreportPlayRequestedメソッドを呼び出すべきであると書かれています。 私は遊びと再開を区別する必要がありますか？

A: はい、あなたはこれら2つのケースを区別する必要があります。 再生は、ユーザがプレーヤ上で最初に再生ボタンをクリックしたときであり、ビデオが一時停止状態になった後にユーザが再生をクリックしたときに履歴書がカウントされるべきである。

Q：同じプレーヤーで再生される複数のビデオを含むプレイリストがある場合はどうすればいいですか。 イベントは再カウントされますか？

A: プレーヤで再生される各ビデオに対してメソッドinitualizeMediaを呼び出す必要があります。 これにより、各ビデオについてイベントが確実にカウントされます。 同じプレーヤーを使用して複数のビデオを再生する場合でも、各ビデオは同じイベントライフサイクルを経て、すべてのイベントメソッド（initializeMedia、setMediaDuration、reportPlayRequestedなど）を呼び出す必要があります。

Q：Ooyala IQ JavaScript SDKの場合、reportPlayHeadUpdateをどのくらいの頻度で呼び出す必要がありますか？

A: あなたは頻繁にreportPlayHeadUpdateを呼び出すべきです（2-3秒ごとに）。

Q：ユーザーが次のボタンで別のビデオに切り替えた場合の流れはどうなりますか？ Ooyala IQ JavaScript SDKを使用している場合、メディアが完成しなかったこの使用例ではreportCompleteを呼び出す必要がありますか？

A: いいえ。メディアが終了しなかった場合は、reportCompleteを呼び出さないでください。 新しいビデオに対してinitializeMediaを呼び出し、新しいビデオに対して残りのイベントメソッド（initializeMedia、setMediaDuration、reportPlayRequestedなど）を呼び出す必要があります。

Q：どうやってサードパーティ製のプレーヤアダプタをテストすればいいですか？

A: Ooyalaの担当者に連絡して、CSMまたはテクニカルサポートに連絡してアダプタをテストしてもらいます。

顧客の移行

Q：Ooyala IQに移行したら、どのくらいの履歴データにアクセスできますか？

A: すべてのお客様にとって、1月の1から現在までの2014のデータが利用可能になります。

Q：もっと過去のデータにアクセスできますか？

A: vNNX Analytics UIのエクスポート機能が無効になっているため、1月の1より前のデータ、2014は使用できなくなりました。また、v2 Analytics APIは、6月以降廃止されました2。

Q：古いv2 Analytics APIは無効にされましたか？

A: はい。 2、30の6月以降、古いv2016 Analytics APIは非推奨になりました。

Q：Ooyala IQに移行した後、古いAnalytics UIにアクセスできますか？

A: いいえ、完全に有効にしてOoyala IQに移行すると、v2 Analytics UIにアクセスできなくなります。 Backlot UIに古い[Analytics]タブと古い[Now]タブが表示されなくなります。 代わりに、IQの[ダッシュボード]とIQの[ビジネスインテリジェンス]ページをサブタブとして含む、新しい[分析]タブが表示されます。

Q：Ooyala IQのデータと古いアナリティクスのデータに違いはありますか？

A: はい、あります。 これにはいくつかの理由があります。
  1. 多次元機能に対応するためのデータの再処理のために、データの集計は、古い分析製品とはわずかに異なる値になる可能性があります。 また、Ooyala IQでは新しいアーキテクチャを使用しているため、エラー処理が向上しています。
  2. Ooyala IQは、古い分析製品よりも正確な地理レポートサービスを使用しています。
  3. Ooyala IQは、以前の分析製品よりも正確なデバイスプロファイリングプロバイダーを使用しているため、デバイス間およびOS間でレポートデータが異なる場合があります。
  4. メトリクスはOoyala IQでより正確に定義されています。 計算における新しい定義またはバグ修正を伴うメトリクスには、プレイスルーおよびユニークユーザが含まれます。 これらのメトリクスの計算がどのように変更されたかの詳細な説明については、Ooyala IQユーザガイドの「データの潜在的な違い」というタイトルのセクションを参照してください。

Q：サポートされなくなる機能はありますか？

A: Ooyala IQから廃止予定のいくつかの項目があります。
  • 市区町村はサポートされなくなります。 国、地域、州/県、DMAは引き続きサポートされます。
  • ソーシャルシェアリング（UI） - ソーシャルシェアリングメトリクスはv3 Analytics APIを介してのみ利用可能になり、UIではサポートされなくなります。

Q：都市とカスタムタグを見ることができる必要があります。 どうやってそれらを受け取り続けることができますか？

A: 正式にOoyala IQに移行する前に、都市とカスタムタグにアクセスすることができます。 切り替えた後、そのデータは新製品では使用できなくなりますが、Ooyala IQへの移行が開始されてから約2ヶ月までは、古いv6 Analytics APIを介して引き続きアクセスできます。

Q：私のモバイルプレーヤーの名前の代わりに、多くの "不明な"プレーヤー名が表示されています。 これは何を意味するのでしょうか？

A: iOSおよびAndroid SDKのドメインを設定していない場合は、ooyala.comドメインに属するこれらのSDKの分析データが表示されます。

Q：Geographyディメンションで[Show by Country]をクリックしてリストビューを見ると、各国ごとの内訳に "none"と "unknown"のエントリが多数あります。 これは何を意味するのでしょうか？

A: [国別に表示]オプションで各国の第一レベルの管理部門を表示するには、[一覧表示]を選択して各国名の横にある小さな矢印をクリックします。 第1レベルの管理部門がその国で利用できない場合は、 "none"または "unknown"が表示されます。

Q：ダッシュボードのKPIバーを見ると、現在のユーザー数が、今日要求された再生数よりも多くなっています。 なんでこんなことが起こっているの？

A: ダッシュボードのKPIバーでは、現在のユーザー数が毎秒更新されます。 KPIバーの残りの部分（今日のユーザー、今日の再生要求、今日のビデオ開始など）のカウンターの待ち時間は約2-4時間です。 そのため、その日の総ユーザー数よりも同時ユーザー数が多いように見えることがあります。

Q：Analytics v3 APIに適用できるフィルタとディメンショングループの数に制限はありますか？

A: はい。 以下の制限があります。
  • 最大2年（1日）までの日付範囲に対して、最大366フィルタでクエリを作成できます。
  • 最大3月（1日）までの日付範囲に対して、最大31フィルターで照会を作成できます。
  • 結果を最大3次元でグループ化するクエリを作成できます。

Q：特定の資産の分析データを調べていて、日付範囲として特定の1日を選択しましたが、ビデオ開始数がセグメント監視グラフの開始数と一致しません。 なんでこんなことが起こっているの？

A: ビデオの長さに関係なく、各ビデオを40セグメントに分割します。 ユーザーが最初のセグメントを視聴すると、最初のセグメントの数が「視聴済み」として増加します。 これは、最初のセグメント全体を見た視聴者の数よりも、ビデオの開始数が多い場合があることを意味します。 一部のユーザーは、ビデオの最初の1 / 40番目が完了する前に中断することがあります。

Q：[ビジネスインテリジェンス]ページの指標はどのくらいの頻度で更新されますか？

A: ユーザーアクションによってページが更新されます。 別の操作をしないと、新しい情報を更新する必要がないため、ページは通常更新されません。 歴史的な見方です。 唯一の例外は、「今日」を含めるように時間範囲を選択した場合です。 それから、今日から来るデータを継続的に処理しているので、データはおよそ4時間後に異なります。

Q：Ooyala IQは、リアルタイム分析のためにサブアカウントからマスターアカウントへのデータの集約をサポートしていますか？

A: いいえ、IQは、非リアルタイム分析のためにサブアカウントからマスターアカウントへの情報の集約のみをサポートします。

Q：Ooyala IQは運用資産の指標を取得しますか？

  • 視聴したセグメント
