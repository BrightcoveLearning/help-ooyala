Ooyala Discovery REST APIリクエストの概要

Discovery REST APIを使用して、サービスのさまざまな側面を構成し、推奨事項を入手し、その他のユーザー関連のデータを入手します。

以下の表は、要求を要約し、それらを読み取り/書き込み機能別に分類したものです。 わかりやすくするために、ルートの正確な構文は表示されていません。リンクをクリックして詳細な説明を参照してください。

登録リクエスト 目的 読み書き
1 / v2 / discover / trending？discovery_profile = profile_id 最も人気のあるコンテンツを入手してください。 詳細
2 / v2 /発見/トレンド/運動量 最新のコンテンツ（最も勢いのある動画）を入手してください。 詳細
3 / v2 / discover / similar / assets, / v2 / discover / similar / externalassets, / v2 / discover / similar / live_assets 関連コンテンツを入手 詳細
4 / v2 / player /player_id/ discovery / profile 推奨事項を取得するための構成プロファイルを作成します。 書きます
5 / v2 /発見/フィードバック/印象, / v2 /発見/フィードバック/再生 ビデオの表示を記録します。 ビデオの再生を録画します。 書きます
6 / v2 /ディスカバー/編集 プロモーション用のコンテンツのバンドルを作成します。 書きます
読み取り要求（#1、＃2、および#3）は、Ooyala Discoveryの「主力」です。 これらはすべて、発見の機械学習アルゴリズムからの基本的な推奨事項のセットを返します。 この基本セットは、相互に排他的な2つの方法で作成できます。
  • 希望する動作に従って結果を設定する定義済みのディスカバリプロファイルを使用します（ ディスカバリープロファイル） これは推奨されるメカニズムであり、上記の要求#1に示されている使用法です。 ここに示されている要求の例は「人気のある」コンテンツ推奨のためのものですが、プロファイルは任意の読み取り要求で使用できます。
  • APIリクエスト自体に "編集上の"クエリ文字列パラメータを使用する（「 ディスカバリー編集用照会ストリングのパラメーター).
書き込み要求（#4、＃5、および#6）は、1回限りの構成設定であることが多いため、使用頻度は少なくなります。
