Ooyala API V1 (Deprecated) to API V2 Migration

Ooyala V1 APIには限られた機能しかありません。 の Ooyala V2 APIを使用すると、リソースをより細かく制御でき、新しい機能がリリースされたときに更新が容易になります。

Ooyala V2 APIはRepresentational State Transfer（REST）の規則に従います。 各リソースには独自のURLがあり、独自の設定可能なパラメータセットがあり、標準のHTTPメソッドを通じて更新できます。 使用可能なHTTPリクエストメソッドは次のとおりです。

  • GET：リソースを見る
  • POST：新しいリソースを作成する
  • DELETE：リソースを削除します
  • PUT：既存のリソースを置き換えます
  • PATCH：既存のリソースを更新または修正する

ほとんどのHTTPライブラリまたはツールでは、要求を出すときにこれらのHTTPメソッドのいずれかを指定できます。

  • 属性更新API（廃止予定）
    属性更新APIはアセットAPIの一部になりました。 資産に対してGETを実行してそのデータを返すか、POSTを実行して属性設定を更新できます。
  • カスタムメタデータAPI（廃止予定）
    カスタムメタデータAPIは、アセットAPIの一部になりました。 これで、カスタムメタデータまたはアセットを表示、更新、置換、または削除できます。
  • ラベル管理API（廃止予定）
    新しいラベル管理APIには、Backlot API v1よりも多くの機能があり、それほど複雑ではありません。
  • プレーヤ管理API（廃止予定）
    古いプレーヤ管理APIでは、プレーヤに関する情報を一覧表示してそれをアセットに割り当てることができました。 新しいプレーヤAPIを使用すると、プレーヤの属性と設定を作成および変更するだけでなく、これらのタスクを実行できます。
  • チャネルAPI（廃止予定）
    古いチャンネルAPIには、チャンネルのコンテンツの一覧表示、チャンネルへのアセットの割り当て、またはチャンネルの作成を可能にする3つのモードがありました。 チャンネルAPIは、アセットAPIの一部になりました。これにより、チャンネルの作成、編集、削除、およびチャンネルラインナップの作成、編集、削除を行うことができます。
  • チャネルセットAPI（廃止予定）
    古いチャンネルセットAPIには、チャンネルセットの内容の一覧表示、チャンネルセットへのチャンネルの割り当て、または新しいチャンネルセットの作成を可能にする3つのモードがありました。 チャンネルセットAPIがアセットAPIの一部になりました。これにより、チャンネルセットの作成、編集、削除、およびチャンネルセットのラインナップの作成、編集、削除が可能になります。
  • クエリAPI（廃止予定）
    新旧のクエリAPIの違いはこのセクションの範囲を超えています。
  • サムネイルクエリAPI（廃止予定）
    サムネイルクエリAPIは、アセットAPIの一部になりました。
