Backlot REST APIを使用したプレーヤカスタムメタデータの管理

You can add your own name-value pairs to any player.

注意： Ooyalaプレーヤー用のREST APIコマンドについて詳しくは、以下を参照してください。 バックスロットAPIリファレンス.

Set/Replace Player Custom Metadata

[PUT] / v2 / players /player_id/metadata{  
   "name1「」value1「」name2「」value2「」name3「」value3",
   . 
 . 
 . 
 "name100「」value100"}

Modify Custom Metadata

[パッチ] / v2 / players /player_id/metadata{  
   "name1「」value1「」name2「」value2「」name3「」value3",
   . 
 . 
 . 
 "name100「」value100"}

Get Custom Metadata

[GET] / v2 / players /player_id/メタデータ

Delete All Custom Metadata for Player

[削除] / v2 / players /player_id/メタデータ

Delete A Single Custom Metadatum for Player

[パッチ] / v2 / players /player_id/metadata
{
   "名":"null"
}

ルート属性

次の表は、ルートを通じて表現できるすべての属性を示しています。

ルート属性 説明
player_id

The ID of the player.

タイプ：文字列

デフォルト：なし

Example: /players/IzNnllMjphu2XF3_UgPROoCi9B2BwtSg

例

This example creates custom metadata for the IzNnllMjphu2XF3_UgPROoCi9B2rtWs プレーヤー： 

[PUT]/v2/players/IzNnllMjphu2XF3_UgPROoCi9B2rtWs/metadata
{
   "skin": "branded"
}
