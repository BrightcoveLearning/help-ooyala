設定可能なDRM

デジタル著作権管理（DRM）を使用すると、視聴者がDRMポリシーを介して高価値コンテンツをどのように使用してアクセスするかを制御できます。 DRMポリシーはアカウントまたは資産レベルで設定できます。

注意： Backlot REST APIコマンドについて詳しくは、以下を参照してください。 バックスロットAPIリファレンス.

Ooyalaは、Apple FairPlay、Adobe Access、Widevine Modular、PlayreadyなどのDRMテクノロジをサポートしています。 Adobe AccessはFlashで使用されます。 Widevineはモバイル機器に使用されます。 自分のアカウント用に特別に設定されている場合は、Playreadyが使用されます。

注意： アカウントまたは個々の資産に割り当てることができる複数のDRMポリシーを設定できますが、Ooyalaはポリシーを設定する必要があります。 詳細については、カスタマーサクセスマネージャーまたはOoyalaサポートにお問い合わせください。
  • DRMポリシー
    特定のDRMポリシーまたは自分のアカウントに関連付けられているすべてのポリシーを表示できます。
  • アカウントとDRMポリシーを関連付ける
    デフォルトのDRMポリシーを自分のアカウントに関連付けることができます。 デフォルトポリシーは、特定のポリシーに関連付けられていないすべてのアセットに割り当てられます。
  • DRMポリシーとアセットの関連付け
    DRMポリシーを1つ以上の特定の資産に関連付けることができます。 DRMポリシーがアセットに関連付けられていない場合は、デフォルトのポリシーが使用されます。
