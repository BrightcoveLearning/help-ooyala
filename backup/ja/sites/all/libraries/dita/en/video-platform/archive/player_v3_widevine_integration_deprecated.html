Widevineコンテンツ保護（廃止予定）

注意： Widevine Classic DRMとのOoyala統合は推奨されておらず、無効にされる予定です。 詳細と代替案については、 OVPリリースノート.

Ooyalaは、オンデマンドおよびライブストリーミング形式で、高品質のコンテンツをオンラインで保護するためのWidevineのサポートを提供しています。 WidevineのマルチプラットフォームDRMは、あらゆるコンシューマデバイスでマルチメディアコンテンツの再生をライセンス、安全に配布、保護する機能を提供します。 コンテンツ所有者とデジタルメディアプロバイダは、Widevineのソリューションを使用して、収益を生み出すサービスが、あらゆるデバイス消費者が望むものに流れ続けるようにすることができます。

OoyalaとWidevineを使用してコンテンツを保護するための機能を提供するには、Widevineの機能と、iOS、Android、またはConnected TVデバイスでのWidevineの使用方法を理解する必要があります。 このドキュメントではこれを行う方法について説明します。

Widevineは、コンテンツを保護する機能を提供するために連携して機能する一連の包括的なコンテンツ保護機能の一部です。 これらの機能は次のとおりです。

  • Ooyala Secure Player Tokenなどのトークンベースのオプションによるユーザー認証。 この機能の設定と使い方については、を参照してください。 プレイヤーV3のOoyalaプレイヤートークン（廃止予定）.
  • Widevineなどのメカニズムによるコンテンツ認証。 Widevineに関する情報はこの文書で提供されています。
  • すべての許可要求を処理するメカニズムである許可API。 Authorization APIの詳細については、 プレーヤV3用プレーヤ認証API（廃止予定）.
Ooyalaが提供するコンテンツ保護は、スタンドアロンまたは組み合わせて複数レベルのコンテンツ保護を提供します。 Ooyalaでは、これらの機能を組み合わせてコンテンツ保護戦略を作成することができます。 追加のコンテンツ保護機能（Adobe Passなど）については、営業、カスタマーサクセスマネージャー、またはテクニカルサポートにお問い合わせください。

OoyalaとWidevine DRMのワークフロー

Widevineをご希望のお客様は、Widevineサポートを有効にするために、Ooyalaカスタマーサクセスマネージャーまたはプロフェッショナルサービスと協力する必要があります。 OoyalaのWidevineサーバーを使用すると、ライセンスはWidevineクラウドに存在します。

Widevine DRMの実装では、コンテンツが暗号化されていてライセンスなしでは役に立たないため、各コンテンツビューアには個別のライセンスが必要です。 Widevineは暗号化を実行しません、CDNからのコンテンツはすでに暗号化されています。 Widevineは、ライセンスキーの保存と取得のための安全なクラウドを提供するだけです。

Widevineと統合する場合は、コンテンツの保護がOoyalaのコンテンツ保護機能のさまざまな部分とどのように機能するかを理解する必要があります。 次の図は、WidevineがOoyalaプレーヤートークン（オプションですが推奨されるユーザー認証機能）と、ユーザー認証要求を処理するOoyala Authorization APIとの関係を示しています。

Ooyala SDKベースのアプリケーションでWidevineを使用する

iOSおよびAndroidのプレーヤーでWidevineを使用するには、ネイティブSDKをダウンロードして使用し、クライアントサイドページを作成する必要があります。 Ooyalaをダウンロードするには：
  • Android SDK - Ooyala ダウンロード
  • iOS SDK - Ooyala ダウンロード
SDKをダウンロードしたら、AndroidまたはiOSアプリを実装するためのパッケージの指示に従ってください。 Widevine機能を有効にすると、Ooyala SDKベースのアプリはWidevineとシームレスに連携するようになります。

Widevineを接続テレビで使用する

Widevineを接続されたテレビで使用したい場合は、次の作業が必要です。
  1. デバイス固有のSDKに従ってWidevineを実装します。 SDKの指示に従って、デバイスのSDKとWidevineを事前に設定します。
  2. WidevineおよびデバイスSDKをSDK固有の言語で初期化します（これはデバイスによって異なります）。 該当するプラットフォームプログラミングガイドに従ってアプリを設定します。
  3. あなたは電話をかけるでしょう 認証API （これはJSON RESTful API呼び出しです）
  4. Widevine URLを取得するか、データをストリーミングするために電話をかけます。
  5. 取得したURLの要素をWidevineフィールドの対応する値にマッピングする必要があります。 下記の値を差し込みます。
