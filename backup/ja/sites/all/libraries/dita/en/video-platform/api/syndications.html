シンジケーション

シンジケーションは、コンテンツのリストと、コンテンツのシンジケート先となる宛先で構成されています。

注意： Backlot REST APIコマンドについて詳しくは、以下を参照してください。 バックスロットAPIリファレンス.

説明

フィードタイプによって、シンジケーションで使用されるストリーム（mp4ストリームやソースファイルストリームなど）と出力形式（プレーンなMRSS、ファイルのYouTubeへの送信など）が決まります。 Ooyalaは現在、以下のコンテンツ配信先をサポートしています。

シンジケーション タイプ 説明
3GPフィード three_gp 3GPコンテナファイルへのURLを含むMRSSフィード
高度なフィード 高度な TubeMogulのMRSSへの拡張
Boxeeフィード ボクシー BoxeeのMRSSへの拡張に従ってフォーマットされたフィード
Googleフィード グーグル Googleサイトマップフィード
MP4フィード mp4 MP4コンテナファイルへのURLを含むMRSSフィード
プレイヤーフィード フィード プレーヤ埋め込みスニペットを含むMRSSフィード
Podcast（iTunes） iTunesの MRSSに対するiTunesの拡張機能に従ってフォーマットされたフィード
ろくフィード 六 Roku対応のMRSSフィード
ソース source ソースファイルへのURLを含むMRSSフィード。
ユニバーサル ユニバーサル ユニバーサルシンジケーションテンプレート言語を使用したフィード
ユーチューブ ユーチューブ アセットとそのメタデータをYouTubeにプッシュするシンジケート
iPadアダプティブビットレートフィード ipad_abr iPad（Adaptive Bitrate）互換フィード
iPadのフィード アプリ iPad互換のMRSSフィード
iPhoneの適応型ビットレートフィード iphone_abr iPhone（Adaptive Bitrate）対応のMRSSフィード
iPhoneフィード iphone iPhone互換のMRSSフィード

シンジケートアセットの種類

オーディオアセット以外のあらゆるタイプのアセットは、ユニバーサルシンジケーションおよびここで説明されているAPI要求を介してシンジケートすることができます。

新しいシンジケーションを追加

新しいシンジケーションを追加するには、名前とタイプのパラメータを指定します。

[POST] / v2 / syndications {
   プロパティ

}

一覧表示

アカウントのすべてのシンジケーションを一覧表示するには

[GET] / v2 / syndications

配信を受ける

シンジケーションを取得するには

[GET] / v2 / syndications /syndication_id

USTLテンプレートをアップロードする

USTLテンプレートを高度なシンジケーションにアップロードするには、リクエストの本文にテンプレートを含めます。

[PUT] / v2 /シンジケーション/syndication_id/ template <template>

アクセスキーをオンにする

シンジケーションのアクセスキーをオンにするには、PATCHリクエストを使用してシンジケーションの "require_access_key"を "true"に設定します。 

[パッチ] / v2 /シンジケーション/syndication_id{"require_access_key"：true}
注意：

"require_access_key"がtrueに設定されていると、フィードのsyndication_urlに移動しても機能しなくなります。 フィードにアクセスするには、 "access_key = ..."クエリ文字列パラメータをsyndication_urlに追加する必要があります。

http://api.ooyala.com/v2/syndications/syndication_id/feed?
pcode=yourPcode＆access_key = accessKey

シンジケーション用のアクセスキーを作成する

シンジケーションのアクセスキーを作成するには 

[POST] / v2 / syndications /syndication_id/ access_keys {"name"： "yourAccessKeyName"}

アクセスキーの一覧表示

シンジケーションに関連付けられているアクセスキーを一覧表示するには 

[GET] / v2 / syndications /syndication_id/ access_keys

アクセスキーを削除する

シンジケーションからアクセスキーを削除するには 

[削除] / v2 / syndications /syndication_id/ access_keysACCESS_KEY

ルート属性

次の表は、ルートを通じて表現できるすべての属性を示しています。

ルート属性 説明

タイプ：文字列
