v3アナリティクス レポートAPI

This section details the v3 Analytics Reporting API, also referred to as the v3 Analytics API and the ウーヤラIQ APIです。
The v3 Analytics Reporting API relies on the following basic concepts, almost all of which are described in Ooyala IQのしくみ の中に ウーヤラIQ ユーザーガイド：
  • レンディション: common criteria that are used to aggregate data, such as the date on which the user activity occurred or the country where the users were located.
  • メトリック: measurements of individual events related to your content, such as video plays requested or pauses.
  • フィルター: dimension values that constrain the retrieved data. For example, you can retrieve data for a specific type of device, a specific video, or specific dates.

ベストプラクティス

以下のベストプラクティスは、最適なクエリパフォーマンスとスピードを確実に得るためのものです。

  • 可能であれば、大量のデータを取得する必要がある場合は、クエリを細かく分割してください。 これにより、クエリがずっと速くなります。 例えば：
    • 1年間のデータに対してクエリを実行するのではなく、四半期（4ヶ月）ごとにクエリを実行し、その後CSVデータを結合します。
    • 調査対象のすべてのドメインに対してクエリを実行するのではなく、各ドメインに対して個別にクエリを実行し、後でCSVデータを結合します。
    • 調査対象のすべての国に対してクエリを実行するのではなく、少数の国のグループに対して別々のクエリを実行し、後でCSVデータを結合します。
  • 必要に応じて多次元クエリを使用してください。 多次元クエリを実行する場合（最大3次元で結果をグループ化する）、クエリのパフォーマンスを向上させるためにクエリをフィルタで制限するようにしてください。
  • 1分あたりv300 Analytics APIへの3呼び出しまで実行できます。 ただし、最適なパフォーマンスを得るために、一度に1つのクエリを実行して、より多くの顧客のためにキャパシティを増やすことをお勧めします。

Request Signing and Expiration

All reporting requests must be signed by your Ooyala-issued API key and secret, and your GET requests must have an explicit expiration time. How to do this is detailed in v3 Analytics Reporting APIのセットアップ/メカニズム.

注意： For ease of reading, the majority of examples here are Studio上ではサポートされていません。 signed requests.
