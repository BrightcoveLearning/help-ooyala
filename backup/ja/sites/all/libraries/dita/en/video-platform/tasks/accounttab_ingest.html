バックスロットでの取り込みログの表示

Backlotにログインすると、Ooyala Ingestion Service（OIS）を介して送信したインジェストジョブのステータスを表示できます。

[Ingestion Feed Activity]タブ（「取り込みダッシュボード」）には、特定のファイルの処理ステータスとその他の関連情報が表示されます。 小グループの取り込みジョブに焦点を合わせるためにフィルタを適用できます。 関連情報については、以下を参照してください。
  • 摂取 OISの紹介
  • ログ取得REST API OIS取り込みログへのプログラムによるアクセス用。 Backlot UIにログインしたユーザーは、[Ingestion Feed Activity]タブでOIS摂取ログを見ることができます。

Backlotにログイン

[Ingestion Feed Activity]タブを表示するには、Backlotにログインする必要があります。

[Ingestion Feed Activity]タブに移動します

次のURLに行きます アカウント > 摂取フィード活動.

取り込みダッシュボードを表示する

取り込みダッシュボードに取り込みジョブのリストが表示されます。
注意： アップロード専用権限を持つユーザーとしてログインした場合は、自分のユーザーアカウントによって送信された取り込みジョブのみを見ることができます。 何らかの権限でログインした場合は、他のユーザーから送信されたものも含め、すべての取り込みジョブが表示されます。
コラム 説明
状態 この取り込みジョブの現在の状況 詳細については、後述の「取り込みプロセスのステータス」を参照してください。
ファイル名 取り込みのために送信されたファイルの名前。
ファイルの種類 処理中のファイルの種類 以下のいずれかの値です。
  • ビデオ
  • マニフェスト
  • サムネイル
  • キャプション
作成した 取り込みジョブが作成された日時。
埋め込みコード ファイルの取り込みに成功した後にBacklotによって割り当てられたコードを埋め込みます（status = Done）。
資産ID ファイルが正常に取り込まれた後にBacklotによって割り当てられた資産ID（状況=完了）。

取り込みダッシュボードをスクロールする

デフォルトでは、取り込みダッシュボードには、過去の月のデータが一度に100レコードとともに表示されます。 後続のページに移動するには、データセットの下部までスクロールします。 ページ上部の検索アイコンを使用して、探しているファイルをすばやく見つけることもできます。

取り込みジョブをフィルタリングする

次の方法で表示を絞り込むことで、検索に集中できます。
  • ファイル名
  • 状態
  • 作成日：
    • カスタム日付範囲のカレンダー日付ピッカー
    • 先月の1月
    • 先週の1週
    • 最後の1日
  • 資産ID

取り込みプロセスのステータス

取り込みプロセス中に、取り込まれたファイルは次のような状態になります。
状態 説明
受信された ファイルがOISによって受信されたことを示します。 ファイルが受信されると、システムによる処理に適格になります。
注意： ファイルがシステムに取り込まれることはないため、リモート資産が受信状態になることはありません。
加工、処理 次のことを示します。
  • マニフェストファイル（CSVまたはXML） - 構文検証は現在行われています。
  • メディアファイル（ビデオ、サムネイル、クローズドキャプションファイル） - Ooyalaエコシステムへの取り込みが現在行われています。
ファイル待機中 マニフェストファイル（CSVまたはXML）が取り込まれて処理され、他のメディアファイル（ビデオ、サムネイル、およびクローズドキャプションファイル）が取り込まれるのを待っていることを示します。
メタデータ待機 メディアファイル（ビデオ、サムネイル、またはクローズドキャプション）が取り込まれ、関連するメタデータを含むマニフェストファイル（CSVまたはXML）が取り込まれるのを待っていることを示します。
クリックします 正常に処理された場合は、以下のことを示します。
  • マニフェストファイル - マニフェストは構文的に有効です。
  • ビデオファイル - ファイルは（そのメタデータと共に）処理され、埋め込みコードがOoyalaによって割り当てられました。
  • サムネイルファイルと字幕ファイル - ファイルは取り込まれて動画に関連付けられています。
エラー ファイルの取り込み中または処理中に問題が発生した場合、ステータスはエラー状態になります。 この場合、取り込みダッシュボードには、問題の原因を説明するエラーメッセージが含まれます。
  • マニフェストファイル - 構文的に無効であることが判明したことを示します。
  • メディアファイル（ビデオ、サムネイル、クローズドキャプション） - ファイルの処理中に問題が発生したことを示します。
