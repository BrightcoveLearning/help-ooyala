はじめに：HLS再生のアップグレードの有効化

SDKのアップグレード版（バージョン2.6.0 +）は、VisualOnとの統合により、Androidデバイス間でのVoDおよびライブストリーミング用の高度で一貫性のあるHLS再生エクスペリエンスを提供します。 アップグレード版のSDK（バージョン2.6.0 +）については、CSMまたはテクニカルサポートからVisualOnライブラリへのアクセスを要求してください。
警告：

4月の時点で、1、2016 Ooyala Android SDKはVisualOnライブラリの代わりにGoogle ExoPlayerライブラリと共に出荷されています。 実行する必要がある手順はありますが、ExoPlayerに移行すると、AndroidデバイスでのVODおよびライブストリームに対して、より一貫性のある高品質のHLSおよびMPEG-DASH再生エクスペリエンスが提供されます。

8月現在、1、2016、Ooyalaは、HLSプレーヤーとVisualOnとの統合をサポートしないか、またはVisualOnライブラリーに固有のバグ修正または改善を提供しません。

アップグレードするには、あなたがする必要があります
  • 最新のAndroid SDK 4.13.0 +をダウンロードしてください。 Ooyala ダウンロード）あなたの統合を開始します。
  • の統合手順に従ってください。 Android上のGoogle ExoPlayerとの統合.
  • アプリケーションを再コンパイルしてGoogle Playストアに送信してください。
  1. CSMからVisualOnライブラリにアクセスした後、VisualOnライブラリをダウンロードしてください。
  2. すべてをコピー zip / libs / *。jar ファイルを project / libs / フォルダにコピーします。
  3. すべてをコピー zip / libs / armeabi / *。so ファイルを project / libs / armeabi / フォルダにコピーします。
  4. すべてをコピー zip / assets / * ファイルをproject / assets /フォルダーに入れます。
  5. アプリケーションで、呼び出す前に次のSTATICプロパティを設定します。 setEmbedCode: 
    OoyalaPlayer.enableCustomHLSPlayer = true;
  6. HLS再生機能を持つ埋め込みコード（または.m3u8ファイルを指すリモートアセット）を再生します。
  7. Androidネイティブアプリを新しいOoyala Android SDKおよびVisualOnライブラリで再コンパイルします。
  8. アプリケーションをテストして、HLSストリームが正しく再生されることを確認します。
注意： Android用のVisualOnまたはOoyalaのSDKをすでに使用している場合は、最新バージョンに切り替えてアプリを再コンパイルしてください。
