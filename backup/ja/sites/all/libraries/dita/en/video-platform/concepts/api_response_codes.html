HTTP応答コード

要求結果に応じて、Backlot REST APIはさまざまなHTTP応答コードを返します。

注意： Backlot REST APIコマンドについて詳しくは、以下を参照してください。 バックスロットAPIリファレンス.
応答コード 短い説明 注釈
200 OK バックスロットAPI 要求を正常に処理しました。
204 コンテンツなし サーバーは要求を実行しましたが、entity-bodyを返す必要はありません。 応答にメッセージ本文がありません。
400 不正な要求 要求の形式が正しくないか、無効なデータが含まれています。 たとえば、 バックスロットAPI リクエストに誤った形式のJSONが含まれているか、 "400：25：00"のような無効な設定が含まれている場合、00レスポンスコードを返します。
401 許可されていません 要求が正しく署名されていないか、無効なAPIキーで署名されています。
403 禁断の 要求された操作に対して要求が正しく認証されていません。
404 見つかりませんでした リソースが存在しません。 たとえば、間違ったコンテンツIDを使用して動画を削除しようとした場合などです。
429 要求を満たすには「APIクレジット」が不十分です。 見る APIレート制限.
