資産への公開ルールの割り当て

にアセットを追加した後 バックスロットいつ、どこでコンテンツを再生できるかに影響する公開ルールを割り当てることができます。

注意： Backlot REST APIコマンドについて詳しくは、以下を参照してください。 バックスロットAPIリファレンス.
注意： POSTで初めてリモートアセットを作成するときは、 time_retsrictions 初期POSTのプロパティ。 リモートアセットを最初に作成して、埋め込みコード（コンテンツIDまたはアセットID）がOoyalaシステムに存在するようにする必要があります。 資産に対して返された埋め込みコードを受け取ったら、次のようにしてPUTを使用できます。 time_retsrictions。 の例を参照してください 既存の資産への外部IDの追加.

公開ルールの設定については、 公開ルール.

公開ルールとアセットを操作するには：

  1. を使用してルールを割り当てる PUT とともに / v2 / assets ルート、アセットID、および / publishing_rules 修飾子 次の例では 79feefbe24d9424786 への公開ルール Y1dTdvMjq9QtOM 資産： 
    [PUT]/v2/assets/Y1dTdvMjq9QtOM/publishing_rule/79feefbe24d9424786

    バックスロット 200応答を返します。

    注意： Ooyala Scratchpadでアカウントの認証情報を使用してコードサンプルを試してみてください。 スクラッチパッドを起動するには、にアクセスしてください。 Ooyala APIスクラッチパッド。 スクラッチパッドの使用方法については、 スクラッチパッドを使った練習の依頼.
  2. 公開ルールが追加されたことを確認するには、 のGET とともに / v2 / assets ルート、アセットID、および / publishing_rules 修飾子 次の例では、アセットの公開ルールを Y1dTdvMjq9QtOM ID： 
    [GET] / v2 / assets / Y1dTdvMjq9QtOM / publishing_rule

    バックスロット 公開ルールの詳細を返します。 この例では、アメリカとイギリスで、いつでも、どのドメインからでもデスクトップ/ノートブックでの再生が許可されています。

    注意： Ooyala Scratchpadでアカウントの認証情報を使用してコードサンプルを試してみてください。 スクラッチパッドを起動するには、にアクセスしてください。 Ooyala APIスクラッチパッド。 スクラッチパッドの使用方法については、 スクラッチパッドを使った練習の依頼.
