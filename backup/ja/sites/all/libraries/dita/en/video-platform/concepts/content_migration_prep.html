コンテンツ移行

このドキュメントでは、標準のコンテンツ移行、オプション、およびベストプラクティスについて説明します。

あなたが大規模なビデオライブラリを持っていて、あなたのコンテンツをOoyalaに移動するのに援助を必要とするならば、あなたはOoyalaのコンテンツ移行サービスから最も利益を得るでしょう。 自己移行（つまり、Backlot Webアップロード、FTP、Aspera、API経由）が選択肢にならない場合は、プロフェッショナルサービスによる移行が理にかなっています。

コンテンツの移行は3段階のプロセスです。
  1. ソースファイルとマニフェストファイルを準備します。
  2. ソースファイルとマニフェストファイルをアップロードします。
  3. あなたのビデオがライブになったら、それらをあなたのウェブサイトに公開するか、新しいものとあなたの古い埋め込みを交換し始めてください。

ステップ1：ソースファイルとマニフェストファイルを準備する

あなたは常に利用可能な最高品質のソース素材から始めるべきです。 Ooyalaは、以下に具体的な推奨事項を示します 素材の品質と推奨事項（ビデオとオーディオ）。 ソースファイルの準備ができたら、マニフェストファイルで作業を始める必要があります。

用語 メタデータ ここでは2つの方法で使用されます。
  • 「カスタムメタデータ」とは、ビデオ自体を説明するデータを格納するために使用される最大100の名前と値のペアを意味します。 たとえば、映画のライブラリでは、値のペアという名前を見つけることができます。 
    "監督"： "ミシェルゴンドリー"、 "年"： "2004"、 "評価"： "R"
  • 「ラベル」は、表示、分析、シンジケーションなどのためにビデオをグループ化するために使用できるタグです。任意のビデオに1つ以上のラベルを付けることができ、すべてのラベルに多数のビデオを含めることができます。 たとえば、映画ライブラリーでは、制作スタジオに従ってビデオにラベルを付けることができます。

カスタムプレーヤー、広告セット、またはシンジケーションルールの一括割り当てを計画している場合は、アセットを複数選択する最も簡単な方法は、それらをラベルでグループ化することです。 たとえば、 "Blue Player"、 "Green Player"、 "Only Colombia"、 "Kid Friendly"などのようなアセットラベルを追加して割り当てることは意味があります。

カスタムメタデータ。 動画ライブラリの例を続けると、マニフェストファイルが処理されたら、Backlotにログインして各アセットの[カスタムメタデータ]タブで割り当てられたカスタムメタデータを見つけることができます。


ラベル階層。 [管理]ウィンドウでラベルの階層を確認してください。


資産ごとのラベル割り当て。 詳細タブのアセットごとのラベル割り当てを参照してください。


XMLまたはCSVでマニフェストファイルを作成します。 見る マニフェストファイル形式 詳細については。

ステップ2：ソースファイルとマニフェストファイルをアップロードする

Ooyalaはいくつかをサポートしています コンテンツを取り込む方法:
  • バックスロットUI
  • Ooyala Import Services（OIS）。動画に関するメタデータの有無にかかわらず、カンマ区切り（CSV）形式またはXML形式のいずれかで、次の転送メカニズムを使用します。
    • FTP
    • アスペラ
    • MRSS
  • Backlot API、特に/ v2 / assetsルート
見る 摂取 詳細については。

ステップ3：あなたのビデオがライブになったら、あなたのウェブサイトに公開しましょう。

コンテンツの移行プロセスを通じて、ビデオアセットはいくつかの状態を通過します。


エラー状態のビデオが表示された場合は、テクニカルサポートに連絡してトラブルシューティングを行ってください。 ライブ状態の動画は公開の準備ができています。

Webページにプレーヤーを埋め込む方法とその構造については、を参照してください。 プレーヤー.

あなたが古い選手を交換することを計画しているならば、あなたはする必要があるでしょう：
  1. あなたの古いプレイヤーの埋め込みをすべて見つけて特定します。
  2. それぞれの動画に古い埋め込みと結び付けることができる一意の識別子を割り当て、その識別子をどちらかのフォルダに保存します。 id CSVマニフェストファイルのフィールドまたは GUID XMLマニフェストファイルのフィールド。
  3. コンテンツが公開されたら、APIを使用して新しい埋め込み識別子と古い埋め込み識別子をマッピングするファイルを作成する必要があります。 また、あなたのサイトが動的ページ、プレイリスト、またはチャンネルを使っているなら、あなたはOoyalaのAPIを使ってそれらを再プログラムする必要があるでしょう。 見る バックスロットREST API.
