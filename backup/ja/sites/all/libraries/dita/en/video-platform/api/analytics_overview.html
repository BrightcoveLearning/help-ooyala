v2 Analytics API（廃止予定）

アカウントに基づいて、さまざまな分析パッケージを利用できます。

警告： v2 Analytics APIは非推奨になりました。 見る ウーヤラIQアナリティクス 新しいv3 Analytics APIの詳細については、

以下は、現在提供されているOoyalaアカウントパッケージです。

  • カスタム分析パッケージ - カスタム分析
  • デフォルトの分析パッケージ - 合計、ビデオ、エンゲージメント、共有
  • デバイスタイプ分析パッケージ - デバイスタイプ、プラットフォーム
  • ドメイン分析パッケージ - ドメイン、URL
  • 地域分析パッケージ - 国、地域、都市、DMA

すべてのパッケージはここで議論されています。

当社のすべてのプラットフォームサービスは現在、分析データの生成元であるOoyalaのプレイヤーに依存しています。

一般化された構文

1つの例外を除いて、AnalyticsのすべてのAPIリクエストは のGET あなたがそれを書くのではなく、データを読んでいるだけなので。 （唯一の例外は、追跡用の独自のカスタムタグを作成するためのカスタム分析です。 カスタム分析：タグ.)

したがって、直接 のGET 要求に応じて、さまざまな詳細レベル、さまざまな種類のデータ、またはさまざまな制約を指定します。 ルート上のどちらかがあります 属性 or クエリ文字列パラメータ または両方：
[GET] / v2 / analytics / reports /attributes_suchas_levels_geo_andsoforth[/optional_attributes]?query_string_parms

その結果、ルート自体が長く表示されたり、冗長になったりすることがありますが、その冗長性のために、ルートの意味は明らかです。

アナリティクスルートの構文では、表記で [/optional_qualifiers] 角括弧は、その属性がオプションであることを示します。 それらは実際の構文の一部ではありません。

  • Analytics APIに関する考慮事項（廃止予定）
    v2 Analytics APIに関するいくつかの注目すべき点に注意してください。 バックスロットUI そしてAPI。
  • 表示、再生、および再生の開始（廃止予定）
    Ooyala Playerの使用は、表示、再生、および再生開始のイベントによって細分化されます。
  • v2 Analytics APIを使用してデータをエクスポートする方法（廃止予定）
  • パフォーマンス（廃止予定）
    資産実績レポートは、アカウントまたは資産レベルでの包括的な実績を示します。
  • 共有（非推奨）
    共有レポートには、アカウント内のすべてのコンテンツまたは個々のアセットに対する共有操作（Facebook、Twitterなど）の数が表示されます。
  • エンゲージメント（非推奨）
    エンゲージメントレポートには、ビデオの再生中の視聴者数が表示されます。これにより、視聴者が興味を失うポイントを特定し、広告掲載の効果を見つけ、その全体的な効果を評価できます。
  • 配達（廃止予定）
    配信レポートは、アカウント内のすべてのコンテンツまたは個々のアセットに配信されたVODバイト数を返します。 配信APIはUsage REST APIに移行されました。
  • カスタム分析：タグ（廃止予定）
    タグを使用すると、カスタムレポートを作成するために使用できるOoyalaアナリティクスにカスタム情報を挿入できます。
  • カスタム分析：レポーティング（廃止予定）
    カスタム分析レポートには、設定したタグの掲載結果情報が表示されます。
  • 共通属性とクエリ文字列パラメータ（廃止予定）
    これらの属性とクエリ文字列パラメータは、Analyticsルートで使用されます。
