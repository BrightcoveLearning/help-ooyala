共通配置アーキテクチャ：複数層

Ooyalaは、サーバー配備には多層設計をお勧めします。

あなたの視聴者にとって最高のパフォーマンスとあなた自身のサービスの効率のために、「関心の分離」、サーバーの複数の層を持つ典型的なデザイン、全体のサービスのある部分を満たすことに専念するサーバーアーキテクチャを計画してください。 単純な概念図を以下に示します。前面、中央、および背面という一般的な層です。

フロント 正面にあなたのWebサイトがあります。

データベースやその他のクエリの結果を保持するために、前面にキャッシュサブシステムを実装することをお勧めします。 これは、視聴者に役立つ最もパフォーマンスの高い方法です。 ベストプラクティス より多くの情報があります。

前面の代表的なプログラミング言語はJavaScriptで、JQuery（データベース検索用）などの補助ツールがあります。 （もちろん、さまざまなプログラミング言語が使用されています。）
真ん中 通常、中間層は、前面と背面の中間にあるある種のデータベースまたはその他のサービスです。

真ん中は、アプリケーションプログラミングインターフェース（API）やその他のメカニズムを通じて、裏側でOoyalaサービスと通信します。 ウーヤラから取り出されたデータは、一般的に最前線に役立つように中間データベースに格納されます。

すべてのアーキテクチャが中間層を使用または必要としているわけではありません。これは、大規模配置ではおそらく最も一般的です。 真ん中がない場合は、正面自体が一般的に真ん中に追いやられる可能性がある機能を果たします。
バック このシンプルでお勧めのアーキテクチャでは、バックはOoyalaのAPIと他のサービスで構成されています。 （すべてのサービスが表示されているわけではなく、またあなた自身のバックエンドサービスも表示されていません。）
