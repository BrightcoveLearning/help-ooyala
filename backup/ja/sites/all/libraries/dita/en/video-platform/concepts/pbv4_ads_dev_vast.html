VAST広告とVPAID広告の統合

You can integrate VAST and VPAID ads with Player V4 using the VASTとVPAIDの広告プラグイン (ad_manager_vast.js).

Enabling the VAST/VPAID Ad Source(s) for Your Account

Before you can use VAST and VPAID ad integration:
  1. にログインします Ooyala カスタマーポータル.
  2. Submit a ticket requesting that the VAST and VPAID ad source be added to your Ooyala account.

広告コントロールバーとマーキー機能をカスタマイズする

You can modify skin.json to customize ad control bar and ad marquee functionality for the VAST and VPAID plugin rendering VAST 2.0 or VAST 3.0 ads.
注意： For the VAST and VPAID plugin rendering VPAID 2.0 ads, the ad control bar and ad marquee configurations in skin.json are ignored (and are forcibly set to off) to avoid blocking ad interactivity.

Step 1: Create an Ad Set Using Backlot

After you have VAST and/or VPAID enabled enabled for your account, you can associate VAST and/or VPAID ad tags with your videos. For VAST and VPAID ads, you must create an ad set with Backlot using:

  • バックスロットUI:To create an ad set using the Backlot UI, see 広告ソースと統合するための広告セットの作成.
  • Backlot API: To create an ad set programmatically, see 広告セット.
注意：
  • For Ad Source, choose VAST Compliant In-Stream. Disregard all other options with "VAST" in the name.
  • For this version, the steps for configuring VAST and VPAID ad sets are identical.

VAST Compliant In-Stream Ad Set

Step 2: Assign the Ad Set to an Asset Using Backlot or the Player API

以下を使用して、広告セットをアセットまたは複数のアセットに割り当てます。
  • バックスロットUI: To assign your VAST ad set to a single asset, see Managing Monetization. For instructions on how to assign your VAST ad set to multiple assets, see Bulk Applying Settings.
  • Player API: With the Player API, you can associate an ad set with only an asset on your web page. To associate an ad set with an asset on multiple players, you must replicate the code for each player. See Assigning Ad Sets Dynamically 詳細については。
  • バックスロットAPI: With the Backlot API, you can associate an ad set with an asset more concretely. When you associate an asset with an ad set using the Backlot API, the asset and the ad set will be paired together on any player and page on which you play the asset. See 広告セットをアセットに関連付ける 詳細については。

Step 3: Integrate the VAST and VPAID Plugin with the Ooyala Player on the Page

注意：
  • If you use Backlot to generate your HTML embed code, and you choose the V4 HTML5標準プレーヤー埋め込みコード（推奨） option, the VAST and VPAID plugin is automatically included and should not be explicitly added to the web page where you launch the player. For details, see バックスロットでのプレーヤー埋め込み設定.
  • あなたが V4 HTML5プレーヤー埋め込みコード（詳細） option in Backlot, or if you manually create the HTML embed code yourself, you need to add the VAST and VPAID plugin to your web page so that the VAST and VPAID plugin loads before the player is created.
For every page on which you want to use the VAST and VPAID plugin with a player (regardless of how you associated VAST / VPAID ads with your videos):
注意： どこにいても url_where_hosted サンプルコードでは、（コード内の）これを、リソースがホストされている場所を指すURLに置き換えます。 Ooyalaがホストするパスのリストについては、を参照してください。 OoyalaがホストするプレイヤーV4リソース。 URLは、同じホスト上の場所（内部リンク）または別のホスト上（前に次の文字列が付いた場所）を指すことができます。 http:// or https://） 自分でリソースをホストしている場合（ ホスティングプレイヤーV4リソース）、ファイル内のパスの依存関係を必ず確認してください。
  1. 加えます ad_manager_vast.min.js script to the page where you are loading the player. You must load this plugin After あなたがロード core.min.js. 
    <script language =“ javascript "src ="url_where_hosted/ad_manager_vast.min.js"></script>
  2. プレーヤV4を使う OO.Player.create function to create the player.
  3. グローバルパラメータを渡します。 見る 広告パラメータを設定する.
  4. Pass VAST ad tags to the Ooyala player using the 広大 parameter and its child parameters. See VAST-Specific Embedded Parameters for the list of key/value pairs you can use with the 広大 パラメータ。
注意： In this version, you can override only the Ad Tag setting with a page-level parameter (vast.tagUrl).
