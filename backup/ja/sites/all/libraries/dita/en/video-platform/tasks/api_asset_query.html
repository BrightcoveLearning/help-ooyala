資産の検索

ラベルやメタデータなど、さまざまな特性を持つ検索キーワードをクエリ文字列パラメータとして指定することで、資産を検索できます。

注意： Backlot REST APIコマンドについて詳しくは、以下を参照してください。 バックスロットAPIリファレンス.
/ v2 / assets routeは一般的なクエリ機能を持ち、クエリ文字列パラメータもあります where, 含まれる, 含ま そして そして （ラベルのみ）など。

  • ソート用のクエリ文字列パラメータ ORDERBY、はオプションの引数を持ちます。 上昇 or 降順.

  • とともに 制限 クエリ文字列パラメータを使用すると、結果を目的のエントリ数だけに制限できます。

  • とともに そして 演算子あなたはラベルの交差点を検索することができます。 つまり、2つ以上の異なるラベルを持つアセットの場合です。

  1. というメタデータフィールドを持つアセットを見つけるには スピード の値で バースティ: 
    [GET] /v2/assets?where=metadata.speed='bursty'&include=metadata
  2. ラベルが付いているアセットを探す スポーツの: 
    [GET] / v2 / assets？where = labelsは 'Sports'を含みます
  3. 両方のラベルを持つアセットを探す スポーツの そしてラベル 野球: 
    [GET] / v2 / assets？where = labelsは 'Sports'を含み、labelsは 'Baseball'を含みます
  4. 特定の日時以降に更新されたアセットを見つけるには： 
    [GET] /v2/assets?where=updated_at>'2011-06-04T01:22:50Z'
  5. 並べ替える ORDERBY で結果を制限する 制限: 
    [GET] / v2 / assets？where = description = '海中'。 AND期間> 600＆orderby = created_at降順＆limit = 5
