インプレッションイベントを返す

APIを介してOoyala Discoveryの推奨事項をユーザーに表示するには、表示される結果を報告するためにOoyalaを呼び出す必要があります。 報告された情報はDiscoveryの実装に供給され、推奨アルゴリズムが強化され、サイトと視聴者の結果が最適化されます。

最良のフィードバックを得るために、Ooyalaは、推奨事項に表示されている各ビデオについてインプレッションイベントを送信することをお勧めします。 インプレッションはの計算の一部です インサイトレポート。 詳細なインプレッションイベントを取得すると、分析レポートの精度が向上します。

インプレッションを記録するための実装を設定するには、次の手順に従います。

  1. Ooyala Discoveryの推奨事項を表示するようにWebページまたはアプリを設定します。
  2. 生成された デバイスID.

    Flashの場合、次の関数例はFlashのデバイスIDを取得します。

    public function get deviceId（）：String {var so：SharedObject = SharedObject.getLocal（ "auth"）; var deviceId：String = so.data.auth？ so.data.auth：null。 return（deviceId == null || deviceId == ""）？ "不明"：deviceId; }
    注意： デバイスIDが利用できない場合、関数例は以下を返します。 未知の.
  3. 加えます デバイスID パラメータ。
  4. を含む投稿リクエストを行います bucket_info ユーザーに推薦および/または表示した各ビデオのパラメータ。 この手順は、次のコードサンプルに示されています。 
    [POST] / v2 / discover / feedback / impression {"device_id"： "デバイスID"、" bucket_info "："bucket_info"、“ ui_tag”：“ tag1”}

    Ooyala Discoveryは200応答を返します。

注意： Ooyala Scratchpadでアカウントの認証情報を使用してコードサンプルを試してみてください。 スクラッチパッドを起動するには、にアクセスしてください。 Ooyala APIスクラッチパッド。 スクラッチパッドの使用方法については、 スクラッチパッドを使った練習の依頼.
