DRMポリシーの割り当て

設定が完了したら、アカウントまたは資産レベルでDRMポリシーを割り当てることができます。

Ooyalaカスタマーサクセスマネージャーまたはサポートエンジニアは、ポリシーを資産に割り当てる前に、ポリシーを設定する必要があります。

資産にDRMポリシーを割り当てるには：

  1. DRMポリシーのIDを取得します。 
    [GET] / v2 / drm_policies

    バックスロット 次のような応答を返します。

    {"アイテム"：[86ff97ae7c81495eacbd9a01feff0f21、43ff97ae7c81495eacbd9a01feee0f38]}
    注意： Ooyala Scratchpadでアカウントの認証情報を使用してコードサンプルを試してみてください。 スクラッチパッドを起動するには、にアクセスしてください。 Ooyala APIスクラッチパッド。 スクラッチパッドの使用方法については、 スクラッチパッドを使った練習の依頼.
  2. DRMポリシーをアセットに割り当てます。 
    [PUT] /v2/assets/IzNnllMjphu2XF3_UgPROoCi9B2rtWs/drm_policy/86ff97ae7c81495eacbd9a01feff0f21

    バックスロット 200応答を返します。

    注意： Ooyala Scratchpadでアカウントの認証情報を使用してコードサンプルを試してみてください。 スクラッチパッドを起動するには、にアクセスしてください。 Ooyala APIスクラッチパッド。 スクラッチパッドの使用方法については、 スクラッチパッドを使った練習の依頼.
  3. DRMポリシーをアカウントのデフォルトにする場合は、アカウントレベルでポリシーを割り当てることができます。 
    [PUT] /v2/provider_drm_policy/86ff97ae7c81495eacbd9a01feff0f21

    バックスロット 200応答を返します。

    注意： Ooyala Scratchpadでアカウントの認証情報を使用してコードサンプルを試してみてください。 スクラッチパッドを起動するには、にアクセスしてください。 Ooyala APIスクラッチパッド。 スクラッチパッドの使用方法については、 スクラッチパッドを使った練習の依頼.
