カスタム分析：タグ（廃止予定）

タグを使用すると、カスタムレポートを作成するために使用できるOoyalaアナリティクスにカスタム情報を挿入できます。

警告： v2 Analytics APIは非推奨になりました。 見る ウーヤラIQアナリティクス 新しいv3 Analytics APIの詳細については、

タグを作成する

新しいタグを作成するには：

[PUT] / v2 / analytics / tags /tag_id{
  
  プロパティ 
 
}

リストタグ

アカウント内のすべてのタグを一覧表示するには

[GET] / v2 / analytics / tags

タグを取得

特定のタグに関する情報を取得するには

[GET] / v2 / analytics / tags /tag_id

タグを削除

タグを完全に削除するには：

[削除] / v2 / analytics / tags /tag_id

タグを更新

タグを更新するには

[パッチ] / v2 / analytics / tags /tag_id{
  
  プロパティ 
 
}

プロパティ

次の表は、シンジケーションに関連付けることができるすべてのプロパティについて説明しています。

プロパティ 説明 必須？
DISPLAY_NAME

レポートに表示される名前。

タイプ：文字列。

デフォルト：なし

制約事項：255文字未満です。

条件：作成操作に必要です。

例：「18から24年まで」

条件付きの
タグ

タグのID

タイプ：文字列。

デフォルト：なし

制約事項：8未満の英数字、スペースを含めることはできません。

条件：作成および削除操作に必要です。

例：「18to24」

条件付きの

例

この例では、次の名前のタグを作成します。 premium_customer: 

[PUT] / v2 / analytics / tags / premium_customer {"display_name"： "Premium Customers"}

そしてこのレスポンスを返します：

{"display_name"： "プレミアム顧客"}
注意： Ooyala Scratchpadでアカウントの認証情報を使用してコードサンプルを試してみてください。 スクラッチパッドを起動するには、にアクセスしてください。 Ooyala APIスクラッチパッド。 スクラッチパッドの使用方法については、 スクラッチパッドを使った練習の依頼.

この例ではタグを変更します。 

[PATCH] / v2 / analytics / tags / premium_customer {"tag"： "super_premium_customer" "display_name"： "スーパープレミアムのお客様"}

そしてこのレスポンスを返します：

{"tag"： "super_premium_customer" "display_name"： "スーパープレミアムのお客様"}
注意： Ooyala Scratchpadでアカウントの認証情報を使用してコードサンプルを試してみてください。 スクラッチパッドを起動するには、にアクセスしてください。 Ooyala APIスクラッチパッド。 スクラッチパッドの使用方法については、 スクラッチパッドを使った練習の依頼.
