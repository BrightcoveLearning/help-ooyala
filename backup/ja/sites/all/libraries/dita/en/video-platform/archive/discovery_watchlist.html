キューの管理（廃止予定）

Discovery APIを使用して、視聴者の都合に合わせて再生するためのビデオのキューまたはライブストリームの機能を含めることができます。

重要： キューAPIは無効です。 代替オプションについては、Ooyalaの担当者にお問い合わせください。

ディスカバリーキューはアセットのリストで、それぞれに有効期限、再生の位置（再生ヘッド、または最後の再生の位置）、およびその他の情報が含まれています。 キューは特定のビューアに属し、そのビューアまたはそれを操作できるのはその人だけです。 キューを作成する必要はありません。 アイテムを追加または削除するだけで済みます。

視聴者は自分のキューに任意のアセットを追加できます。 次のいずれかの条件が満たされるまで、アイテムはキューに残ります。
  • 関連付けられたアセット自体が削除されるか、関連付けられたライブイベントが終了して「再実行」されません（ライブストリームのVODバージョン）。
  • 資産全体が表示されます。

アセットを視聴者のキューに追加する

[PUT] / v2 / discover / queue / [asset | live_asset] /asset_id{"expires_at"： "dateTimeInISO8601-UTC形式"}

キューを表示する

[GET] / v2 / discover / queue

キューからアイテムを削除する

[削除] / v2 /ディスカバー/キュー/ [資産|ライブ資産] /asset_id

ルート属性

次の表は、ルートを通じて表現できるすべての属性を示しています。

ルート属性 説明
[資産|live_assets] キューに追加されているアセットの種類

タイプ：文字列

デフォルト：なし

例：[PUT] / v2 /ディスカバー/キュー/ビューア/ 54321 /アセット/ 2162e39c331d407d9176895f34e36af8
asset_id 資産ID（埋め込みコード、またはコンテンツID）。 自分がその所有者（プロバイダ）である場合にのみ、アセットを視聴者のキューに追加（PUT）できます。

タイプ：文字列

デフォルト：なし

例：pxcm9lMzpfyOKKtj1LiplPwn3eqm67dM

クエリ文字列パラメータ

次の表は、クエリ文字列のすべてのパラメータについて説明しています。

パラメーター 説明 必須？
order_by

並べ替えるフィールド。

タイプ：文字列

デフォルト：none

有効な値： expires_at[- desc|-asc] | added_at[- desc|-asc] | 再生中 | last_watched_at[- desc|-asc]どこ DESC そして ASC 降順または昇順のソート順です。

例：[GET] / v2 / discover / queue / viewers / 091iut98ycbkdwp9y8fwjhk？orderby = expires_at-asc

いいえ

プロパティ

次の表は、キューに関連付けることができるすべてのプロパティについて説明しています。

プロパティ 説明 必須？
expires_at

ISO-8601 / UTC形式の資産の有効期限/時間

タイプ：dateTime

デフォルト：有効期限なし

Example: "2014-04-15T21:00Z"

いいえ

例

この例では、アセット2162e39c331d407d9176895f34e36af8をビューア54321に関連付けられたキューに追加し、オプションの有効期限を設定します。 
[PUT]/v2/discover/queue/viewers/54321/assets/54321/2162e39c331d407d9176895f34e36af8{  
   "expires_at":"2013-08-15T21:00Z",
}
