推奨コンテンツの傾向

Ooyala Discoveryのトレンド分析アルゴリズムは、現在あなたのサイトで人気のあるコンテンツを見つけ、その人気を高めるように設計されています。

注意： （要求ごとに個別のパラメーターを使用して）アドホック要求を作成するのではなく、ディスカバリー・プロファイルを使用して設定と推奨動作を一元管理することを検討してください。 検出プロファイルは、ユーザーが作成した設定の定義済み構成です。 のGET ルート 詳細については、 ディスカバリープロファイル.

トレンドコンテンツについて

たとえば、RedditまたはFacebookであなたのビデオの1つが人気になった場合、それはトレンドの結果に現れるかもしれません。 表示されたら、それはあなたの通常のサイト訪問者によって発見され、その人気を強化するでしょう。 さらに、あなたの通常の視聴者はトレンドの結果に影響を与えています。 新しい訪問者があなたのサイトを見つけたとき、良いアドバイスは彼らがより長く滞在する可能性を高くするでしょう。

さらに、APIを介してトレンド分析を使用すると、表示できる内容をより細かく制御できます。 たとえば、次のようなロジックを定義できます。

  • 視聴者が30日を超えてサイトにアクセスしていない場合は、その月のトレンドコンテンツを表示します。
  • 視聴者が数日前に8から30の間にサイトにアクセスした場合は、その週のトレンドコンテンツを表示します。
  • 視聴者が数日前に1から7の間にサイトにアクセスした場合は、その日のトレンドコンテンツを表示します。
  • 視聴者が24時間前にそのサイトにアクセスした場合は、その時間のトレンドコンテンツを表示します。

ステップ

トレンドコンテンツに基づいて推奨を返すようにDiscoveryを設定するには、次の手順を使用します。

  1. 次のオプションから選択します。

    • 最も人気のあるビデオを入手してください。

      次の例では、すべての国でその日に最も人気のある5つのビデオを返します。

      [GET] / v2 /発見/トレンド/ top？国=すべて＆時間=今＆ウィンドウ=日＆制限= 5

    • 最も勢いのある動画を入手してください。

      次の例では、すべての国でその日の勢いが最も大きい5つのビデオを返します。

      [GET] / v2 /発見/トレンド/勢い？国= all＆time = now＆window =日＆limit = 5
