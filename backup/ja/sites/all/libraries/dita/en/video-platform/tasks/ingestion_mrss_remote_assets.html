MRSSを介したリモートアセットの取り込み

Media RSS（MRSS）を使用してリモートアセットを追加または更新できます。

詳細については、を参照してください。 リモートアセット.

MRSSを使用してリモートアセットを追加または更新するときは、次のように指定してアセットをリモートとして識別します。 <ooyala：remoteasset> タグ（の代わりに <メディア：コンテンツ>MRSSフィード項目の）。

注意： リモートアセットを追加するか更新するかを決定するために、Ooyalaは指定されたものを使用します。 <guid> （ない場合は <guid> 指定されている、指定されている <url>）リモートアセットが現在存在する（更新）かどうか（追加）を判断します。

リモートアセットの追加

次のコード例は、リモートアセットを作成します（指定された<guid>は存在しません）。

<rss xmlns：media = "http://search.yahoo.com/mrss/" xmlns：dcterms = "http://purl.org/dc/terms/" xmlns：fh = "http://purl.org / syndication / history / 1.0 "xmlns：ooyala =" http://www.ooyala.com/mrss/ "version =" 2.0 "> <チャンネル> <item> <ooyala：remoteasset durationInMs = '1000' url =" http ：//mysite.com/RCTTestAssets/mrss/assets/Test.mp4 "/> <media：thumbnail url =" http://mysite.com/test.jpg "/> <media：subTitle href =" http：/ /mysite.com/test.dfxp "/> <media：title>テスト</ media：title> <media：description>テスト</ media：description> <ooyala：labels> /テスト</ ooyala：labels> <link > http://mysite.com/videos/sports/test_0010.html </link> <！ - フライト時間の設定 - > <dcterms：有効>開始= 2011-06-28T15：15：00-07：00 ; end = 2022-12-01T16：00：00-07：00;スキーム= W3C-DTF

リモートアセットの更新

既存のリモート資産の場合は、MRSSを介して以下のフィールドを更新できます。

  • メディアタイトル
  • メディア：説明
  • ooyala：字幕
  • ooyala：メタデータ
  • dcterms：有効
  • メディア：サムネイル
  • ooyala：ラベル
MRSSフィード項目を送信するときに、MRSSフィード項目に <項目> tagを指定すると、システムは該当する変更内容でリモート資産を更新します。
