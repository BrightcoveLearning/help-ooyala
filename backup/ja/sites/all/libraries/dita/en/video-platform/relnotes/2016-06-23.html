Android用モバイルSDK v.4.15.0

ダウンロード 最新バージョン Android用モバイルSDKの概要

新機能と強化

  • FacebookはもはやMavenリポジトリと一緒にReact Native AARファイルの最新バージョンを配布していないので、React Nativeファイルは現在で配布されています。 OoyalaSkinSDK.jar。 詳細については、を参照してください。 Android用プレイヤースキン.
  • SecurePlayerがバージョン3.8.2.0に更新されました。 SecurePlayerを使用している場合は、SecurePlayerライブラリをこのバージョンにアップデートする必要があります。

解決済みの問題

  • ファイルが重複しているためにgradle2でビルドに失敗した場合は、build.gradleの除外リストにファイル名の重複を追加してください。 例えば：
    包装オプション{除外 'jsr305_annotations / Jsr305_annotations.gwt.xml'}
  • ミッドロール広告があるときにExoPlayerSampleAppで再生の最後までシークしようとすると、アプリがクラッシュまたはフリーズすることがあるという問題がありました。 この問題は解決されました。
  • シーク時にOoyala Skin SDKスクラバーバーをタッチに反応させるのが難しいという問題がありました。 この問題は解決されました。
  • 広告イベントの順序に問題がありました。 これらの問題は解決され、以下のイベントの順序が適用されるようになりました。
    1. An AdPodStarted イベントは、1つまたは複数の広告になる可能性があるものの始まりを示します。
    2. この時点で、複数の広告開始イベントと広告完了イベントが発生する可能性があります。
    3. An AdPodEnded イベントは広告の終了とコンテンツの再開を意味します。

既知の問題点

React NativeにAndroidアプリケーションのメモリリークとして現れるバグがあります。 この問題は現在調査中ですが、この問題を回避したい場合は、次のようにしてください。
  1. アプリケーションのSkin.jsonで、 "startScreen" - > "showPromo"：trueと設定します。
  2. アプリケーションのPlaybackアクティビティで、Optionsオブジェクトを作成したら、OptionsBuilder.setShowPromoImage（true）を呼び出します。
これは従来のOoyalaのプロモ画像機能を使用するため、メモリリークを回避しながらプロモ画像を作成することができます。

iOS用モバイルSDK v.4.14.0

ダウンロード 最新バージョン iOS用モバイルSDKの概要

新機能と機能強化

iOS用Adobe Marketing Cloud SDKは、Ooyala Mobile SDK for IOSに含まれるようになり、ビデオと広告の両方の分析をサポートします。

APIの変更

  • から発生する以下のイベント OOOoyalaPlayer 追加されました：
    • OOOoyalaPlayerAdStarted
    • OOOoyalaPlayerAdCompleted

解決済みの問題

  • Fairplayの再生で過度のログ記録が発生するという問題がありました。 この問題は解決されました。
  • シークに関する問題がありました。 これらの問題は解決されました。
  • 1つのVAST広告に複数のリニア広告が含まれるという問題がありました 創造的 タグを使用すると、プレロール広告が1つだけ表示されるのではなく、プレーヤーがクラッシュします。 この問題は解決されました。

新しいtvOS SDK

最新バージョンの tvOS SDK そして tvOSスキンSDK.

新しいOoyala tvOS SDKは、HLSおよびMP4アセットの基本的な再生を提供し、FairPlay DRM、Ooyala広告、VAST広告、およびOoyala分析をサポートします。

見る tvOS用モバイルSDK tvOS SDKとの統合方法およびOoyala tvOSプレーヤーUIのカスタマイズ方法について詳しくは、を参照してください。

新しいOoyala IQ JavaScript SDK

新しいOoyala IQ JavaScript SDKは、サードパーティのプレイヤーをOoyala IQと統合するために利用可能です。 新しいライブラリーは、Thriftエンドポイントを経由せずに直接イベントをJSON APIエンドポイントに送信します。これにより、既に使用可能だったものに加えて、以下のメトリックを報告することができます。
  • イベントを探す：視聴者がビデオを通してシークする（進行インジケータを前方または後方にドラッグする）ときはいつでも。
  • 選手情報：一意の識別子、プレイヤーの名前、およびプレイヤーのバージョン。
注意： たとえば、Kaltura Player、FlowPlayerなどのためにOoyala IQ JavaScript SDKの上に構築されている現在の標準アダプタは、これらの追加のメトリックをサポートしていません。

新しいライブラリについて詳しくは、を参照してください。 JS SDKを使用したサードパーティ製プレイヤーとIQの統合.
