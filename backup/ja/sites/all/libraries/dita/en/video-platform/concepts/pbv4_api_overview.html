プレイヤーV4 米国

ランニング中のプレイヤーはそのライフサイクルの間に様々な状態を経験します。 例えば：
  • ビデオとメタデータは ローディング プレーヤーに。
  • ビデオ再生は 準備 開始します。
  • プレイヤーは現在 再生 ビデオコンテンツ
  • プレイヤーは現在 一時停止 （再生が始まった後）。
  • プレイヤーは バッファリング - 継続するのに十分なビデオデータがないため、再生が停止し、さらにダウンロード中です。
  • プレイヤーは エラー それはビデオの再生を妨げます。
現在のプレーヤの状態によって、現在の状況でどの機能が利用可能かが決まります。 たとえば、プレーヤがプレーヤに入っているときに再生操作を使用できます。 準備 状態で、一時停止操作は 再生 状態。 状態変化はイベントによって引き起こされます（下記参照）。 プレイヤーの状態の一覧については、を参照してください。 https://docs.brightcove.com/apidocs-ooyala/player_v4_js/OO.STATE.html.

プレイヤーV4 イベント

プレーヤーのランタイムイベントは、次のいずれかの方法で発生させることができます。
  • エンドユーザーの操作によって手動で（たとえば、[再生]ボタンまたは[一時停止]ボタンをクリックする）
  • API呼び出しを介してプログラム的に
一般的な再生イベントの例をいくつか示します。
  • 再生が始まりました
  • 再生が完了しました
  • 広告の掲載が開始されました
  • 1つ以上の広告の再生が完了しました
  • エラーが発生しました
イベントはプレイヤーの状態の変化を引き起こす可能性があります。 たとえば、ユーザーが再生ボタンをクリックすると、「再生が開始されました」イベントが発生し、プレーヤーの状態が次のように変更されます。 再生。 プレイヤーイベントのリストについては、を参照してください。 https://docs.brightcove.com/apidocs-ooyala/player_v4_js/OO.EVENTS.html.
