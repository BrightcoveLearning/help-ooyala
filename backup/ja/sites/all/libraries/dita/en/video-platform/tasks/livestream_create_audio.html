ライブオーディオストリームの作成（廃止予定）

Ooyalaライブストリーミングは、既存のライブストリーミングURLの上のコントローラーおよびスキンとして機能します。

注意： Backlot Liveストリーミングは推奨されておらず、無効にされる予定です。 詳細と代替案については、 OVPリリースノート.

ライブオーディオストリームをBacklotに追加するには、ブロードキャスト元の場所にHTTPエンコーダのURLが必要です。 現在、Ooyalaは着信ストリームからオーディオを取得しないため、URLを生成するオーディオエンコーダを介してライブイベントキャプチャを実行する必要があります。

始める前に、次のものが揃っていることを確認してください。

  • HTTPを介してストリーミングできるエンコーディングソフトウェア（Windows Mediaエンコーダ、Ogg Vorbis、Speex、Adobe Flash Media Liveエンコーダ（FMLE）など）。
  • ストリーミングサーバ（例：Akamai Media Delivery、Flash Media Server、Wowza Media Server、Linux用のFlumotion、Unix / Linux用のIcecast、Red5）。
  • 配布ソフトウェア（Ooyala BacklotおよびOoyala Player）
  • Ooyalaの埋め込みコードを埋め込むWebページ。
注意： FMLEなどのオーディオエンコーディングソフトウェアをインストールすると、HTTP接続を通じてAkamaiなどのコンテンツ配信ネットワーク（CDN）にライブストリームをプッシュできます。 CDNでライブストリーミングが有効になっている必要があります。

HTTP用のオーディオライブストリームを作成するには：

  1. コンピューターに接続（または再生）するオーディオソースを設定します。

  2. 専用のオーディオストリームを作成するには、ソフトウェアエンコーダ（Adobe FMLE、Windows Mediaエンコーダ、Ogg Vorbisなど）を介してライブストリーミングイベントを実行します。

    注意： FMLE 3.2を使用している場合は、フォーマットをmp3からoggに変更する必要があります。 さらに、HTTPを使ってストリーミングできることを確認してください。

  3. URLをホストするストリーミングサーバーまたはCDN（Akamaiなど）にオーディオストリームをプッシュするようにソフトウェアエンコーダーを設定します。 ソフトウェアエンコーダコンピュータから次の情報が必要です。

    • エンコーダのホスト名またはIPアドレスとポート番号。
    • ユーザー名とパスワード（ユーザー名は必要ないかもしれません）
    • マウントポイント（Flumotion HTTP Audio Streaming）。
    • ライブストリームの名前（ソフトウェアエンコーダで指定）。

  4. オーディオストリームをメディアサーバーまたはCDNにプッシュするには、メディアサーバーまたはCDNのURLを取得し、それをソフトウェアオーディオエンコーダーで指定します。 たとえば、Adobe FMLEエンコーダーでは、ストリーミングサーバーまたはCDN情報を FMSのURL フィールド。

    注意： ストリーミングサーバーまたはCDNのURL情報は、利用可能な管理ツールに応じてさまざまな場所にあります。 例えば、Flomotionメディアサーバーはこの情報を 統計 Flomotionの管理ツールのタブをクリックしてください。 出力 タブまたはダイアログボックス この情報を見つけるには、ソフトウェアソフトウェアに付属のマニュアルを参照してください。
  5. エンコード処理中に、エンコーダは個々のライブオーディオセグメントファイルのURLを含む1つ以上のインデックスファイルまたはマニフェストファイルを作成します。 インデックスは、生のオーディオ放送に対応するために定期的にリフレッシュすることができます。
  6. これらのファイルのライブストリーミングURLをBacklotに提供するには、ストリーミングサーバーまたはCDN管理ツールからURLを取得します。 ソフトウェアエンコーダのIPアドレス、ポート番号、およびライブストリーム名を指定して、ソフトウェアエンコーダからBacklotを直接プロビジョニングすることもできます。 この情報はすべてエンコーダソフトウェアで利用できます。

オーディオは圧縮され、Ooyalaの埋め込みプレーヤーリンクを介してストリームに接続するすべてのデバイスにHTTP経由で配信されます。 当社のオーディオプレーヤーはこのリンクを拾い上げ、ストリームを識別するURLに基​​づいてライブストリームをブロードキャストします。
