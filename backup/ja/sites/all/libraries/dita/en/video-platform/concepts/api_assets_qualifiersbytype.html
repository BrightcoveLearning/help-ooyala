資産のタイプと/ v2 / assetsとの相関関係

/ v2 / assets あらゆる種類の資産との取引をルーティングします。

注意： Backlot REST APIコマンドについて詳しくは、以下を参照してください。 バックスロットAPIリファレンス.

新しいアセットを作成すると、 asset_type リクエストボディのプロパティは、以下の値のいずれかを指定できます。

  • ビデオ
  • 広告：広告は単なる動画であり、広告では バックスロットUI.
  • remote_asset
  • youtube：YouTube向けのビデオ
  • live_stream
  • live_audio
  • チャンネル
  • channel_set

で資産を作成した後 [POST] / v2 / assets （に記載されているように 資産の作成：POST）、一般的にあなたは様々な方法でそれを更新することができます [パッチ] / v2 / assets。 一般化されたプロセスを見てください。 資産情報の編集：パッチ.

加えて、 / v2 / assets routeには、一般的なアセット、またはラベルの関連付け、公開ルール、広告セットなどの特定の種類のアセットを操作するための修飾子があります。 これらの修飾子の正確な使用法は他のセクションで詳述されています。
