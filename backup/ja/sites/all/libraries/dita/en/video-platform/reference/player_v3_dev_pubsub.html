作成、接続、確認、および読み込み

重要： Ooyala Player V3は非推奨であり、予定されている予定です。 無効 2018-01-31について その日以降、Player V3はあなたのビデオまたはオーディオコンテンツを再生しなくなります。 まだPlayer V3を使用しているお客様は、Player V4に移行する必要があります（を参照）。 Player V3からの移行 そして プレイヤー移行に関するFAQ).
プレーヤメッセージバスは、モジュールがイベント処理のためにメッセージをパブリッシュおよびサブスクライブするための手段を提供します。 それは使用しています OO.Player.create そして onCreate メッセージバスを作成するには、モジュールをメッセージバスに接続し、3rdパーティモジュールをチェックしてそれらを接続してからすべてをロードします。

OO.Player.create

呼び出されると、 OO.Player.create（） 関数：

  • 新しいメッセージバスを作成します。
  • 登録済みモジュールのリストを調べて、それらすべてを作成します。
  • 登録されているすべてのモジュールをメッセージバスに接続します。

onCreate

呼び出されると、 関数onCreate（プレーヤー）:

  • 追加モジュール（カスタム、3rdパーティー、またはその他のタイプ）がないか調べます。
  • これらの追加モジュールがメッセージバスに接続できるようにします。
  • 各モジュールに起動を通知するメッセージをメッセージバスに送信します。

あなたが呼び出す必要があります onCreate（） 何かが起こる前に。 それ以外の場合は、既存のモジュール、追加のモジュール、または他社製のモジュールはメッセージバスに接続されず、初期化されません。

Comparing Player V3 and V2 Event Handling

The simplest way to handle events in Player V2 is to define a callback function, then pass its name to the embed tag using the callback parameter. In the current player (Ooyala V3 Player), to handle events, specify an onCreate function that registers for messages in the embedded parameters to the OO.Player.create（） call. For example:

     
    function onCreate(player) {
       player.mb.subscribe('*','myPage', function(eventName) {});       
    }

    OO.Player.create('playerwrapper',embedCode, {
      onCreate: window.onCreate
    });
