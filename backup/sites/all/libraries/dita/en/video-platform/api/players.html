Players (Backlot REST API)

Note: For more information about REST API commands for the Ooyala player, see the Backlot API Reference.
  • Managing Players Using the Backlot REST API
    When you create a player and assign it to one or more assets in your account, it defines the playback options and the look and feel of your content.
  • Upload a Watermark Image Using the Backlot REST API
    You can upload a partially transparent image that displays during content playback.
  • Upload Player Scrubber Image for Player V3 (Deprecated)
    You can upload an image to show as viewers scrub through a video for Player V3.
  • Managing Player Custom Metadata Using the Backlot REST API
    You can add your own name-value pairs to any player.
  • Managing Player V3 Third-Party Modules Using the Backlot REST API (Deprecated)
    With third-party modules, you can associate additional features with V3 players.
  • Playlists
    A playlist consists of a list of movies. Playlists can be organized, and can be asset-based or label-based. Playlists are generally very lightweight and fast.
  • Cross-Device Resume: Getting the Playback Position Using the Backlot REST API
    This request is only one part of implementing cross-device resume (XDR).
