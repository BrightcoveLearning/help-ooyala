Ooyala API V1 (Deprecated) to API V2 Migration

The Ooyala V1 APIs had a limited set of functions. The Ooyala V2 APIs provide more control over your resources and are easier to update when new functionality is released.

The Ooyala V2 APIs follow Representational State Transfer (REST) conventions. Each resource has its own URL, has its own set of configurable parameters, and can be updated through standard HTTP methods. The following are the HTTP request methods you can use:

  • GET: view a resource
  • POST: create a new resource
  • DELETE: delete a resource
  • PUT: replace an existing resource
  • PATCH: update or modify an existing resource

Most HTTP libraries or tools allow you to specify any of these HTTP methods when making a request.

  • Attribute Update API (Deprecated)
    The attribute update API is now part of the asset API. You can perform a GET against an asset to return its data or a POST to update attribute settings.
  • Custom Metadata API (Deprecated)
    The custom metadata API is now part of the assets API. You can now view, update, replace, or delete custom metadata or an asset.
  • Label Management API (Deprecated)
    The new label management API has more features and is less complex than the Backlot API v1.
  • Player Management API (Deprecated)
    The old player management API enabled you to list information about a player and assign it to an asset. The new player API enables you to perform these tasks as well as create and modify player attributes and settings.
  • Channel API (Deprecated)
    The old channel API provided three modes that enabled you to list the contents of a channel, assign assets to a channel, or create a channel. The channel API is now part of the assets API which enables you to create, edit or delete a channel and create, edit, or delete a channel lineup.
  • Channel Set API (Deprecated)
    The old channel set API provided three modes that enabled you to list the contents of a channel set, assign channels to a channel set, or create a new channel set. The channel set API is now part of the assets API which enables you to create, edit or delete a channel set and create, edit, or delete a channel set lineup.
  • Query API (Deprecated)
    The differences between the new and old query APIs are beyond the scope of this section.
  • Thumbnail Query API (Deprecated)
    The thumbnail query API is now part of the asset API.
