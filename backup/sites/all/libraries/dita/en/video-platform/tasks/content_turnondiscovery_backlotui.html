Creating Discovery Profiles Using the Backlot UI

You can use the Backlot UI to create discovery profiles to use with defined players or on Discovery API requests.

To complete the following steps, you must have administrator privileges and your account must have Discovery enabled.
Note: Discovery must be enabled for your account for the Discovery tab to appear on the user interface. For more information about enabling Discovery, contact your Ooyala representative.

Although the profiles that you create in the Backlot UI appear to be intrinsically linked with individual players, the Player ID at the bottom of the page is the same as the programmatic profile_id that you use on the individual recommendation API requests, as shown in Managing Discovery Profiles with the Discovery API.

Use the following steps to associate a discovery profile with a player in Backlot.

  1. Sign on to the Backlot application using administrator credentials.
  2. Select Publish > Player Branding > Discovery.
  3. Configure the following Discovery options:
    Option Description
    Turn On Discovery For This Player Enable Discovery by checking this checkbox.
    Note: Even if you do not turn on this profile for the player, you can still use the profile on API requests.
    Discovery Interface Discovery recommendations are displayed after a video has been played. The message Up Next appears in the player with the title of the next video. Use the radio buttons to select from the following recommendation configuration options:

    • Up Next Video Plays - Enter up to 99 seconds in the selector.

    • immediately after the current video ends- Recommendations appear immediately.
      Note: Up Next does not appear during ad playback.
    Display Discovery Check this checkbox to enable Discovery recommendations during a player pause action. When this option is activated the Discovery "tray" (set of options and actions) appears when the player is paused.
    Discover Tray Click Behavior Use this option to customize the action invoked when the Discovery tray is selected. Select one of the following options from the dropdown menu.
    Note: Custom tray click behavior is currently not implemented on our HTML player. For HTML5, your Discovery recommended videos will open in the same page as the original video.
    • Open in New Page (default)
    • Open in Same Page
    • Open in Same Player
    Discover tray has a section for More Videos. For all settings except Open in Same Player, the videos list in More Videos must be defined as remote assets (assets that have a value for the property hosted_at in Backlot), either via the user interface or the API.
    1. For Promote content from a specific label to position, select the desired position (1 is first, 10 is last) and click Select a Label to choose it.
    2. For Promote sequential content with the custom metadata key, enter the name of the metadata key and for to position, select the desired position (1 is first, 10 is last).

      All assets (videos) can have metadata keys associated with them for your own special purposes. For information, see Custom Metadata.

    3. If you have sub-accounts and want the assets that they manage to be included in this profile, click the checkbox for Include sub-account.
    Discovery Editorial Controls To customize how Discovery recommendations are generated, configure the following:
    • Use the dropdown menu to select recommended content generation from:
      • Ooyala Discovery (default) - Select from the following options:
        • All labels (default)
        • Only from Certain Labels
        • Never from Certain Labels

          For the non default options, click Add Labels to select the labels to include and exclude.

      • A Specified Label
      • Shared Labels
    • Sort By - Use the dropdown to select one of the following specific label options:
      • Upload Time (Newest to Oldest)
      • Upload Time (Oldest to Newest)
    Player ID The profile ID to provide with the discovery profile on the recommendation API requests. For information about the end screen controls, see End Screen Controls.
