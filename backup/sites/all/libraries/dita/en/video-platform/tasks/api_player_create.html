Creating a Player Using the Backlot REST API

You can create as many players as you like and assign them to videos, channels, or channel sets.

Note: For more information about REST API commands for the Ooyala player, see the Backlot API Reference.

Backlot displays the player associated with the piece of content that the viewer accesses. For example, if the viewer accesses a video directly, the video is displayed in the video's player. If the customer accesses the same video through a channel, the video is displayed in the channel's player.

To create a player:

Use the players route. The following example creates a player that uses the default settings, but is set up for social sharing.
Note: Player sharing settings with the Backlot API are only valid for Ooyala Player V3. Customize Player V4 sharing settings by modifying skin.json or the player CSS. 
[POST]/v2/players{  
   "name":"Social Player",
   "ooyala_branding":{  
      "show_share_button":true,
      "twitter_sharing":true,
      "facebook_sharing":true,
      "digg_sharing":true,
      "url_sharing":true,
      "email_sharing":true
   }
}
