Supported Closed Captions

Before uploading closed caption files to Backlot, verify that you are using a supported language and file format.

Supported Closed Caption File Formats

You can upload closed caption files that use any of the following file formats.
Important: To ingest or upload a closed caption file, be sure to use the appropriate option (Ooyala Ingestion Service, Backlot UI, or Backlot API) that supports (✔) the format that you want to load into your Backlot account.
Note: If the source file uploaded is not of the same type as what the receiving device uses, styling added into the closed caption files is currently removed. For the Ooyala Web Player, WebVTT is used. For the Ooyala iOS and Android SDKs, TTML format is used. If the uploaded closed caption file type does not match, our closed caption conversion service converts the format to be compatible, but removes any style formatting in the conversion.
Extension Description OIS Backlot UI Backlot API
.TTML (formerly DXFP) TTML (Timed Text Markup Language file. Generally, a closed caption file will have a .dfxp (Distribution Format Exchange Profile) extension. For details, see Supported Closed Caption DFXP (now TTML) Format. ✔ ✔ ✔
.SCC SCC (Scenarist Closed Caption) file. ✔ ✔  
.SRT WebSRT (Web Subtitle Resource Tracks) file. ✔    
.VTT WebVTT (Web Video Text Tracks) file. ✔ ✔
