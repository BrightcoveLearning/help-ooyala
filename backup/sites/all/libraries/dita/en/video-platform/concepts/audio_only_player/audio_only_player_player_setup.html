Audio-Only Player: Player Setup

Default Audio-Only Player

Ooyala provides a default Audio-Only Player in addition to the existing default Video Player:

Any new Audio-Only Player you create based on the default one, initially has the following settings enabled by default:

  • Volume Bar - enabled by default from the Player configuration sub-tab.
  • Skip Forward Button - enabled by default from the Player configuration sub-tab.
  • Skip Backward Button - enabled by default from the Player configuration sub-tab.
  • Replay - visible in the Player skin after at least one full play, and enabled by default from the End Screen configuration sub-tab.

For Audio-Only Player, the Player setup process is identical to the one used before for Video Players (see Managing Players in Backlot). However, there are some differences, as follows:

  • An audio-specific Player is created, which gets identified as audio-only type.
  • The asset is processed for audio-only output.
Note: Customers cannot create default Audio-Only Players from the UI or using Player APIs.

Creating an Audio-Only Player

To create an Audio player, perform the following steps:

  1. Go to Backlot and navigate to the PUBLISH tab:

  2. Select Audio Player from the drop-down menu:

  3. Click the +NEW button:

    This action will create an Audio Player to use for any Audio-Only assets, and restrict the subsequent configuration options to only what is available to this Audio Player type.

    The new Player will be shown in the players list as Default Audio Player - Audio Player, as shown in the example below:

  4. Configure the Audio Player's properties in the Player Branding tab - it contains the Player, Branding, Start Screen, and End Screen sub-tabs. These naturally differ from the properties available for a Video Player type. The default Audio-Only Player comes with default settings so you can do this configuration at any time later.
    Note: Make sure that the relevant Audio Player is selected in the Player Selection list.

    • The Player sub-tab

    • The Branding sub-tab

    • The Start Screen sub-tab

    • The End Screen sub-tab

  5. The next step is to embed content for playback in your new Player. To do so, navigate to the MANAGE tab and select the (already imported) asset in the list:

  6. Open the Embed sub-tab:

  7. Select the relevant Audio Player from the Custom Player drop-down list:

    Note:

    Ooyala currently only supports two script-type embed code options - Standard and Advanced HTML5 Player embed code. iFrame embed code is not yet available for Audio-Only Player. The Standard and Advanced embed codes for Audio-Only Player have the same settings as for the Video Player, and we recommend using the Standard embed unless the customer needs broader configuration options on how the Player embed is constructed. The Standard embed option provides all the basic features such as CORE and Skin; the Advanced embed option contains all the scripts for additional configuration. For both options, in the embed code script, ensure that the playerParam is set to "playerType":"audio" (var playerParam = {"playerType":"audio"};). This must already be the case after you have selected an Audio Player for embedding the asset. so that an Audio Player is retrieved.
