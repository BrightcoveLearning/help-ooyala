Customizing the Receiver Application for the Ooyala Chromecast Integration

Important: This topic contains older information and has been deprecated. For the latest information about Ooyala integration with Chromecast, see Chromecast Integration.

The sample custom receiver is now open source. Go to https://github.com/ooyala/chromecast-sample-receiver and open receiver_custom.html.

You can customize many parts of the custom receiver application, including but not limited to:
  • Logo
    • watermark
    • TV ratings
  • Video metadata
    • title
    • description
    • duration
  • Player controls
    • play
    • pause
    • buffer bar
    • scrubber bar
  • Player state screens
    • splash screen
    • loading screen
    • player screen
    • error screens
  • Error Messages

Customizable parameters are described in the code comments of the receiver. We will describe some ways you can customize the receiver here, but you are free to customize it in any way you like. You are not limited to the customizations shown here.

The following is a simplified skeleton of the custom receiver application code: 
<html>
<head>
   (Load Chromecast scripts, other javascript files, css, etc.)
</head>
<body>
   (User Interface related HTML here)
   <script>
     window.onload = function() {
     // Set up the Cast Receiver Manager
     // Set up the Cast Message Bus
     // Start the Cast Receiver Manager
     }
     ...
   </script>
</body>
</html>
