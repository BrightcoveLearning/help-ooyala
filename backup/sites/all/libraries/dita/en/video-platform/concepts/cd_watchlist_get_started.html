Getting Started with Watchlists

Basic Syntax

To leverage the Watchlist REST API routes, start with the type of request and the base URI. This syntax is shown in the following example:

GET /watchlist/v3

REST Routes

You can use the following REST routes in the Watchlist API:

  • GET
  • POST
  • PUT
  • DELETE

Request Authentication

The Watchlist Service works with authenticated users only. The Watchlist REST API requires that you include the <account_token> parameter in each request, as shown in the following example. 

GET http://player.ooyala.com/watchlist/v3?account_token=<account_token>
The Watchlist Service uses the specified <account_token> to reference the watchlist associated with a particular end user (identified by their UUID). For more information, see Ooyala Account Token API.
