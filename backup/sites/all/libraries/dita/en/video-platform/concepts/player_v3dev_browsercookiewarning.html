Warning About Web Browser Third-party Cookies

Note: Applies only when using device registration with Adobe Access DRM or if you are not using an Ooyala player.

Unblock All Cookies in Browser Settings

Some browsers disallow third-party cookies by default. This can interfere with your Ooyala Player Token and Player Authorization API requests under some circumstances.

For both Ooyala Player V3 and Player V4, when using device registration with Adobe Access DRM, OPT and authorization rely on setting cookies in a viewer's browser on behalf of the provider (you). Such cookies are called "third-party cookies".

Also, if you are not using an Ooyala player, you might require cookies in order for authorization and DRM to work.

If you are using device registration with Adobe Access DRM or if you are not using an Ooyala player, in your viewers' web browser settings, be sure to unblock all restrictions against cookies. For example, in Apple Safari,in the Safari > Preferences dialog, set Block cookies to Never:
