Cast and Play vs. Play and Cast

Important: This topic contains older information and has been deprecated. For the latest information about Ooyala integration with Chromecast, see Chromecast Integration.

A Cast session can start with either the "cast and play" or "play and cast" method. The following flows show the difference between “cast and play” and “play and cast”, both of which are supported in the Ooyala Chromecast Integration.

Cast & Play

  1. User taps Cast Button.
  2. User selects Receiver device from Cast menu.
  3. User selects content to Play.

    Content starts playing on Receiver screen.

Play & Cast

  1. User selects content to Play.

    Content starts playing on Sender device.

  2. User taps Cast Button.
  3. User selects Receiver device from Cast menu.

    Content starts playing on Receiver screen and is paused on the Sender device.

In the case where there are multiple senders, the last sender always gets the controls and starts casting a new app from scratch. The old video will get disconnected if the new sender plays a different video.
