Backlot REST API

Use the Backlot REST API to customize the chrome (design elements) of your player. This includes UI elements such as fonts, colors, scrubber bar and so forth.

Important: The topics in this section are being migrated to the Ooyala API Docs page.
  • About the Backlot REST API
    Welcome to the Backlot REST API.
  • HTTP Response Codes
    Depending on request results, the Backlot REST API returns different HTTP response codes.
  • Assets
    The /v2/assets route deals with all types of assets.
  • Labels
    When added to external publishing targets, labels specify which videos, channels, and channel sets to publish externally. Additionally, labels are also useful for organizing your video library, searching for videos, and retrieving targeted analytics.
  • Players (Backlot REST API)
  • Ad Sets
    Ad sets are ad configurations that control the ad experience of an asset during playback.
  • Publishing Rules
    Publishing rules specify when and where your content can be played. With publishing rules, you can restrict your content to specific geographic regions, ensure that it is only embedded on specific domains, and only allow it to be viewed at specific times.
  • Configurable DRM
    Digital Rights Management (DRM) enables you to control how viewers consume and access your high value content through DRM policies. DRM policies can be set at the account or asset level.
  • Syndications
    A syndication consists of a list of content and a destination to which you want to syndicate the content.
  • Country and Location Codes
    These country codes are used in a variety of Ooyala features.
