2012-04-16 Release

Updates include new Backlot SDKs, new mobile SDKs, correct processing of 5.1 channel audio, support for negative timestamps, a fix for storing audio assets in multiple locations, and a fix for creating audio-only HLS streams.

The following are release items:

  • New Backlot SDKs— we now have Backlot SDKs in five programming languages (Java, PHP, Python, Ruby, and C#).
  • New Mobile SDKs—The Android, iOS, and Adobe AIR SDKs are now available.
  • 5.1 channel audio now processes correctly—Previously, 5.1 channel audio did not process correctly. Now, you can upload 5.1 channel video and audio files, which are automatically converted to stereo. This is an automatic change and does not require any action on your part.
  • Negative timestamp support—Previously, negative videos with negative timestamps (typically iOS devices) were rejected during transcoding. We now support videos that have negative timestamps. This is an automatic change and does not require any action on your part.
  • Fix for storing audio assets in multiple locations—For customers that used non-Ooyala storage, audio-only assets did not process correctly. Now, audio-only assets are supported for all customers. This is an automatic change and does not require any action on your part.
  • Fix for creating audio-only HLS streams—Video assets that had no audio track would fail for HLS streams. These assets are now processed correctly. This is an automatic change and does not require any action on your part.
