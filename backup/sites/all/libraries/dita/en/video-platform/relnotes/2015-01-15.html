2015-1-15 Release

Notable changes for this release include the Android and iOS Mobile SDK 3.3. releases and Hook update to support supports Android 5.0 (Lollipop).

Hook Updates for Android 5.0 Support

Hook is now updated to provided support for Android 5.0 (Lollipop). To start using this version the user must update the Hook App.

Known Issues

Video playback can sometimes be interrupted if the closed caption button is clicked twice during a video. If this occurs the video can be resumed by activating the closed caption button again.

Mobile SDK for Android 3.3.0 Released

Release 3.3.0 Release Date 1/15

Updates and Enhancements

  • The SDK is updated to supported FreeWheel version 5.18.0.

  • The SDK is updated to supported Google IMA version Beta9.

  • The SDK is enhanced to improve load time for pre-roll ads. Options are now available for preload and promoimage.

Resolved Issues

The following issues are resolved in this release:

  • Closed captions were sometimes obfuscated by the scrubber bar.

  • Closed captions were not displaying at intervals in the video playback.

  • Videos sometimes play before a play event is activated.

  • Closed captions were not appearing on Android devices.

Known Issues

Please review the following known issues:

For Google IMA ads, if the same ad is served on multiple ad slots, and the user skips the ad on the first ad slot, the second ad resumes from the skipped position.

Mobile SDK for iOS 3.3.0 Released

Release 3.3.0 Release Date 1/15

Updates and Enhancements

  • The SDK API documentation in now found in the SDK APIDocs directory and the SDK release notes are included in the readme.txt file.
  • The SDK is enhanced to improve load time for pre-roll ads. Options are now available for preload and promoimage.

Resolved Issues

The following issues are resolved in this release:

Post-roll ads did not play properly on AirPlay.

Known Issues

Please review the following known issues:

Google IMA : If a user seeks to the end of a video while there is an available mid-roll ad, the mid-roll ad will play audio only before the video content completes.

Backlot Paywall Deprecated

Ooyala has deprecated the legacy Backlot Paywall feature in favor of third-party paywall offerings. For a list of supported paywalls see Paywall Integration.

YuMe Adset Migration

YuMe now uses the VPAID OPF module for Flash players. You need to migrate all of your content using the YuMe ad source to the VPAID ad source in Backlot. For a guide on how to migrate your YuMe ad sets to the VPAID Ad Source, see Migrating from YuMe to VPAID Ad Source.

Documentation Release Items

  • File Name Formatting Update
    • Ingesting with FTP / FTPS - Added documentation detailing how file names should be formatted for optimal efficiency when uploading assets.
  • Export Your Data With the Analytics API
    • How to Export Your Data With the v2 Analytics API (Deprecated) - Added a guide to getting and saving your analytics data through the V2 APIs before the migration to V3 APIs.
  • Legacy Backlot Paywall Documentation Archived - Ooyala has archived the documentation for the deprecated Backlot Paywall, including the functional overview and tasks for creating, editing, and deleting Backlot paywalls.
